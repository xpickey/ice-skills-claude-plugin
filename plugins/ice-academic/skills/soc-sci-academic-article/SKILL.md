## SECTION 1: นิยามและขอบเขต

### บทความวิชาการ คืออะไร
บทความที่นำเสนอ **การวิเคราะห์ / วิจารณ์ / สังเคราะห์** ความรู้เชิงทฤษฎี โดยอ้างอิงแหล่งวิชาการที่น่าเชื่อถือ มีคุณูปการต่อองค์ความรู้ใน 5 สาขาของวารสาร

### 3 ประเภทย่อย
- **TYPE-A: Analytical Article** — วิเคราะห์ประเด็นผ่าน Theoretical Lens เช่น "ทฤษฎี NPM อธิบายการปฏิรูประบบ GFMIS ได้อย่างไร"
- **TYPE-B: Critical Review Article** — วิจารณ์/ประเมินทฤษฎีหรือนโยบาย เช่น "ข้อจำกัดของ Good Governance ในบริบท อปท. ไทย"
- **TYPE-C: Conceptual Synthesis** — สังเคราะห์กรอบแนวคิดใหม่ เช่น "บูรณาการทศพิธราชธรรมกับ Digital Government Framework"

### สิ่งที่บทความวิชาการ "ไม่ใช่"
ไม่มี Research Methodology / Population / Sample / Data Collection / Statistical Analysis / Findings จากการเก็บข้อมูลจริง — หากผู้ใช้ขอเนื้อหาลักษณะนี้ ให้แนะนำปรับเป็นการวิเคราะห์เชิงทฤษฎีแทน

### 5 สาขาที่วารสารรับ
1. พระพุทธศาสนาประยุกต์กับสังคม
2. รัฐศาสตร์และการปกครอง
3. รัฐประศาสนศาสตร์และการบริหารสาธารณะ
4. บริหารธุรกิจและการศึกษา
5. สังคมศาสตร์ประยุกต์

## SECTION 3: Argument Building Framework

ทุกย่อหน้าวิเคราะห์ใช้ 4 ขั้นตอน:

1. **CLAIM** — ระบุข้อเสนอที่ต้องการพิสูจน์
2. **THEORY** — เลือก Theoretical Lens ที่เหมาะสม
3. **EVIDENCE** — อ้างอิง Scholarly Sources สนับสนุน
4. **SYNTHESIS** — สรุปว่า claim ได้รับการยืนยันอย่างไร พร้อมชี้ข้อจำกัด

### Argument Patterns ที่ใช้บ่อย
- **Pattern A**: "[ทฤษฎี X] อธิบาย [ปรากฏการณ์ Y] ในบริบท [Z] ได้อย่างไร"
- **Pattern B**: "เปรียบเทียบ [A] กับ [B] ภายใต้กรอบ [ทฤษฎี C] พบว่า..."
- **Pattern C**: "หลักธรรม [X] สอดคล้องกับ [ทฤษฎีสากล Z] ในแง่มุม..."

### เทคนิคการเขียนย่อหน้า (จากคู่มือ ดร.สมชาย)
- ย่อหน้าละ 1 ประเด็น: ประโยคหลัก → ขยายความ → อ้างอิง → สรุป
- ใช้คำเชื่อม: ประการแรก/ประการต่อมา, กล่าวอีกนัยหนึ่ง, ในทำนองเดียวกัน, ตรงกันข้าม, ดังนั้น/ด้วยเหตุนี้, เมื่อพิจารณาโดยสรุป

## SECTION 4: คุณภาพวิชาการ (7Cs + Scholarly Balance)

บทความต้องผ่านเกณฑ์ 7Cs:
1. **Correct** — ข้อมูลถูกต้อง ไม่ hallucinate กฎหมาย/มาตรา/อ้างอิง
2. **Complete** — ครบทุกองค์ประกอบตามโครงสร้าง
3. **Clear** — ภาษาวิชาการกระชับ ชัดเจน
4. **Concise** — ≤8,000 คำ ไม่เยิ่นเย้อ
5. **Consistent** — ใช้ศัพท์สม่ำเสมอทั้งบทความ
6. **Convincing** — Argument มีน้ำหนักทางวิชาการ
7. **Current** — อ้างอิงงานวิจัยย้อนหลังไม่เกิน 5–10 ปี (ยกเว้นทฤษฎีคลาสสิก)

**Scholarly Balance** — ทุก argument ต้องมี counterargument หรือข้อจำกัด ห้ามนำเสนอเพียงด้านเดียว

## SECTION 5: Workflow — ขั้นตอนการทำงาน

### STEP 1: รับโจทย์และจำแนกประเภท
- ถามผู้ใช้: หัวข้อ/ประเด็น, สาขาเป้าหมาย (1 ใน 5), ความต้องการเฉพาะ
- จำแนก TYPE-A / TYPE-B / TYPE-C
- หากโจทย์เข้าข่าย Research Article → แนะนำปรับเป็นวิเคราะห์เชิงทฤษฎี

### STEP 2: เลือกทฤษฎีและวางกรอบ
- อ่าน `references/theoretical-frameworks.md` เพื่อเลือกทฤษฎี ≥2–3 ทฤษฎี
- ถ้าเป็นหัวข้อบูรณาการพุทธ อ่าน `references/domain-buddhist-framework.md`
- นำเสนอ Theoretical Framework ให้ผู้ใช้ยืนยัน

### STEP 2.5: ค้นหางานวิจัยภายนอก (ถ้าจำเป็น)
- ⚠️ **ต้องถามผู้ใช้ยืนยันก่อนค้นหาภายนอกเสมอ** — ไม่ค้นหาโดยอัตโนมัติ
- อ่าน `references/academic-research-guide.md` สำหรับกลยุทธ์และเทคนิคการค้นหา
- ใช้เมื่อ: เนื้อหาใน reference files ไม่เพียงพอ / ต้องการข้อมูลล่าสุด / ต้องการเสริม evidence
- ดู SECTION 8 สำหรับรายละเอียดกระบวนการค้นหา

### STEP 3: ร่างโครงสร้างเนื้อหา
- ร่าง Outline ตาม Word Budget Distribution
- ระบุแต่ละประเด็นวิเคราะห์ (3–4 ประเด็น) + ทฤษฎีที่ใช้
- ประมาณการ word count แต่ละส่วน

### STEP 4: เขียนเนื้อหา
- เขียนทีละส่วนตามลำดับ: บทคัดย่อ → บทนำ → ทฤษฎี → เนื้อหาวิเคราะห์ → บทสรุป
- ใช้ Argument Building Framework (CLAIM → THEORY → EVIDENCE → SYNTHESIS)
- แจ้ง [word count โดยประมาณ] ท้ายแต่ละส่วน

### STEP 5: จัดรูปแบบและอ้างอิง
- อ่าน `assets/apa7-thai-examples.md` สำหรับรูปแบบอ้างอิง
- อ่าน `assets/word-template-spec.md` สำหรับการจัดหน้า
- ตรวจ APA7 ทุกรายการ

### STEP 6: ตรวจสอบก่อนส่ง
- อ่าน `references/article-checklist.md` แล้วรันทุกข้อ
- ตรวจ Word Count ≤ 8,000
- ตรวจ blind review (ไม่มีข้อมูลระบุตัวตนใน manuscript)

## SECTION 6: Boundary Rules

**RULE 1 — Type Guard**: เมื่อผู้ใช้ขอเนื้อหาที่เป็น Research Article (Methodology, แบบสอบถาม, SPSS) ให้แจ้งสุภาพว่าหัวข้อนี้เหมาะกับบทความวิจัย และแนะนำวิธีปรับเป็นบทความวิชาการ

**RULE 2 — Word Count**: ทุกครั้งที่ร่างเนื้อหา ให้แจ้ง word count ที่ประมาณการสำหรับแต่ละส่วน

**RULE 3 — Source Integrity**: ห้าม hallucinate มาตรา กฎหมาย หรือ reference ที่ไม่มีจริง ข้อมูลกฎหมายต้องระบุแหล่งที่มา

**RULE 4 — Scholarly Balance**: ทุก argument ต้องมี counterargument หรือข้อจำกัด

## SECTION 8: Academic Research — ค้นหางานวิจัยและเอกสารอ้างอิงภายนอก

### หลักการสำคัญ
บทความวิชาการต้องอ้างอิง scholarly sources ≥15 แหล่ง Skill นี้มีความสามารถในการค้นหางานวิจัย บทความ และเอกสารอ้างอิงจากแหล่งภายนอก เพื่อเสริมความแข็งแกร่งให้กับ evidence base ของบทความ

### ⚠️ Consent-First Protocol
**ต้องถามผู้ใช้ยืนยันก่อนค้นหาภายนอกทุกครั้ง** ไม่ค้นหาโดยอัตโนมัติ

เมื่อเนื้อหาใน reference files ไม่เพียงพอ หรือต้องการข้อมูลเพิ่มเติม ให้ถามผู้ใช้:
> "สำหรับประเด็น [X] ข้อมูลใน reference files ของ skill ยังไม่เพียงพอ ต้องการให้ค้นหางานวิจัย/เอกสารอ้างอิงเพิ่มเติมจากแหล่งภายนอกไหมครับ?"

### เมื่อผู้ใช้สั่งให้ค้นหา หรือยืนยันแล้ว

**ขั้นตอนการค้นหา** (อ่านรายละเอียดใน `references/academic-research-guide.md`):

1. **วางแผนคำค้น** — กำหนด Keywords ทั้งไทยและอังกฤษ (3–5 ชุดคำค้น)
2. **ค้นหาหลายแหล่ง** — ใช้ WebSearch ค้นจาก:
   - ThaiJO (วารสารไทย TCI)
   - Google Scholar
   - แหล่งเฉพาะทาง (ตามโดเมน)
3. **กรองและประเมิน** — ตรวจสอบ relevance, credibility, recency
4. **สังเคราะห์ผลค้นหา** — สรุปผลเป็น Research Synthesis Report
5. **จัดทำบรรณานุกรม** — APA 7th ทุกรายการที่ใช้

### ใช้ Claude Research Mode เมื่อ:
- ผู้ใช้สั่งค้นหาอย่างชัดเจน ("ช่วยค้นหางานวิจัยเกี่ยวกับ...")
- ต้องการ literature review เชิงลึก
- ต้องการข้อมูลล่าสุด/สถิติปัจจุบัน
- ต้อง verify ข้อมูลที่อ้างอิง

### Research Synthesis Output Format
เมื่อค้นหาเสร็จ นำเสนอผลลัพธ์ในรูปแบบ:
