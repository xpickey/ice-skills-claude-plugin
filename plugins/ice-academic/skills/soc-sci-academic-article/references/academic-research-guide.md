# Academic Research Guide — ค้นหางานวิจัยและเอกสารอ้างอิงภายนอก

คู่มือการค้นหา สังเคราะห์ และประเมินงานวิจัยเพื่อเป็นข้อมูลในการเขียนบทความวิชาการ
ใช้เมื่อผู้ใช้สั่งให้ค้นหา หรือยืนยันการค้นหาภายนอกแล้วเท่านั้น

---

## 1. Consent-First Protocol — ขออนุญาตก่อนค้นหาเสมอ

### เมื่อไหร่ต้องถาม
- เมื่อ reference files ใน skill ไม่มีข้อมูลเพียงพอสำหรับประเด็นที่วิเคราะห์
- เมื่อต้องการสถิติ/ข้อมูลล่าสุดที่ไม่มีในคลังความรู้
- เมื่อต้องการ verify ข้อมูลกฎหมาย มาตรา หรือ reference ที่ไม่แน่ใจ
- เมื่อต้องการ literature review เพิ่มเติมเพื่อเสริม evidence base

### ตัวอย่างคำถามยืนยัน
```
"สำหรับประเด็น [ธรรมาภิบาลเชิงพุทธกับ AI Governance] ข้อมูลใน reference files
ยังไม่เพียงพอ ต้องการให้ค้นหางานวิจัยเพิ่มเติมจากแหล่งภายนอกไหมครับ?
ผมจะค้นหาจาก ThaiJO และ Google Scholar ด้วยคำค้น:
  • 'ธรรมาภิบาลเชิงพุทธ AI' (ไทย)
  • 'Buddhist governance artificial intelligence' (อังกฤษ)"
```

### เมื่อไม่ต้องถาม (ค้นหาได้เลย)
- ผู้ใช้สั่งอย่างชัดเจน เช่น "ช่วยค้นหางานวิจัยเกี่ยวกับ..." / "search for..."
- ผู้ใช้บอก "ค้นหาได้เลย" / "ใช้ Research mode"

## 2. Research Strategy — กลยุทธ์การค้นหา

### 2.1 วางแผนคำค้น (Query Planning)

> 🔎 **2.1.0 สร้างคำค้นด้วย PICO (ตัวช่วยคิดก่อนวางชุดคำค้น · ดัดแปลงสาย วช.)** — *เป็นขั้นตอนคิดเพื่อให้คำค้นครบ-คม ไม่ใช่เนื้อหาที่เขียนลงบทความ; ผลลัพธ์ = เติม "ชุดคำค้น 3–5 ชุด" + ตาราง Core/Context ด้านล่าง*
> 1. **ตีความ PICO/PICo จากหัวข้อ** — P (ประชากร/หน่วยศึกษา) · I (สิ่งที่ศึกษา/ปรากฏการณ์) · C (เปรียบเทียบ — *ถ้าไม่มี ปล่อยว่าง อย่าแต่งให้มี*) · O (ผลลัพธ์) · Context (บริบทไทย/พุทธ/พื้นที่)
> 2. **แตกแต่ละองค์ประกอบเป็นคำกว้าง ↔ คำแคบ + คำพ้อง/สะกดหลายแบบ 2 ภาษา** (เช่น "ผู้สูงอายุ/ผู้สูงวัย"; "ดิจิทัล/ดิจิตอล/digital"; "ธรรมาภิบาล/good governance") — กว้างไว้ค้นรอบแรก แคบไว้คัดกรอง
> 3. **(ออปชัน) ต่อ Boolean** — `(คำพ้อง1 OR คำพ้อง2) AND (...)` + คำตัดออก (NOT) กัน false positives; ปรับ syntax ตาม ThaiJO / Google Scholar จริง
> ⚠️ **Guard:** คำค้น = *สมมติฐานการสืบค้น* — เอกสารจริงต้องมาจากการค้น (สอดคล้อง §6 Source Integrity) · ไม่แน่ใจคำเทียบเคียง = ระบุ "ต้องตรวจโดยผู้วิจัย"
>
> *PICO ที่แตกแล้ว → map เข้าตาราง Core/Context/Theory/Modifier ด้านล่างได้ทันที (P/Context → Core+Context · I → Core · O → Modifier-ผลลัพธ์)*

สร้างคำค้น 3–5 ชุด ทั้งไทยและอังกฤษ ตามโครงสร้าง:

| องค์ประกอบ | ตัวอย่าง |
|---|---|
| **Core Concept** | ธรรมาภิบาล, Good Governance |
| **Context/Scope** | ภาครัฐไทย, Thai public sector |
| **Theory/Lens** | ทศพิธราชธรรม, Buddhist governance |
| **Modifier** | วิเคราะห์, บทวิจารณ์, สังเคราะห์, framework |

**ตัวอย่างชุดคำค้น:**
```
ชุด 1 (ไทย — กว้าง):     "ธรรมาภิบาล ภาครัฐ บทความวิชาการ"
ชุด 2 (ไทย — เฉพาะ):    "ทศพิธราชธรรม Good Governance อปท."
ชุด 3 (อังกฤษ — กว้าง):  "good governance public administration Thailand"
ชุด 4 (อังกฤษ — เฉพาะ):  "Buddhist governance framework local government"
```

## 3. Claude Research Mode — ใช้ Extended Thinking + Multi-Source Search

### เมื่อไหร่ใช้ Research Mode
- ค้นหาจากหลายแหล่งพร้อมกัน
- ต้องการ comprehensive literature review
- ต้องการตรวจสอบข้อมูลข้ามแหล่ง (cross-reference verification)

### กลยุทธ์ Research Mode

**Phase 1: Scout Search (ค้นหาเบื้องต้น)**
- ใช้ WebSearch ค้นด้วยคำค้นกว้าง 1–2 ชุด
- ประเมินจำนวนผลลัพธ์และความเกี่ยวข้อง
- ปรับคำค้นตามผลที่ได้

**Phase 2: Deep Search (ค้นหาเชิงลึก)**
- ค้นหาเฉพาะทางด้วยคำค้นแคบ 3–5 ชุด
- ใช้ WebFetch อ่านบทคัดย่อ/เนื้อหาจากแหล่งที่เกี่ยวข้อง
- เก็บข้อมูล: ชื่อบทความ / ผู้แต่ง / ปี / วารสาร / สาระสำคัญ

**Phase 3: Synthesis (สังเคราะห์)**
- จัดกลุ่มผลลัพธ์ตามประเด็น/ทฤษฎี
- ระบุ consensus / debate / gap ในวรรณกรรม
- จัดทำ Research Synthesis Report

### ตัวอย่างการใช้ WebSearch

```
# ค้นหาวารสารไทยจาก ThaiJO
WebSearch: "ธรรมาภิบาล New Public Management ภาครัฐไทย site:so03.tci-thaijo.org"

# ค้นหา Google Scholar ภาษาอังกฤษ
WebSearch: "good governance digital transformation public sector Thailand scholarly"

# ค้นหากฎหมาย/ระเบียบ
WebSearch: "พรบ.จัดซื้อจัดจ้าง 2560 site:ratchakitcha.soc.go.th"

# ค้นหาวิทยานิพนธ์ มจร
WebSearch: "ธรรมาภิบาลเชิงพุทธ site:tdc.thailis.or.th มหาจุฬาลงกรณราชวิทยาลัย"
```

## 4. การประเมินแหล่งอ้างอิง (Source Evaluation)

### เกณฑ์ CRAAP Test (ปรับสำหรับบทความวิชาการไทย)

| เกณฑ์ | คำถาม | เกณฑ์ผ่าน |
|---|---|---|
| **Currency** (ความทันสมัย) | ตีพิมพ์เมื่อไหร่? | ≤5 ปี (ยกเว้นทฤษฎีคลาสสิก/กฎหมาย) |
| **Relevance** (ความเกี่ยวข้อง) | ตรงกับประเด็นที่วิเคราะห์? | ตรงหรือใกล้เคียงมาก |
| **Authority** (ความน่าเชื่อถือ) | ผู้เขียนมีคุณวุฒิ/สังกัดที่น่าเชื่อ? | มีตำแหน่งวิชาการ/สังกัดสถาบัน |
| **Accuracy** (ความถูกต้อง) | ข้อมูลมีหลักฐานสนับสนุน? | มี references/methodology ชัดเจน |
| **Purpose** (วัตถุประสงค์) | เป็นงานวิชาการ ไม่ใช่ opinion/blog? | ตีพิมพ์ในวารสาร TCI/Scopus/WoS |

### ลำดับชั้นความน่าเชื่อถือของแหล่งอ้างอิง

```
ระดับ 1 (สูงสุด): วารสาร Scopus/WoS Q1-Q2, IPSAS/IFRS Standards
ระดับ 2 (สูง):    วารสาร TCI Tier 1, วารสารต่างประเทศ Peer-reviewed
ระดับ 3 (ดี):     วารสาร TCI Tier 2 (รวม JSSR), ดุษฎีนิพนธ์/วิทยานิพนธ์
ระดับ 4 (ยอมรับ): รายงานหน่วยงานรัฐ (CGD/DGA/BOT), หนังสือวิชาการ
ระดับ 5 (ระวัง):  เว็บไซต์ข่าว, Working Papers → ใช้เฉพาะข้อมูลสถิติ/ข่าว
❌ ไม่ยอมรับ:      Blog, Wikipedia, Social Media, แหล่งไม่ระบุผู้เขียน
```

### เทคนิคการสังเคราะห์ (จาก Research Synthesis Patterns)

1. **Thematic Synthesis** — จัดกลุ่มตามประเด็น/ธีม
   - ดูว่า source หลายแหล่งพูดถึงประเด็นเดียวกันอย่างไร
   - ระบุ prevalence: "X จาก Y แหล่ง กล่าวถึงประเด็นนี้"

2. **Theoretical Mapping** — จัดกลุ่มตามทฤษฎี
   - ดูว่าแต่ละทฤษฎีมี evidence สนับสนุนกี่แหล่ง
   - ระบุ strength of evidence

3. **Chronological Mapping** — จัดลำดับตามเวลา
   - ดูวิวัฒนาการของแนวคิด/นโยบาย
   - ระบุ trend และ emerging themes

4. **Critical Appraisal** — ประเมินจุดแข็ง-จุดอ่อนของแต่ละแหล่ง
   - แยกข้อเท็จจริง (observation) จากการตีความ (interpretation)
   - ระบุ bias หรือข้อจำกัดเชิงวิธีการ

## 6. Source Integrity Rules — กฎเหล็กเรื่องแหล่งอ้างอิง

1. **ห้าม hallucinate** — ห้ามสร้าง reference ที่ไม่มีจริง ไม่ว่ากรณีใด
2. **ตรวจสอบซ้ำ** — ถ้าไม่แน่ใจว่า reference มีจริง ให้ค้นหา verify ก่อนอ้าง
3. **ระบุข้อจำกัด** — ถ้าเข้าถึง full-text ไม่ได้ ให้บอกผู้ใช้ตรงๆ
4. **อ้างต้นฉบับ** — อ้างต้นฉบับเสมอ ไม่อ้างจากแหล่งที่สอง (secondary citation) ยกเว้นจำเป็น
5. **APA7 ทุกรายการ** — ทุก reference ที่ค้นหามาต้องจัดรูป APA7 ให้ถูกต้อง
