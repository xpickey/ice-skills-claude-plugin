## 3. 24 AI Writing Patterns to Avoid

จาก CLAUDE.md V6.0 §10.5 — ตรวจทุก chapter

### 3.1 Content Patterns (P1-P10)

**P1. Inflated Symbolism**
- ✗ "stands as a testament to..."
- ✓ "proves..." / "shows..."

**P2. Notability Name-dropping**
- ✗ "the renowned scholar X..."
- ✓ "X (year) found that..."

**P3. Superficial -ing Analyses**
- ✗ "showcasing... highlighting... emphasizing..."
- ✓ "shows... highlights... emphasizes..." (active voice)

**P4. Promotional Language**
- ✗ "groundbreaking, revolutionary, transformative"
- ✓ "new" / "different" / "significant"

**P5. Vague Attributions**
- ✗ "many experts believe..."
- ✓ ระบุชื่อ + ปี

**P6. Formulaic Challenges**
- ✗ "however, challenges remain..."
- ✓ ระบุปัญหาเฉพาะ

**P7. Excessive AI Vocabulary** (ดู §2)

**P8. Copula Avoidance**
- ✗ "stands as" แทน "is"
- ✓ ใช้ "is" / "เป็น" ตรง ๆ

**P9. Negative Parallelisms**
- ✗ "not just X, but also Y"
- ✓ "X และ Y" / "ทั้ง X และ Y"

**P10. Rule of Three Abuse**
- ✗ ทุกประโยคมี 3 รายการ
- ✓ ใช้ 2, 4, 5 รายการตามจริง

### 3.2 Structural Patterns (P11-P18)

**P11. Synonym Cycling**
- ✗ เปลี่ยนคำใหม่ตลอด ("research" → "study" → "investigation" → "inquiry")
- ✓ ใช้คำหลักเดียวสม่ำเสมอ

**P12. False Ranges**
- ✗ "from X to Y" ที่ไม่มี continuum
- ✓ ระบุจริง

**P13. Em Dash Overuse**
- ✗ — — — มากเกินไป
- ✓ ใช้สลับกับ , ; () ตามบริบท

**P14. Boldface Overuse**
- ✗ **bold** ทุกประโยค
- ✓ Bold เฉพาะหัวข้อ/keyword สำคัญ

**P15. Inline-header Lists**
- ✗ "**Key Point:** ..." ทุกบรรทัด
- ✓ ใช้ paragraph form หรือ bullet เหมาะสม

**P16. Title Case Headings**
- ✗ "Section On Buddhist Integration"
- ✓ ใช้ "หลักพุทธธรรม" (ไทย) หรือ "Buddhist integration" (sentence case)

**P17. Emoji Decoration**
- ✗ ใช้ ⭐ 📚 🎓 ในงานวิชาการ
- ✓ ห้ามใน เอกสารสุดท้าย (allow ใน Skill MD ภายใน)

**P18. Curly Quotes Inconsistent**
- ✗ ใช้ " ... " และ " ... " ปนกัน
- ✓ ใช้ "..." อย่างสม่ำเสมอ

### 3.3 Voice Patterns (P19-P24)

**P19. Chatbot Artifacts**
- ✗ "Sure! Here's..." / "Great question!"
- ✓ ตอบเนื้อหาตรง ๆ

**P20. Knowledge-cutoff Disclaimers**
- ✗ "As of my last update..."
- ✓ ระบุปีเฉพาะ

**P21. Sycophantic Tone**
- ✗ "Excellent point!" / "Wonderful!"
- ✓ ตรงประเด็น ไม่อวดอ้าง

**P22. Filler Phrases**
- ✗ "It's worth noting that..."
- ✓ ตัดทิ้ง

**P23. Excessive Hedging**
- ✗ "may potentially possibly..."
- ✓ "may" หรือระบุชัด

**P24. Generic Conclusions**
- ✗ "In conclusion, X is multifaceted..."
- ✓ ระบุ key finding + impact เฉพาะ

## 4. Burstiness Rules

จาก CLAUDE.md V6.0 §10.1-10.4

### 4.1 Sentence Length Distribution

ในย่อหน้า 5-7 ประโยค ต้องมี

| ประเภท | ความยาว | จำนวน |
|--------|---------|------|
| ประโยคสั้นเด็ดขาด | 3-12 คำ | ≥ 1 |
| ประโยคยาว | 25-35 คำ | ≥ 1 |
| ประโยคกลาง | 15-22 คำ | ส่วนใหญ่ |
| Standard Deviation | — | ≥ 5 |

**ตัวอย่าง:**
```
[สั้น 5 คำ]: "ผลการวิจัยน่ากังวล"
[กลาง 18 คำ]: "ผู้วิจัยพบว่าบุคลากรร้อยละ 78 ขาดทักษะดิจิทัลพื้นฐานที่จำเป็นต่อการให้บริการประชาชน"
[ยาว 32 คำ]: "เมื่อพิจารณาในรายละเอียด พบว่าปัจจัยหลักที่ส่งผลต่อความสามารถในการเรียนรู้ดิจิทัลของบุคลากรคือแรงจูงใจภายในตามหลักฉันทะของอิทธิบาท ๔ ซึ่งสอดคล้องกับ Spencer และ Spencer (1993)"
[สั้น 6 คำ]: "นี่คือข้อค้นพบหลัก."
```

### 4.2 Sentence Opening Variety (≥ 5 รูปแบบในย่อหน้า)

จาก CLAUDE.md V6.0 §10.2

1. **Subject-First:** "การวิจัยพบว่า..."
2. **Adverbial:** "ในทางปฏิบัติ..."
3. **Subordinate:** "เมื่อพิจารณา..."
4. **Question:** "อะไรเป็นปัจจัยหลัก?"
5. **Quote/Reference:** "ตามที่ Smith (2566) ระบุ..."
6. **Prepositional:** "ภายใต้บริบทดังกล่าว..."
7. **Conditional:** "หาก... แล้ว..."

**กฎ:** ใช้ ≥ 5 รูปแบบในย่อหน้าหลัก ห้ามขึ้นต้นด้วย Subject ทุกประโยค

### 4.3 Paragraph Architecture (สลับขนาด)

| ขนาด | จำนวนประโยค | ใช้สำหรับ |
|------|------------|-----------|
| สั้น | 1-3 | Punchline / Conclusion |
| กลาง | 4-6 | Main Points |
| ยาว | 7-12 | Deep Analysis |

**Anti-Symmetry:** ห้าม
- ทุกย่อหน้ามีจำนวนประโยคเท่ากัน
- ทุกย่อหน้ามีโครงสร้าง Topic + 3 Support เหมือนกัน
- ทุก Bullet ความยาวเท่ากัน

## 5. Soul Techniques (6 ข้อ + Academic Calibration)

จาก CLAUDE.md V6.0 §11

### 5.1 S1 — Have Opinions (มีจุดยืน)

**Academic Calibration:** ใช้ในบทอภิปราย (5.2) — **ห้ามในบทนำ + วิธีวิจัย**

❌ "ผลการวิจัยน่าสนใจ มีนักวิจัยบางคนเห็นด้วย บางคนไม่เห็นด้วย"
✅ "ผู้วิจัยเอนเอียงไปทางที่สอง เพราะการสำรวจของ Smith (2566) พบรูปแบบเดียวกัน"

### 5.2 S2 — Vary Rhythm Aggressively (ปรับจังหวะหลากหลาย)

**Academic Calibration:** ใช้ใน Key Findings + Conclusions

✅ "ระบบนี้ใช้งานยาก. ผู้ใช้บ่นมาตลอดสามเดือนแรก แต่หลังจากเดือนที่สี่ ทีมงานเริ่มเข้าใจว่าปัญหาไม่ได้อยู่ที่ระบบ — และนั่นคือสิ่งที่ระบบไม่ได้แก้. มันแก้ได้แค่บางส่วน."

สังเกต: 5 คำ → 35 คำ → 5 คำ

### 5.3 S3 — Acknowledge Complexity (ยอมรับความซับซ้อน)

**Academic Calibration:** ใช้ใน Discussion + Limitation

✅ Academic: "ข้อค้นพบสนับสนุนสมมติฐานที่ ๑ ขณะเดียวกันก็ตั้งคำถามต่อกรอบทฤษฎีของ X"
✅ Business: "แม้ ROI สูง แต่ความเสี่ยงด้าน Adoption ยังเป็นจุดที่ทีมต้องบริหารให้ดี"

### 5.4 S4 — First-Person ("ผู้วิจัย")

**Academic Calibration:** ใช้ตลอด — ไม่หลบเลี่ยง

- "ผู้วิจัยจึงสนใจศึกษา..."
- "ผู้วิจัยขอเสนอ..."
- "ผู้วิจัยพบว่า..."
- "ผู้วิจัยสังเคราะห์..."

### 5.5 S5 — Let Some Mess In

**Academic Calibration:** ใช้น้อย — Academic ต้องการ Polish

✅ "ผลตอบแทนคาดการณ์ที่ ๑๒% ต่อปี แม้ตัวเลขดูดี แต่ไม่รวมความผันผวนของอัตราแลกเปลี่ยน ทำให้การเปรียบเทียบไม่ตรงประเด็นนัก"

**เกณฑ์:** ใช้ไม่เกิน 1-2 จุด/บท ไม่ใช้ใน Abstract / Methodology

### 5.6 S6 — Be Specific About Reactions

**Academic Calibration:** ใช้เต็ม โดยเฉพาะตัวเลขผลวิจัย

❌ "ผลการวิจัยน่ากังวล"
✅ "ผลที่น่ากังวลที่สุดไม่ใช่ตัวเลขที่ลดลง แต่เป็นการที่บุคลากรรุ่นใหม่ ๓ ใน ๕ คนระบุว่าไม่เคยเห็นแบบฝึกอบรมมาก่อน"

### 5.7 Soul Hard Rules (ห้าม)

- ✗ เติม Fake Personality
- ✗ Fabricate Emotions
- ✗ Counterargument ที่ไม่อิงงานวิจัยจริง
- ✗ First-Person Statements ที่ไม่ใช่ Facts ของผู้ใช้

## 6. Voice Profile Workflow

ผู้ใช้เลือก: **Default มาตรฐาน + Extract จากงานเขียนตนเอง + Synthesize จาก Reference Folder**

### 6.1 Mode A — Standard Default Profile

ใช้ Profile มาตรฐานสำหรับงาน ปร.ด. รปศ. มจร เมื่อ
- ผู้ใช้ยังไม่มี writing sample
- ผู้ใช้ขอ Default

**Standard Profile Spec:**
```
- First-person: "ผู้วิจัย" (consistent)
- Sentence length: SD ≥ 5, สั้น 3-12 คำ + ยาว 25-35 คำ
- Opening variety: ≥ 5 รูปแบบ/ย่อหน้า
- AI vocab density: ≤ 4/1,000 คำ
- Soul: S4 เต็ม + S6 เต็ม + S1, S3 ตามบท
- Connector: นอกจากนี้, ดังนั้น, เนื่องจาก, ทั้งนี้
- Tone: Academic Thai formal — สุภาพ ไม่อวดอ้าง
```

### 6.2 Mode B — Extract from User's Own Writing

ใช้เมื่อ ผู้ใช้มีงานเขียนของตนเองแล้ว → สกัด Voice Profile ของผู้ใช้

**Workflow:**
```
1. ผู้ใช้ส่งไฟล์งานที่เคยเขียน (ดุษฎีนิพนธ์เก่า / บทความ / รายงาน)
2. Claude วิเคราะห์
   - Common phrases ของผู้ใช้
   - Sentence length distribution
   - Opening pattern preference
   - Connector pattern
   - Vocabulary preference
3. สร้าง User Voice Profile
4. ใช้ Profile นี้ใน writing/checking งานใหม่
```

**Output Format (User Voice Profile):**
```yaml
user_voice_profile:
  preferred_opening_patterns:
    - subject_first: 35%
    - adverbial: 25%
    - subordinate: 20%
    - quote_reference: 15%
    - other: 5%
  sentence_length:
    mean: 18.5
    sd: 6.2
    short_pct: 15%
    long_pct: 12%
  preferred_connectors:
    - "ดังนั้น" (used 23 times)
    - "เนื่องจาก" (used 18 times)
    - "นอกจากนี้" (used 14 times)
  preferred_vocabulary:
    - "พิจารณา" (frequent)
    - "ระบุ" (frequent)
    - "วิเคราะห์" (very frequent)
  signature_phrases:
    - "เมื่อพิจารณาในภาพรวม..."
    - "ผู้วิจัยจึงสนใจ..."
    - "จากการศึกษาดังกล่าว..."
```

### 6.3 Mode C — Synthesize from Reference Folder

ใช้เมื่อ ผู้ใช้มี folder ของเอกสารหลายไฟล์ → สังเคราะห์ Human Footprint

**Workflow:**
```
1. ผู้ใช้ระบุ folder path
2. Claude อ่านทุกไฟล์ใน folder (.docx, .pdf, .md)
3. สังเคราะห์ Human Footprint:
   - คำสำคัญที่ใช้บ่อย
   - โครงสร้างประโยคที่ปรากฏซ้ำ
   - Tone & Register
   - Domain-specific vocabulary
4. สร้าง Synthesized Voice Profile
5. Save ใน state/voice-profile.yaml
```

### 6.4 Voice Profile Storage

เก็บใน `state/voice-profile.yaml` (จะร่างขั้นที่ 18)
- ใช้ข้าม session ได้
- Update ได้เมื่อมีงานใหม่
- Override Default เมื่อมี User Profile

### 6.5 Voice Profile Application

เมื่อ Claude เขียน → ตรวจ profile match
เมื่อ Claude ตรวจงาน → เทียบกับ profile + แจ้ง deviation

## 7. AI Detection Threshold

### 7.1 Default Threshold

**Moderate ≤ 30%** — ตามมาตรฐาน Turnitin AI ของ มจร ปัจจุบัน

### 7.2 Strictness Levels

| Level | AI Score | Use Case |
|-------|---------|----------|
| Aggressive | ≤ 5% | งานวารสารต่างประเทศ Top-tier |
| Standard | ≤ 15% | วารสารวิชาการระดับ Q1-Q2 |
| **Moderate** ⭐ | **≤ 30%** | **Turnitin AI ของ มจร — Default** |
| Loose | ≤ 50% | Working draft / ภายในเท่านั้น |

### 7.3 User Override

ผู้ใช้ระบุ threshold เฉพาะได้
```
"ขอใช้เกณฑ์ AI Score ≤ 15% เพราะวารสารที่จะส่งกำหนดเข้ม"
```
Skill จะใช้ threshold ที่ผู้ใช้ระบุแทน Default

## 8. Humanization Workflow

เมื่อ AI Detection score เกิน threshold → ใช้ Workflow นี้

### 8.1 Diagnosis (วิเคราะห์)

```
Step 1: Identify high-AI-probability sections
   - มาก่อนหน้า / ที่ไหนใน paragraph
   - Pattern ที่พบบ่อย (P1-P24)

Step 2: Categorize issues
   - Vocabulary (Banned list)
   - Structure (Burstiness ต่ำ)
   - Voice (ขาด first-person, ขาด Soul)
   - Pattern (24 patterns)
```

### 8.2 Remediation (แก้ไข)

```
Step 1: Replace AI vocabulary (per §2)
Step 2: Add Burstiness (per §4)
   - Insert ประโยคสั้นเด็ดขาด
   - Vary opening (≥ 5 patterns)
Step 3: Add Soul (per §5)
   - Inject "ผู้วิจัย" first-person
   - Add S6 specific reactions
   - Add S3 complexity acknowledgment
Step 4: Remove 24 AI patterns (per §3)
   - Remove inflated symbolism
   - Remove rule of three abuse
   - Fix em dash overuse
```

### 8.3 Re-Verify

```
Step 1: Run AI detection ใหม่
Step 2: ตรวจ Voice Profile match
Step 3: ตรวจ Academic register
Step 4: ถ้ายังเกิน threshold → repeat
```

### 8.4 Humanization Hard Rules

- ห้าม **เพิ่ม Fake Voice** เพื่อให้ผ่าน detection
- ห้าม **เปลี่ยนเนื้อหา** — เปลี่ยนเฉพาะการเขียน
- ห้าม **อ้างอิงใหม่** ที่ไม่มีต้นทาง
- ต้อง **คงความถูกต้องทางวิชาการ** ทุกการเปลี่ยน

## 9. Common Mistakes Library (8 Checkpoints)

**[CP-78] ขาด first-person "ผู้วิจัย"** [High]
- ✗ "ในการวิจัยนี้ จะทำการ..."
- ✓ "ในการวิจัยนี้ ผู้วิจัยทำการ..."
- *Source: Voi-G2 + S4*

**[CP-79] ใช้คำ AI ใน บทนำ** [CRITICAL]
- ✗ "ในยุคปัจจุบัน ปฏิเสธไม่ได้ว่า..."
- ✓ ระบุปี + ลบคำ AI
- *Source: §2.2 + Voi-G1*

**[CP-80] ขาด Burstiness (Sentence Length สม่ำเสมอ)** [High]
- ✗ ทุกประโยคยาว 18-22 คำ (SD < 3)
- ✓ ผสมสั้น 3-12 + ยาว 25-35 (SD ≥ 5)
- *Source: §4.1*

**[CP-81] ขาด Sentence Opening Variety** [High]
- ✗ ทุกประโยคขึ้นต้นด้วย Subject
- ✓ ≥ 5 patterns (Subject / Adverbial / Subordinate / Question / Quote)
- *Source: §4.2*

**[CP-82] ใช้คำ "แสดง" หน้าหัวข้อ** [Medium]
- ✗ "แสดงกรอบแนวคิด" / "แสดงโครงสร้าง"
- ✓ "กรอบแนวคิด" / "โครงสร้าง"
- *Source: XL-45*

**[CP-83] ตำแหน่ง "บูรณาการ/ประยุกต์" ผิด** [Medium]
- ✗ "Y บูรณาการ" / "Y ประยุกต์"
- ✓ "บูรณาการ Y" / "ประยุกต์ Y" (หน้า) หรือ "Y เชิงพุทธ" (กลาง)
- *Source: XL-47*

**[CP-84] AI Detection score เกิน threshold** [CRITICAL]
- ✗ Turnitin AI score > 30%
- ✓ ≤ 30% (Default) หรือตามที่ผู้ใช้กำหนด
- *Source: §7*

**[CP-85] Voice Profile ไม่สอดคล้อง** [High]
- ✗ บทที่ 1 ใช้สไตล์ A / บทที่ 2 ใช้สไตล์ B / บทที่ 5 ใช้สไตล์ C
- ✓ Voice profile match ตลอดเล่ม
- *Source: §6*

✅ Vocabulary: ไม่มีคำใน §2.1 + §2.2 (Critical AI signatures)
