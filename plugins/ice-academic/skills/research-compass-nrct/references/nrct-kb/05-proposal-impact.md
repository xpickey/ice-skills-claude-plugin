### 1.7 Research Utilization (RU) — เส้นทาง 6 ขั้น (สไลด์5 น.22)

## 2. จากนโยบายสู่โจทย์วิจัย

### 2.1 อ่านโครงสร้างประกาศทุน — ไล่ระดับ S-P-N-KR (สไลด์5 น.26-30)
รหัสโครงสร้างแผน ววน. 2566-2570: **4S** ยุทธศาสตร์ = **25P** แผนงาน = **15F** แผนงานสำคัญ = **56N** แผนงานย่อย = **46O** เป้าประสงค์ = **147KR** ผลสัมฤทธิ์สำคัญ (สไลด์5 น.15, 26)

ไล่ระดับตัวอย่างจริง (สไลด์5 น.28): ยุทธศาสตร์ → แผนงาน (เช่น P12 พัฒนานโยบายและต้นแบบสังคมคุณธรรม) → แผนงานย่อย (N19/N20) → ประเด็น → กลุ่มเรื่อง → KR
**แนวทางเลือกโจทย์ (ซ้ำทุกตัวอย่าง):** เลือกให้ตรงยุทธศาสตร์/แผนงานที่เปิดรับทุน → เลือกแผนงานย่อย (Nxx) ที่สัมพันธ์ปัญหา/ความเชี่ยวชาญ → พัฒนาโจทย์ให้ตอบเป้าหมายและ KR ที่กำหนด — **คติ: เลือกให้ตรงโครงสร้าง = ตอบโจทย์ถูกจุด = เพิ่มโอกาสได้รับทุน**

### 2.2 สิ่งที่กรรมการมองหา — 5 สิ่งสำคัญ (สไลด์5 น.31, 40 — ย้ำ 2 ครั้ง)
ไม่ใช่แค่ "โครงการดี" แต่คือ **"ตอบโจทย์ชัดเจน สร้างผลลัพธ์ได้จริง"**:
1. ตอบโจทย์ชัด (สอดคล้องแผน ววน. และ Program/Topic ที่ประกาศ)
2. นวัตกรรม/แนวคิดใหม่ (องค์ความรู้/เทคโนโลยี/วิธีใหม่ หรือพัฒนาให้ดีขึ้นอย่างเป็นรูปธรรม)
3. มีผลลัพธ์และผลกระทบชัดเจน (กำหนด KR และ Impact ที่วัดได้จริง)
4. เกิดประโยชน์กับกลุ่มเป้าหมาย (เข้าใจปัญหา/ความต้องการจริง)
5. แผนดำเนินงานเป็นไปได้จริง (ทรัพยากรเหมาะสม ทีมเข้มแข็ง บริหารความเสี่ยงได้)

### 2.3 การพัฒนาโจทย์วิจัย — New/Better/Different/Value (สไลด์5 น.32-34)
เกณฑ์ 4 ด้าน: **New** (ไม่เคยมีมาก่อน) · **Better** (ต่อยอดจากงานเดิม) · **Different** (เปิดช่องว่างวิจัยใหม่) · **Value** (นำไปสู่การขยายผล) — คำขวัญ: RESEARCH • IMPACT • INNOVATION

**กำหนดโจทย์ให้สอดคล้อง 4 ยุทธศาสตร์ (อววน.):**
- ยศ.1 พัฒนาเศรษฐกิจ: BCG / AI / เทคโนโลยีดิจิทัล / หุ่นยนต์ / EV / โลจิสติกส์ / SMEs-IDEs / การแพทย์ระดับสูง / ท่องเที่ยวเชิงคุณภาพ
- ยศ.2 ยกระดับสังคม-สิ่งแวดล้อม: สังคมสูงวัย / ความมั่นคงสุขภาพ / ลดความเหลื่อมล้ำ / Smart City / สังคมคาร์บอนต่ำ / ภัยพิบัติ
- ยศ.3 วิทยาศาสตร์ขั้นแนวหน้า (Frontier): ชีววิทยา/เทคโนโลยีชีวภาพ / วัสดุศาสตร์-นาโน / เทคโนโลยีอวกาศ / วิทยาการคำนวณ / สังคมศาสตร์-มนุษยศาสตร์ขั้นแนวหน้า
- ยศ.4 พัฒนากำลังคนและสถาบัน: การเรียนรู้ตลอดชีวิต / Hub of Talent / Hub of Knowledge อาเซียน / ระบบนิเวศวิจัย

### 2.4 กระบวนการ 8 ขั้น "จากนโยบายสู่โจทย์วิจัย" (สไลด์5 น.34-36)
สิ่งที่ต้องศึกษาก่อน: ยุทธศาสตร์ชาติ 20 ปี → แผนแม่บทภายใต้ยุทธศาสตร์ชาติ → แผนด้าน ววน. → Platform → Program → OKR → **Research Question** → **Impact**
**5 คำถามสำคัญ:** ประเทศกำลังมุ่งไปทางไหน? / ปัญหาใดเป็นวาระแห่งชาติ? / แหล่งทุนต้องการผลลัพธ์อะไร (OKR)? / งานวิจัยของเราตอบโจทย์ส่วนใด? / จะสร้างผลกระทบได้อย่างไร?

**ขั้นที่ 3 — วิเคราะห์ความต้องการผู้ใช้ (User Needs):** หลักการ "เริ่มต้นจากผู้ใช้ประโยชน์ ไม่ใช่เริ่มจากสิ่งที่นักวิจัยอยากศึกษา" — วิธีค้นหา: สัมภาษณ์เชิงลึก / แบบสอบถาม / สนทนากลุ่ม (FGD) / เวทีประชาคม / วิเคราะห์ข้อมูลเชิงพื้นที่ · กระบวนการ: User Needs → Research Question → Solution → Impact · **RESEARCHER NEEDS (มักไม่สอดคล้องปัญหาจริง) VS USER NEEDS (ตรงปัญหา นำไปใช้ได้จริง)**

### 2.5 ทำไมต้อง Integrative Research (สไลด์5 น.37)
เหตุผล: ปัญหาประเทศซับซ้อนเชื่อมโยงหลายมิติ → แก้ด้วยศาสตร์เดียวไม่ได้ → ต้องมีผู้ใช้ประโยชน์ร่วมออกแบบ → เกิดนวัตกรรมเชิงระบบและการเปลี่ยนแปลงยั่งยืน — 5 ภาคส่วนล้อมปัญหาเชิงยุทธศาสตร์: ภาครัฐ / ประชาชน / สถาบันการศึกษา-วิจัย / ภาคประชาสังคม (NGO/CSO) / ภาคเอกชน · ผลลัพธ์ที่คาดหวัง 5 เชิง: เศรษฐกิจ / นโยบาย / วิชาการ / สังคม / นวัตกรรม

### 2.6 วิเคราะห์ศักยภาพองค์กร — Capability Matching (สไลด์5 น.38-40)
6 ด้านที่ต้องวิเคราะห์: บุคลากร/ผู้เชี่ยวชาญ · ผลงานวิจัยที่ผ่านมา · ศูนย์วิจัย/ห้องปฏิบัติการ · เครือข่ายความร่วมมือ · เครื่องมือและฐานข้อมูล · ศักยภาพบริหารโครงการ
Formula: โจทย์วิจัย + ความเชี่ยวชาญองค์กร + ทรัพยากร → **HIGH MATCH** (โอกาสสำเร็จและผลกระทบสูง) — ระดับ: สูงมาก/ปานกลาง(เสริมจุดอ่อน)/ต่ำ(ปรับหรือเลือกโจทย์ใหม่) · วิเคราะห์ด้วย **SWOT** · **Research Readiness Model** 5 มิติ: People-Knowledge-Resource-Network-Management

## 3. นิยาม Research Gap / Pain Point / Problem Tree / Theory of Change

### 3.1 Research Gap ที่ถูกต้อง (สไลด์5 น.41)
**ไม่ใช่** "ยังไม่มีใครทำ" **แต่คือ** "ปัญหายังไม่ถูกแก้" — เข้าใจผิดนำไปสู่งานซ้ำซ้อน/รายละเอียดเล็กน้อย/ไม่มีคุณค่า ส่วนความจริงที่กรรมการมองหาคือ: ปัญหาสำคัญที่ยังเป็นช่องว่าง, แนวทางเดิมไม่พอ/ไม่ได้ผล, บริบทเปลี่ยนต้องการวิธีใหม่

### 3.2 Pain Point — กระบวนการ 5 ขั้น (สไลด์5 น.42)
Pain Point (ปัญหาจริง) → Current Situation (สถานการณ์ปัจจุบัน) → Root Cause (สาเหตุแท้จริง) → Research (หาความรู้/นวัตกรรมแก้ปัญหา) → Solution (แนวทางแก้ตรงจุด ยั่งยืน)

### 3.3 Problem Tree — ต้นไม้ปัญหา (สไลด์5 น.43)
4 ระดับ: **ผลกระทบ (Effects) = ใบ/ยอด** (คุณภาพชีวิตลดลง ต้นทุนเพิ่ม ประสิทธิภาพลดลง) → **ปัญหาหลัก (Core Problem) = ลำต้น** → **สาเหตุหลัก (Causes) = ราก** (ปัจจัยบุคคล/กระบวนการ/ระบบ-นโยบาย) → **สาเหตุย่อย (Root Causes)** — หลักการ: **แก้ที่ "ราก" ไม่ใช่แค่ "กิ่งและใบ"**

### 3.4 Theory of Change (สไลด์5 น.44)
Input (งบ บุคลากร เครื่องมือ ความรู้ พันธมิตร) → Activity (อบรม ประชุม ทดลอง นิเทศ ประเมิน) → Output (ผู้เข้าร่วม สื่อ ฐานข้อมูล รายงาน) → Outcome (ความรู้/พฤติกรรม/ประสิทธิภาพ/คุณภาพชีวิตดีขึ้น) → Impact (การเปลี่ยนแปลงเชิงบวกสังคม เศรษฐกิจยั่งยืน ความเสมอภาค สิ่งแวดล้อม)

### 3.5 Impact Pathway (Flow→Benefits — สไลด์5 น.45)
Flow (วิเคราะห์บริบท/ผู้มีส่วนได้ส่วนเสีย) → Knowledge (สังเคราะห์องค์ความรู้) → Innovation (พัฒนา/ทดลองต้นแบบ) → Users (ถ่ายทอด สนับสนุนการใช้) → Benefits (ประโยชน์ยั่งยืน ขยายผล) — คำขวัญ: **"วิจัยไม่ใช่จุดสิ้นสุด แต่คือจุดเริ่มต้นของการเปลี่ยนแปลงที่ยั่งยืน"**

### 3.6 ระดับความรุนแรงของปัญหา 4 ระดับ (สไลด์5 น.65)
ยิ่งรุนแรง ยิ่งสะท้อนความจำเป็นของวิจัย: **01 เฝ้าระวัง (Monitoring)** สัญญาณเริ่มพบ → งานวิจัยเชิงสำรวจติดตาม · **02 ระวังภัย (Warning)** เริ่มกระทบชัด (เช่น ขยะพลาสติกเพิ่ม สารเสพติดในวัยรุ่นเพิ่ม NCDs เพิ่ม) → วิจัยวิเคราะห์ปัจจัยและความสัมพันธ์ · **03 เตือนภัย (Alert)** ขยายวงกว้าง ต้องเร่งด่วน → วิจัยพัฒนาแนวทางแก้ไขและนวัตกรรม · **04 ภาวะวิกฤต (Crisis)** กระทบระดับประเทศ (โรคระบาด ภัยพิบัติใหญ่ อาชญากรรมไซเบอร์ระดับประเทศ) → วิจัยเชิงนโยบายและยุทธศาสตร์ชาติ

### 3.7 คุณลักษณะของปัญหาที่ดี — 5 ระดับ + 5 คำถาม (สไลด์5 น.67)
PROBLEM → SEVERITY → RESEARCH & OUTPUT → OUTCOME → IMPACT · ตอบ 5 คำถาม: **Why? / What if? / Output? / Outcome? / Impact?**
เช็กลิสต์ปัญหาที่ดี: สำคัญชัดเจน · มีข้อมูลหลักฐานรองรับ · มีความรุนแรงเพียงพอ · วิจัยแก้ไขได้ · สร้างผลผลิตใช้ได้จริง · เกิดผลลัพธ์ดีต่อผู้รับประโยชน์ · เกิดผลกระทบยั่งยืน

### 3.8 PAIN POINT & GAIN POINT (สไลด์5 น.69)
Pain Point (ปัญหา/ข้อจำกัด) → **RESEARCH** (ตัวเชื่อม) → Gain Point (คุณค่า/ประโยชน์) → **OUTPUT→OUTCOME→IMPACT**

## 4. เขียนชื่อเรื่อง / บทสรุปข้อเสนอ / ความเป็นมา / วัตถุประสงค์

### 4.1 ตั้งชื่อเรื่อง — VPM-LTa Model (สไลด์5 น.53)
**V**ariable (ตัวแปร) · **P**opulation (ประชากร) · **M**ethodology (วิธีวิจัย) · **L**ocation (ขอบเขต) · **Ta**rget/Aim (เป้าหมาย) — 5 คำถาม: ศึกษาอะไร/ใคร/อย่างไร/ที่ไหน/เพื่ออะไร
ตัวอย่าง: "การพัฒนาสมรรถนะด้าน SROI ของนักวิจัยในสถาบันอุดมศึกษาของรัฐ" → V=สมรรถนะ SROI, P=นักวิจัย, L=สถาบันอุดมศึกษาของรัฐ, Ta=เพื่อพัฒนาสมรรถนะ (M ไม่ปรากฏในชื่อ — Completeness 100%)

### 4.2 โครงสร้าง VPAT / aVPAT (สไลด์5 น.54)
**VPAT** = ตัวแปร(V) + คำเชื่อม + ประชากร(P) + พื้นที่(A) + เวลา(T, มีหรือไม่มีก็ได้) · **aVPAT** = เพิ่มลักษณะการศึกษา (a) นำหน้า เช่น "การเปรียบเทียบ" — ลักษณะที่นิยม: สำรวจ/เปรียบเทียบ/วิเคราะห์/พัฒนา/ประเมิน/ศึกษา
สูตร: **ลักษณะการศึกษา + ตัวแปร + ประชากร + พื้นที่ + เวลา = ชื่อเรื่องวิจัย**

### 4.3 โครงสร้างการเขียนข้อเสนอผ่าน NRIIS — 6 ส่วน (สไลด์5 น.46, 61)
ส่วนที่ 1 ข้อมูลทั่วไป · ส่วนที่ 2 ข้อมูลโครงการ (9 หัวข้อย่อย: บทสรุป, หลักการเหตุผล, วัตถุประสงค์, กรอบการวิจัย, แนวคิด-ทฤษฎี-สมมติฐาน, ระเบียบวิธีวิจัย, ทบทวนวรรณกรรม, เอกสารอ้างอิง, ทรัพย์สินทางปัญญา) · ส่วนที่ 3 แผนงาน (11 หัวข้อย่อย — ดู §6) · ส่วนที่ 4 ผลผลิต-ผลลัพธ์-ผลกระทบ (5 หัวข้อย่อย) · ส่วนที่ 5 เอกสารแนบ · ส่วนที่ 6 ตรวจสอบความถูกต้อง

### 4.4 บทสรุปข้อเสนอโครงการ — องค์ประกอบ (สไลด์5 น.63)
**ไม่เกิน 3,000 คำ ควรมีประมาณ 6 ย่อหน้า**: (1-2) ที่มาพร้อมอ้างอิง · (1-2) ปัญหางานวิจัย+เชื่อมโยงผู้มีส่วนได้ส่วนเสีย · ผลงานที่ผ่านมาของคณะผู้วิจัยที่เกี่ยวข้อง · ในโครงการนี้จะทำอะไร (อ้างแนวคิด/กระบวนการวิจัย) · สรุปว่าทำแล้วได้อะไร (ผลผลิต ผลลัพธ์ ผลกระทบ)

### 4.5 เทคนิคเขียนความเป็นมา/ความสำคัญ — โครงสร้างกรวย 3 ย่อหน้า (สไลด์5 น.64)
"จากภาพกว้างลงสู่ภาพลึก": **(1) ย่อหน้าแรก** บริบท/ระดับปัญหา (ภาพกว้าง — อดีต-ปัจจุบัน-แนวโน้ม, เชื่อมนโยบาย/ยุทธศาสตร์ชาติ) → **(2) ย่อหน้าสอง** เจาะจงประเด็น + ข้อมูลเชิงประจักษ์ (สถิติ แนวโน้ม งานวิจัยเกี่ยวข้อง) → **(3) ย่อหน้าสุดท้าย** สรุป Research Gap → ความจำเป็นวิจัย → Research Question → Objective → Impact

**เคล็ดลับ 5 ข้อ:** เขียนจากภาพกว้างสู่ลึก · ใช้ข้อมูลน่าเชื่อถือ+อ้างอิง · เชื่อมโยงเป็นระบบ · ชี้ "ความจำเป็น" (ทำไมต้องศึกษาตอนนี้) · เน้นคุณค่า/ผลกระทบ

**เทคนิคเรียบเรียง 3 หลัก** (สไลด์5 น.66): ① ตรงประเด็นและครอบคลุม (DO: สรุปสาระสำคัญ / DON'T: ยืดยาวนอกประเด็น) ② ใช้สถิติและอ้างอิง (DO: สังเคราะห์+อ้างอิงทุกครั้ง / DON'T: คัดลอกตารางยาว วางตัวเลขเฉยๆ) ③ ต่อเนื่องลื่นไหล (DO: เชื่อมโยงแต่ละย่อหน้า / DON'T: กระโดดประเด็น)

**โครงสร้างประโยคย่อหน้าสุดท้าย 4 ขั้น** (สไลด์5 น.68): ① เกริ่นสภาพปัจจุบัน ("ปัจจุบัน...", "แม้จะมีการศึกษาและพัฒนา...") ② ระบุ Research Gap ("อย่างไรก็ตาม...", "แต่ปัจจุบันยังพบว่า...") ③ ระบุสิ่งที่ควรจะเป็น ④ ขมวดสู่งานวิจัย ("เพื่อศึกษา/พัฒนา/ประเมิน...") — **สูตร: "ดังนั้น งานวิจัยนี้จึงมุ่งศึกษา... เพื่อ... และนำไปสู่..."**

### 4.6 การเขียนวัตถุประสงค์ — SMART (สไลด์5 น.70-71)
**S**pecific · **M**easurable · **A**chievable · **R**elevant · **T**ime-bound — ห่วงโซ่: Problem → Research Question → Research Objective → Expected Outcome
คำกริยานิยม: เพื่อศึกษา / วิเคราะห์ / เปรียบเทียบ / พัฒนา-สร้าง / ประเมิน / ทดสอบ-ทดลอง
**วิธีเขียน 5 ข้อ** (อ้างอิง: รศ.ดร.นวดล เพชรวัฒนา ม.ศรีนครินทรวิโรฒ, 2568): เรียงลำดับความสำคัญให้เห็นต้นน้ำ-กลางน้ำ-ปลายน้ำ สอดคล้องชื่อเรื่อง · เชื่อมโยงที่มาของปัญหา · ไม่ปะปนกับประโยชน์ที่จะได้รับ · ไม่เขียนเหมือนขั้นตอนทำวิจัย · ต้องตอบได้ทุกข้อในรายงานวิจัย

## 5. กรอบแนวคิด แนวคิด-ทฤษฎี-สมมติฐาน และระเบียบวิธีวิจัย

### 5.1 การสังเคราะห์กรอบแนวคิด — 4 องค์ประกอบ (สไลด์5 น.72-73)
5 ส่วนของ Conceptual Framework: ① ปัญหาวิจัย (Problem/Gap) ② ทฤษฎีและวรรณกรรม (Theory) ③ ตัวแปรที่ศึกษา (X ต้น/M กลาง/Z กำกับ/Y ตาม) ④ ความสัมพันธ์ (เส้นทางอิทธิพล/สมมติฐาน) ⑤ ผลลัพธ์ที่คาดหวัง
**4 ประเภทกรอบแนวคิด:** ① ตรง (X→Y) ② หลายตัวแปรต้น (X1,X2,X3→Y) ③ ตัวแปรส่งผ่าน/Mediator (X→M→Y) ④ ตัวแปรกำกับ/Moderator (Z กำกับ X→Y)
**6 ขั้นพัฒนา:** ศึกษาปัญหา → ทบทวนวรรณกรรม → เลือกทฤษฎี → กำหนดตัวแปร+นิยามปฏิบัติการ → สร้างความสัมพันธ์(สมมติฐาน) → ออกแบบและตรวจสอบ
**Do:** เชื่อมโยงปัญหา/วัตถุประสงค์ · อ้างอิงทฤษฎีน่าเชื่อถือ · ตัวแปรชัดเจน — **Don't:** ตัวแปรมากเกินไป · ไม่มีทฤษฎีรองรับ · วัด/ทดสอบไม่ได้จริง

**อุปนัย vs นิรนัย** (สไลด์5 น.73): Data↔Theory↔Theoretical Framework↔Conceptual Framework — อุปนัย(Inductive)=พัฒนาทฤษฎีจากข้อมูล, นิรนัย(Deductive)=ทดสอบทฤษฎีด้วยข้อมูล

**เปรียบเทียบกรอบแนวคิด 3 ประเภทวิจัย** (สไลด์5 น.74): เชิงปริมาณ(ตัวแปร+สมมติฐาน) · เชิงคุณภาพ(แนวคิด+ความหมาย) · R&D(กระบวนการพัฒนา+ประเมิน)

### 5.2 การเขียนแนวคิด-ทฤษฎี-สมมติฐาน (สไลด์5 น.77)
ลำดับ: **ทฤษฎี→แนวคิด→กรอบแนวคิด→สมมติฐาน→ออกแบบวิจัย**
- ทฤษฎี(Theory) "ทำไมจึงเป็นเช่นนั้น" — ตัวอย่าง: Transformational Leadership, Social Capital Theory, TAM, Stakeholder Theory, RBV
- แนวคิด(Concept) "คืออะไร" — นิยามศัพท์
- กรอบแนวคิด "เกี่ยวข้องกันอย่างไร"
- สมมติฐาน(Hypothesis) "คาดว่าจะเกิดอะไรขึ้น" — ระบุทิศทาง +/− ทดสอบได้ (เช่น H1 ภาวะผู้นำมีอิทธิพลเชิงบวกต่อประสิทธิผลองค์กร)
**เช็กลิสต์:** แนวคิดชัดเจนสอดคล้องปัญหา · ทฤษฎีเหมาะสม · กรอบแนวคิดแสดงความสัมพันธ์ชัด · สมมติฐานสอดคล้องวัตถุประสงค์และวัดได้จริง

### 5.3 การเขียนระเบียบวิธีวิจัย — 7 ขั้น (สไลด์5 น.78)
① รูปแบบวิจัย: Quantitative/Qualitative/Mixed Methods/R&D
② ประชากรและกลุ่มตัวอย่าง: ประชากร, Sampling Technique, ขนาดตัวอย่างและเกณฑ์
③ เครื่องมือวิจัย: แบบสอบถาม/สัมภาษณ์/ประเมิน/สังเกต/เอกสารทุติยภูมิ
④ คุณภาพเครื่องมือ: ความตรงเชิงเนื้อหา (**IOC**), ความสอดคล้อง (**CVI**), ความเชื่อมั่น (Cronbach's Alpha), Try-out/Pilot Test
⑤ เก็บรวบรวมข้อมูล: ขออนุญาต → เก็บตามแผน → ตรวจสอบ → บันทึกเป็นระบบ → จริยธรรมการวิจัย
⑥ วิเคราะห์ข้อมูล: เชิงปริมาณ (Mean/SD/%, t-test/ANOVA, Correlation/Regression, **SEM/Path Analysis**) · เชิงคุณภาพ (Content Analysis, Thematic Analysis, Coding, วิเคราะห์เชิงอุปนัย)
⑦ ผลการวิจัย: นำเสนอเป็นระบบ → อภิปรายตามทฤษฎี → องค์ความรู้ใหม่ → ข้อเสนอเชิงนโยบาย → ข้อจำกัด

**เลือกวิธีวิจัยให้เหมาะกับคำถาม** (สไลด์5 น.79): เชิงปริมาณ=วัดตัวแปร/ทดสอบสมมติฐาน · เชิงคุณภาพ=เข้าใจปรากฏการณ์เชิงลึก · ผสมวิธี=ต้องการทั้งลึกและยืนยันขยายผล · R&D=พัฒนานวัตกรรม+ทดสอบประเมิน — **หลักพิจารณา 5:** สอดคล้องคำถาม/วัตถุประสงค์ · เหมาะบริบท · ทรัพยากร-เวลา-งบ · จริยธรรม-กฎหมาย

**วิธีดำเนินโครงการวิจัย 8 ขั้น** (สไลด์5 น.80): ศึกษาปัญหา → กำหนดวัตถุประสงค์/แผน → ออกแบบวิจัย → เตรียมการ(ขออนุมัติจริยธรรม) → เก็บข้อมูล → วิเคราะห์ → เขียนรายงาน → เผยแพร่และใช้ประโยชน์ — 5 หลักดำเนินการ: มีจริยธรรม · โปร่งใสตรวจสอบได้ · สอดคล้องเชื่อมโยง · เชื่อถือได้ · เกิดประโยชน์

### 5.4 การทบทวนวรรณกรรม — Systematic Review 7 ขั้น (สไลด์5 น.81-82)
① กำหนดขอบเขต/คำถาม ② ค้นหาอย่างเป็นระบบ (Keywords + เกณฑ์คัดเข้า-ออก) ③ คัดกรองและคัดเลือก ④ ประเมินคุณภาพ ⑤ สกัดข้อมูลและจัดหมวดหมู่ ⑥ วิเคราะห์และสังเคราะห์ (Thematic/Conceptual Synthesis, Meta-analysis) ⑦ เขียนและนำเสนอ
**แหล่งค้นคว้า:** สากล=Scopus, Web of Science, PubMed, ERIC, IEEE Xplore · ไทย=ThaiJO, TCI, DRI, TU Digital Collections · อื่น=Google Scholar

**การอ้างอิง** (สไลด์5 น.82): 5 กระบวนการ (ค้นหา→คัดเลือก→ประเมิน→อ้างอิง→รายการอ้างอิง) · รูปแบบ In-text Author-Year (เดี่ยว/2คน &/3คนขึ้นไป et al./หน่วยงาน) · Reference List ตามประเภท (หนังสือ/บทความ/เว็บไซต์) · **Tools:** EndNote, Mendeley, Zotero, RefWorks · **Styles:** APA 7th (สังคมศาสตร์/การศึกษา) · Chicago 17th (ประวัติศาสตร์/มนุษยศาสตร์) · Vancouver (การแพทย์) · ThaiAPA
**ต้องอ้างอิง:** แนวคิด-ทฤษฎีผู้อื่น, ผลงานวิจัย, ข้อมูล-สถิติ-ภาพ, คำกล่าวผู้อื่น, กรอบแนวคิด/โมเดล — **ไม่ต้องอ้างอิง:** ความคิดเห็นตนเอง, ประสบการณ์ส่วนตัว, ความรู้ทั่วไป

### 5.5 ทรัพย์สินทางปัญญา/สิทธิบัตร (สไลด์5 น.83)
นิยามสิทธิบัตร: สิทธิแต่เพียงผู้เดียวที่รัฐให้ผู้ประดิษฐ์ ต้องมีขั้นการประดิษฐ์และประยุกต์ใช้ในอุตสาหกรรมได้ — 5 ประเภท: สิทธิบัตรการประดิษฐ์, อนุสิทธิบัตร, สิทธิบัตรการออกแบบผลิตภัณฑ์, เครื่องหมายการค้า, ลิขสิทธิ์

### 6.5 หลักคิดงบประมาณ — Value for Money (3E) (สไลด์5 น.92)
1. **Economy** (ความประหยัด) — ทรัพยากรในปริมาณ/ราคาเหมาะสม
2. **Efficiency** (ประสิทธิภาพ) — ใช้ทรัพยากรให้ได้ Output มากที่สุด
3. **Effectiveness** (ประสิทธิผล) — บรรลุวัตถุประสงค์เกิด Outcome/Impact

### 6.6 งบประมาณที่กรรมการชอบ — Reasonable/Justifiable/Transparent (สไลด์5 น.93)
- **Reasonable:** สอดคล้องวัตถุประสงค์-ขอบเขต, คำนวณจากกิจกรรมจำเป็นจริง, อ้างอิงราคากลาง/ตลาด
- **Justifiable:** ชี้แจงเหตุผลทุกรายการ, แสดงความเชื่อมโยงกิจกรรม-ผลลัพธ์, มีหลักฐานตรวจสอบได้
- **Transparent:** เปิดเผยข้อมูลครบถ้วน, จัดซื้อจัดจ้างเป็นธรรมตรวจสอบได้

### 6.7 มาตรฐานการวิจัย 5 มิติ + PDCA (สไลด์5 น.94)
① จริยธรรมการวิจัย (Research Ethics) ② คุณภาพการวิจัย ③ ความโปร่งใส ④ ความน่าเชื่อถือ (Validity, Reliability, Credibility, Confirmability, Reproducibility) ⑤ การใช้ประโยชน์ — วงจร: Plan→Do→Check→Act(รับรองมาตรฐาน)→Utilize→Impact

**หน้ากรอกมาตรฐานการวิจัยใน NRIIS** (สไลด์5 น.95): checkbox — จริยธรรมการวิจัย / มาตรฐานการวิจัยในมนุษย์ (ยังไม่รับรอง/รับรองแล้ว) / มาตรฐานความปลอดภัยทางชีวภาพ / มาตรฐานการดำเนินการต่อสัตว์เพื่องานวิทยาศาสตร์ / มาตรฐานความปลอดภัยห้องปฏิบัติการสารเคมี

### 6.8 หน่วยงานร่วมดำเนินการและการร่วมลงทุน (สไลด์5 น.96-97)
ฟอร์ม NRIIS ต้องระบุ: ชื่อหน่วยงาน, ที่อยู่, ปีที่ร่วม, แนวทางร่วมดำเนินการ, **การร่วมลงทุนตัวเงิน (in-cash)**, **in-kind ระบุเป็นตัวเงิน**, in-kind คำอธิบาย (เช่น สนับสนุนข้อมูล บุคลากร สิ่งอำนวยความสะดวก)

### 6.9 TRL — Technology Readiness Level 1-9 (สไลด์5 น.98-99)
4 ระยะ: **ระยะ1 วิจัยเบื้องต้น (TRL1-2)** → **ระยะ2 พิสูจน์แนวคิด (TRL3-4)** → **ระยะ3 พัฒนาต้นแบบ (TRL5-6)** → **ระยะ4 สาธิตและใช้จริง (TRL7-9)**
| TRL | ความหมาย |
|---|---|
| 1 | ศึกษาหลักการพื้นฐานทางวิทยาศาสตร์ |
| 2 | กำหนดแนวคิดและการประยุกต์ใช้เทคโนโลยี |
| 3 | พิสูจน์แนวคิดเชิงวิเคราะห์ (Analytical & experimental proof of concept) |
| 4 | ทดสอบส่วนประกอบในห้องปฏิบัติการ (Lab Scale) |
| 5 | ทดสอบองค์ประกอบหลักในสภาพแวดล้อมจำลอง |
| 6 | ทดสอบระบบต้นแบบในสภาพแวดล้อมใกล้เคียงจริง |
| 7 | สาธิตต้นแบบในสภาพแวดล้อมการทำงานจริง |
| 8 | ระบบสมบูรณ์และผ่านการทดสอบใช้งานได้จริง |
| 9 | ผ่านการพิสูจน์สมบูรณ์และใช้จริงเชิงพาณิชย์ |

**ความสำคัญ:** สื่อสารตรงกันทุกฝ่าย · ประเมินความเสี่ยง · ประกอบตัดสินใจลงทุน · เชื่อมงานวิจัยสู่การใช้ประโยชน์ · มาตรฐานสากล — **หมายเหตุ: TRL ไม่ใช่การวัดคุณค่าเชิงตลาด ควรใช้ร่วมกับ SRL, MRL (Manufacturing Readiness), BRL (Business Readiness)**

### 6.10 SRL — Societal Readiness Level 1-9 (สไลด์5 น.100)
นิยาม: ระดับความพร้อมขององค์ความรู้/เทคโนโลยีทางสังคม เพื่อให้เกิดการยอมรับและใช้ประโยชน์อย่างยั่งยืน — จัดกลุ่ม: **SRL1-3 Research & Co-Creation** (วิเคราะห์ปัญหา, กำหนดแนวทาง, ทดสอบร่วมผู้มีส่วนได้ส่วนเสีย) · **SRL4-6 Pilot & Validation** (ทดสอบพื้นที่นำร่อง, ยืนยันโดยผู้ใช้, ประยุกต์ในสิ่งแวดล้อมจริง) · **SRL7-9 Adoption & Impact** (ปรับปรุงทดสอบซ้ำ, แผนสมบูรณ์ได้รับการยอมรับ, นำไปใช้กับสิ่งแวดล้อมอื่นได้)

### 6.11 TRL vs SRL — ต้องพิจารณาคู่กัน (สไลด์5 น.101)
TRL="เทคโนโลยีพร้อมหรือยัง?" · SRL="สังคมพร้อมหรือยัง?" — ตัวอย่างความไม่สมดุล: AI วิเคราะห์พฤติกรรม TRL9 แต่สังคมไม่ยอมรับ SRL4 (✗) / รถยนต์ไฟฟ้าในไทย ความต้องการสูง SRL8 แต่เทคโนโลยียังไม่สมบูรณ์ TRL4 (✗) — **สำเร็จเมื่อ TRL สูง + SRL สูง**
ตัวอย่างพัฒนาคู่ขนาน "ระบบเตือนภัยผู้สูงอายุผ่านแอป": ต้นแบบ TRL4/SRL2 → ทดลอง 20 คน TRL6/SRL4 → ทดลอง 3 จังหวัด TRL7/SRL6 → อปท.ใช้จริง TRL8/SRL8 → ขยายผลประเทศ TRL9/SRL9

### 6.12 แนวทางขับเคลื่อนผลงานวิจัยสู่ผลลัพธ์-ผลกระทบ (สไลด์5 น.102-103 — อ้างอิง: รศ.ดร.นวดล เพชรวัฒนา ม.ศรีนครินทรวิโรฒ, 2568)
6 ข้อ: (1) เป้าหมายการขับเคลื่อน — ถ่ายทอดเทคโนโลยี/ขยายผลเชิงพาณิชย์-นโยบาย/ยกระดับคุณภาพชีวิต (2) แนวทาง — ถ่ายทอดองค์ความรู้ (อบรม/คู่มือ/แพลตฟอร์ม) หรือเชิงพาณิชย์ (business matching/prototype/สิทธิบัตร) หรือเชิงนโยบาย (เวทีสาธารณะ/ความร่วมมือภาครัฐ) (3) ติดตามและประเมินผล (4) กลุ่มเป้าหมายและผู้มีส่วนได้ส่วนเสีย (5) ผลลัพธ์ที่คาดว่าจะเกิด (6) แผนระยะสั้น-กลาง-ยาว

### 6.13 ความเสี่ยงของโครงการ — 4 ด้าน + มาตรการ (สไลด์5 น.106)
① ด้านการดำเนินงาน (เก็บข้อมูลล่าช้า, กลุ่มตัวอย่างไม่ครบ) ② ด้านงบประมาณ (ไม่เพียงพอ, เบิกจ่ายล่าช้า) ③ ด้านวิชาการ (เครื่องมือไม่มีคุณภาพ, ข้อมูลไม่น่าเชื่อถือ) ④ ด้านจริยธรรมและผลกระทบสังคม (เปิดเผยข้อมูลส่วนบุคคล)
กระบวนการบริหารความเสี่ยง 5 ขั้น: ระบุ→ประเมิน→จัดการ→ติดตามทบทวน→ลดความเสี่ยง — ตัวอย่างมาตรการ: กลุ่มตัวอย่างไม่ครบ(สูง)→สำรองเพิ่ม 20% · ข้อมูลสูญหาย(สูง)→สำรอง Cloud+External HDD

|---|---|---|---|---|
| **Output (ผลผลิต)** | สิ่งที่ได้จากการดำเนินกิจกรรม/โครงการตามแผน — ยังไม่ใช่การเปลี่ยนแปลง | เราทำอะไรเสร็จ/ได้อะไร? | สั้น (ระหว่าง–สิ้นสุดโครงการ) | คู่มือการจัดการขยะชุมชน 1 ชุด, อบรมผู้นำชุมชน 100 คน |
| **Outcome (ผลลัพธ์)** | การเปลี่ยนแปลงกับกลุ่มเป้าหมายจากการใช้ผลงาน | ทำให้เกิดการเปลี่ยนแปลงอะไร? | กลาง (หลังใช้ 6 เดือน–3 ปี) | ผู้นำชุมชนนำความรู้ไปใช้จริง ปริมาณขยะลดลง 30% |
| **Impact (ผลกระทบ)** | การเปลี่ยนแปลงระยะยาวระดับสังคม-เศรษฐกิจ-สิ่งแวดล้อม | ส่งผลระยะยาวอย่างไร? | ยาว (>3 ปี) | ชุมชนสะอาดขึ้น ค่ารักษาพยาบาลลดลง เศรษฐกิจชุมชนเติบโตยั่งยืน |

**KR (Key Result) ≠ Output** (สไลด์5 น.60): KR = ผลลัพธ์สำคัญที่วัดได้ชัดเจน สะท้อนการเปลี่ยนแปลงจริง สอดคล้องยุทธศาสตร์ — Output = สิ่งที่ "ทำได้/ส่งมอบได้" ยังไม่สะท้อนการเปลี่ยนแปลง

### 7.2 ประโยชน์ที่คาดว่าจะได้รับ — 5 ด้าน (สไลด์5 น.109)
Knowledge Impact (องค์ความรู้ใหม่) · Practical Impact (ต่อหน่วยงาน/ผู้ปฏิบัติ) · Policy Impact (เชิงนโยบาย) · Social Impact (สังคม/ชุมชน) · Economic Impact (เศรษฐกิจ)

### 7.3 ผลผลิต (Outputs) — 6 หมวด + 10 ประเภททางการ วช. (สไลด์5 น.110-111)
6 หมวด: องค์ความรู้ใหม่ / เครื่องมือ-นวัตกรรม / ข้อมูล-ฐานข้อมูล / นโยบาย-ข้อเสนอแนะ / การพัฒนากำลังคน / การสื่อสารและเผยแพร่
**10 ประเภทผลผลิตทางการ (แบบฟอร์ม วช.):** (1) กำลังคน/หน่วยงานที่พัฒนาทักษะ (2) ต้นฉบับบทความวิจัย (Manuscript) (3) หนังสือ (Book Chapter/Whole Book/Monograph) (4) ต้นแบบผลิตภัณฑ์/เทคโนโลยี/นวัตกรรมสังคม (5) ทรัพย์สินทางปัญญา (6) เครื่องมือและโครงสร้างพื้นฐาน (7) ฐานข้อมูล ระบบ กลไก (8) เครือข่าย (9) การลงทุนวิจัยและนวัตกรรม (In Cash/In Kind) (10) ข้อเสนอแนะเชิงนโยบายและมาตรการ — **กรอกเฉพาะผลผลิตที่ทำได้จริง**

### 7.4 ผลลัพธ์ (Outcomes) — 16 ประเภท 4 กลุ่ม ประยุกต์จาก Researchfish (สไลด์5 น.112-113)
**1. ACADEMIC:** (1) ผลงานตีพิมพ์ (2) การอ้างอิง Citations (Scopus/Web of Science) (3) เครื่องมือ/ระเบียบวิธีวิจัยใหม่ (4) ฐานข้อมูล/แบบจำลองใหม่
**2. TECHNOLOGY & INNOVATION:** (5) ทรัพย์สินทางปัญญา (9 ประเภท) (6) การถ่ายทอดเทคโนโลยี (7) ผลิตภัณฑ์/กระบวนการ/บริการใหม่ (8) การรับรองมาตรฐานใหม่
**3. UTILIZATION:** (9) การจัดการความรู้และแพลตฟอร์มถ่ายทอด (10) การใช้ประโยชน์เชิงนโยบาย (11) กิจกรรมสร้างการมีส่วนร่วม (12) ทุนวิจัยต่อยอด
**4. NETWORK & CAPACITY:** (13) ความร่วมมือ/หุ้นส่วน (14) ความก้าวหน้าในวิชาชีพบุคลากร ววน. (15) รางวัลและการยอมรับ (16) การใช้ประโยชน์เครื่องมือ/ห้องวิจัย/โครงสร้างพื้นฐาน
**หมายเหตุ:** ต้องมีหลักฐานอ้างอิงตรวจสอบได้ เกิดการเปลี่ยนแปลงอย่างมีนัยสำคัญ

### 7.5 ผลกระทบ (Impacts) — 5 ด้าน + 5 ระดับ (สไลด์5 น.114-115)
5 ด้าน: สังคม / เศรษฐกิจ / นโยบายและประเทศ / สิ่งแวดล้อม / องค์ความรู้และนวัตกรรม
5 ระดับ: บุคคล → องค์กร-ชุมชน → ประเทศ → ภูมิภาค-โลก → ความยั่งยืน
**นิยามทางการ:** การเปลี่ยนแปลงที่เกิดจากผลลัพธ์ในวงกว้าง ทั้งวิชาการ นโยบาย เศรษฐกิจ สังคม สิ่งแวดล้อม — พิจารณาทั้งเชิงบวก/ลบ ทางตรง/อ้อม ตั้งใจ/ไม่ตั้งใจ ผ่าน Engagement Activities และมี **Impact Pathway**

### 7.6 เขียนอย่างไรให้ตรง KR — หลักคิด 5 ขั้น (สไลด์5 น.59)
① เข้าใจ KR (ต้องการผลลัพธ์อะไร) ② เชื่อมโยงงานวิจัย ③ กำหนดผลลัพธ์ที่วัดได้ ④ แสดงผลกระทบ ⑤ สื่อสารชัดเจน
เช็กลิสต์ก่อนส่ง: ตอบ KR โดยตรงไหม / ผลลัพธ์วัดได้จริงไหม / มีผู้ใช้ประโยชน์ชัดไหม / ผลกระทบยั่งยืนไหม / สื่อสารน่าเชื่อถือไหม

### 7.7 ตัวอย่างจริง — โจทย์ วช. P12 KR1-KR4 พร้อมค่าเป้าหมาย (สไลด์5 น.61-62, 120)
- KR1 P12 จำนวนประเด็นปัญหาสังคมคุณธรรม/คอร์รัปชัน/ธรรมาภิบาลที่ถูกแก้ไข = **20 ประเด็น ปี 2566-2570**
- KR2 P12 จำนวนนวัตกรรม/เทคโนโลยี/ดิจิทัลแพลตฟอร์มที่ใช้ป้องกันทุจริต = **10 นวัตกรรม/เทคโนโลยี**
- KR3 P12 จำนวนองค์กรต้นแบบด้านธรรมาภิบาล = **15 องค์กร**
- KR4 P12 จำนวนนโยบาย/มาตรการ/กฎหมาย/ระเบียบที่ปรับปรุงจากงานวิจัย = **3 นโยบาย/มาตรการ**

## 8. เดินระบบ NRIIS จริง — ภาพหน้าจอทีละขั้น

> ที่มา: ภาพหน้าจอ NRIIS จริงของวิทยากร (ข้อเสนอโครงการเลขที่ 4875473 "การพัฒนากลไกและกระบวนการขับเคลื่อนองค์กรปกครองส่วนท้องถิ่นขนาดใหญ่ต้นแบบเพื่อเสริมสร้างธรรมาภิบาลและป้องกันการทุจริตคอร์รัปชันเชิงรุก" — โครงการสาธิตของวิทยากรเอง แสดงเป็นตัวอย่างประกอบการสอน)

- **NRIIS** = National Research and Innovation Information System — "One Platform for Thailand Research Ecosystem" (สไลด์5 น.47) · URL: `www.nriis.go.th` · Dashboard (ณ ขณะบันทึก): Researchers 128,542 · Projects 24,357 · Funding 18,762 ล้านบาท
- **วงจรบริหารทุน 7 ขั้น:** เตรียมข้อเสนอ → ยื่นข้อเสนอ → พิจารณาอนุมัติ → ทำสัญญา/เบิกจ่าย → ติดตามความก้าวหน้า → ผลผลิต → ผลลัพธ์/ผลกระทบ
- **หน้าแรก** (น.48): ลงทะเบียน / เข้าสู่ระบบ (รองรับ ThaID) / ตรวจสอบบัญชี / ติดตามสถานะโครงการ
- **หน้าทุนที่เปิดรับ** (น.50-51): `nriis.go.th/Researcher_ResearchFund.aspx` — ทุนของหน่วยงานเปิดในระบบ **ORIIS**
- **ส่วนที่ 1 ข้อมูลทั่วไป:** ความสอดคล้อง Objective+KR ของแผนงาน, ชื่อโครงการ TH/EN, ประเภท/ลักษณะ/ระยะเวลา, คำสำคัญ, สาขา ISCED+OECD, คณะผู้วิจัย (ยอมรับผ่านอีเมล/ระบบ, สัดส่วนรวมต้อง = 100)
- **ส่วนที่ 2 ข้อมูลโครงการ:** ช่องบทสรุป+หลักการเหตุผล จำกัด **ไม่เกิน 3,000 คำ** ต่อช่อง องค์ประกอบใต้ช่อง: ปัญหาวิจัย / ข้อมูลสนับสนุน / ช่องว่างองค์ความรู้ / แนวทางแก้ไข (น.62)
- **ส่วนที่ 3 แผนงาน:** พื้นที่ทำวิจัย (ประเภท/ภาค/จังหวัด/สถานที่) · พื้นที่ได้รับประโยชน์ (แยกต่างหาก) · งบประมาณ (แตกตัวคูณละเอียด) · มาตรฐานการวิจัย (checkbox 5 ประเภท) · TRL/SRL ปัจจุบัน+เมื่อเสร็จสิ้น · ความเชื่อมโยงผู้เชี่ยวชาญ/ผู้มีส่วนได้ส่วนเสีย · ความเสี่ยงโครงการ
- **ส่วนที่ 4 ผลผลิต-ผลลัพธ์-ผลกระทบ:** ประโยชน์หลัก (radio: วิชาการ/สังคม/นโยบาย/เศรษฐกิจ) · ตาราง Users (กลุ่มผู้ใช้ประโยชน์ ววน.)
- **ส่วนที่ 5 เอกสารแนบ:** ขนาดไฟล์สูงสุด **500 MB** (บางหน้าจอระบุ 50 MB — ไม่สอดคล้องกัน ให้ตรวจสอบหน้าจอจริงขณะยื่น) — ประเภทเอกสาร: ใบรับรองความปลอดภัยชีวภาพ / วุฒิการศึกษา / ใบเสนอราคาครุภัณฑ์ / transcript / ผลงานตีพิมพ์ / ผลสอบภาษาอังกฤษ / แผนดำเนินงาน / มาตรฐานการวิจัย / รักษาความลับ / รายงานความก้าวหน้า
- **ส่วนที่ 6 ตรวจสอบความถูกต้อง:** สรุปทุกช่องพร้อมสถานะ "ถูกต้อง/ไม่ถูกต้อง" — ต้องครบทุกช่อง (บทสรุป, หลักการเหตุผล, วัตถุประสงค์, กรอบการวิจัย, แนวคิด-ทฤษฎี-สมมติฐาน, ระเบียบวิธีวิจัย, ทบทวนวรรณกรรม, เอกสารอ้างอิง) จึงส่งได้

### 8.1 การรายงานผลลัพธ์ต่อเนื่อง 5 ปี (หลังได้รับทุน — สไลด์5 น.125-139)
เมนู: `app.nriis.go.th/researcher/outcomeimpact` → "รายงานผลลัพธ์ต่อเนื่อง 5 ปี" (ต่อเนื่องจากภาระผูกพัน 6 ปีนับจากเซ็นสัญญา — สอดคล้อง `09-training-logistics.md`) → เลือกโครงการ → Pop-up ถาม "มีการใช้ประโยชน์หรือไม่" → เลือกด้าน 4 ด้าน (นโยบาย/เศรษฐกิจ/สังคม/วิชาการ) → เลือกด้านย่อยจาก **5 ด้านผลลัพธ์**: (1) การสร้างองค์ความรู้ใหม่ (2) การต่อยอดสู่ผลิตภัณฑ์/บริการใหม่ (3) การผลักดันนโยบาย (4) การส่งเสริมให้ได้รับการสนับสนุน (5) การพัฒนาบุคลากรและโครงสร้างพื้นฐาน → กรอกฟอร์มย่อย 3 ขั้น (ข้อมูลผลลัพธ์ → แหล่งที่มา/URL/DOI → โครงการที่เกี่ยวข้อง)
**ชุดคำถามผลลัพธ์ประยุกต์จาก Researchfish outcomes question set** — มีคู่มือดาวน์โหลดในระบบ (PDF คู่มือกรอกข้อมูล + XLS ประเภทผลลัพธ์และคำจำกัดความ)

(1) Topic ไม่ตรงโจทย์/แผนงาน (2) ไม่มีผู้ใช้/ผู้รับประโยชน์ชัดเจน (3) ไม่มี Impact (4) วิธีวิจัยไม่ตอบโจทย์ (5) งบไม่สัมพันธ์กิจกรรม (6) วัตถุประสงค์ไม่ชัด/วัดผลไม่ได้ (7) ทบทวนวรรณกรรมอ่อน (8) แผนงานไม่ชัดเจน (9) ทีมวิจัยไม่เหมาะสม (10) การนำไปใช้ประโยชน์ไม่ชัด
