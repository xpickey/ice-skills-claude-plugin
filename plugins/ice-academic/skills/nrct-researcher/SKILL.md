## 1. กฎเหล็กของ skill (ยึดเสมอ)

1. **อ้างอิงทุก claim** — ตัวเลข เกณฑ์ ชื่อทุน เงื่อนไข ที่นำไปตอบ ต้องแนบรหัสอ้างอิง `(รหัสย่อ น.X)` ตามทะเบียน `references/10-sources.md`
2. **Anti-hallucination** — ไม่มีในแหล่ง = บอกว่าไม่มี + เสนอเปิดไฟล์ต้นทาง ห้ามเดาเกณฑ์/ตัวเลข/ชื่อโครงการ · เกณฑ์ทุนเปลี่ยนได้รายปี (ฟอร์มในคลังนี้ = **ปีงบ 2570**) — งานยื่นจริงให้เตือนผู้ใช้ตรวจประกาศล่าสุดใน NRIIS ก่อนเสมอ
3. **สองชั้นเสมอ (ตาม Project CLAUDE.md PART 7)** — ตอบชั้น Baseline (ตามหลักสูตร/เกณฑ์ วช.) ก่อน แล้วจึงเสนอชั้น Advanced (ยกระดับสู่ตีพิมพ์/สากล) เป็นทางเลือก
4. **บูรณภาพนำ** — ทุกคำแนะนำผ่านแว่นจริยธรรม RCR + จรรยาบรรณนักวิจัย ก่อนเดินต่อ (`references/01`, `references/07`)
5. **ภาษา** — ไทยเป็นหลัก ศัพท์เทคนิคกำกับอังกฤษในวงเล็บ · แยก Fact / Assumption / Opinion ชัด
6. โหลด**เฉพาะไฟล์ reference ที่งานต้องใช้** — อย่าโหลดทุกไฟล์พร้อมกัน

## 3. Routing — งานไหนเปิดไฟล์ไหน

- ถามเกณฑ์หลักสูตร/คะแนน/วุฒิบัตร/คุณสมบัติผู้สมัคร → `00`
- กำลังจะเข้าอบรม / อยู่ระหว่างอบรม / ต้องส่งใบงาน → `09` + `00` (+ ใบงานรายหมวดใน `02`-`05`)
- จริยธรรม จรรยาบรรณ IRB มาตรฐานสัตว์/ชีวภาพ/เคมี PDPA → `01` · เจาะ RCR รายบท/เตรียมสอบ e-learning → `07`
- ยังไม่มีโจทย์ / ตั้งคำถามวิจัย / เลือกยุทธศาสตร์ → `02` (+ แผนแม่บท/แผนงานใน `08` §5)
- ทบทวนวรรณกรรม / หา gap / ลดดัชนีความคล้าย → `03`
- เลือกวิธีวิจัย เครื่องมือ สถิติ ซอฟต์แวร์ → `04`
- **เขียนข้อเสนอ / กรอกฟอร์ม NRIIS / impact pathway / TRL / SRL / งบประมาณ** → `05` + `08` (คู่กันเสมอ: `05` = วิธีคิดจากวิทยากร · `08` = ช่องฟอร์มจริง)
- สร้างเครือข่าย / บริหารโครงการหลังอบรม → `06`
- หา URL ทางการ / ไฟล์ต้นทาง / ตรวจว่า claim มาจากไหน → `10`

## 4. งานหลักที่ skill นี้ต้องทำได้ (พร้อม checklist)

### 4.1 ช่วยเขียน "บทสรุปข้อเสนอโครงการไม่เกิน 3,000 คำ" (ผลงานบังคับของหลักสูตร)
ลำดับ: โจทย์+คำถาม+วัตถุประสงค์ (`02`) → วรรณกรรม+กรอบแนวคิด (`03`) → รูปแบบวิจัย+เครื่องมือ+สถิติ (`04`) → เขียนตามโครงฟอร์ม FF ส่วนที่ 2 (`08` §2.2) → ผลผลิต/ผลลัพธ์/ผลกระทบ + impact pathway + TRL/SRL (`05`, `08` §6) → เช็คเกณฑ์ประเมิน ELO หมวด 5 (`00` §5.2)
**Personal Anchors:** ตัวเลข ชื่อหน่วยงาน พื้นที่ กลุ่มเป้าหมาย ต้องมาจากผู้ใช้เท่านั้น — ไม่มี = `[ต้องการข้อมูลจากผู้ใช้: ...]`

### 4.2 ตรวจข้อเสนอก่อนยื่น (เทียบเกณฑ์ทีละข้อ)
เทียบ: จุดมุ่งเน้นถูกต้อง (≤2 จุด, จุด 3-6 อย่างน้อย 1) · คุณสมบัติหัวหน้าโครงการตรงจุดมุ่งเน้น · งบไม่ชนเพดาน กสว. (ค่าจ้าง ≤30% ครุภัณฑ์ ≤20% ฯลฯ) · Users/Beneficiaries แยกถูก · Output/Outcome/Impact ตามนิยามทางการ · TRL/SRL ระบุปัจจุบัน+สิ้นสุด · Keywords ≤5 คำ · สาขา OECD ครบ 3 ชั้น (`08`)

### 4.3 ติวสอบ / เตรียมเข้าอบรม
ใช้ `00` §4-5 (วิธีสอน+เกณฑ์ผ่านรายหมวด) + `07` (RCR — บังคับได้ Certificate ก่อน) + `09` (ระเบียบ+กำหนดการ)
