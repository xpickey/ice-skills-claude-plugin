# Personality and Soul — Adding Voice, Not Just Removing AI — 🟦 CORE + 3 Branches (🟩/🟧/🟪)

**V06R01** · 2026.06.13 · 🟦 CORE (register-agnostic) · Shared Core + 3 Branches (🟩 Academic / 🟧 Business / 🟪 General เท่าเทียม)

## When to Read This

อ่านเมื่อข้อความ "ผ่าน detector" แล้ว (Tier 1 ลบหมด, SD ≥ 5, Voice Profile ใส่แล้ว) **แต่อ่านแล้วยังรู้สึก "เย็นชา" "ไม่มีตัวตน" "ใครเขียนก็ได้"** — ไม่ว่างานนั้นจะเป็น register ไหน. การ "คืนเสียงมนุษย์" เป็นขั้นตอนสุดท้ายที่ใช้ได้ทุก register ไม่ใช่ของวิชาการอย่างเดียว

**Architecture:** ไฟล์นี้เป็น 🟦 **CORE** — Soul Insight, 3-tier signs, S1-S6 skeleton, Soul Check 7Q, กฎเหล็ก 4 ข้อ ทั้งหมด register-agnostic (ไม่มี academic default). หลัง core มี **3 branch soul calibration เท่าเทียม** ที่ปรับ first-person voice / nuance posture / register ของแต่ละ S-technique ให้เข้ากับ register นั้น:

- 🟦 **CORE** — กลไกคืนเสียง ทำงานได้ทุก register (academic = business = general เท่าเทียม)
- 🟩 **Academic** — first-person = "ผู้วิจัย"; soul-demand สูงสุดที่บท**อภิปรายผล**
- 🟧 **Business** — first-person = "เรา / ทีมที่ปรึกษา"; soul = commitment voice + judgment ที่กล้ารับผิดชอบ
- 🟪 **General** — first-person = "ผม/เรา/ฉัน" conversational; plain-language, dev-community เป็น anchor

ทั้ง 3 branch เท่าเทียม ไม่มีตัวใดเกาะตัวอื่น — เลือก branch ตาม register ของงาน แล้ว calibrate S1-S6 ตาม branch นั้น

## 1. The Core Insight 🟦

> **"Avoiding AI patterns is only half the job. Sterile, voiceless writing is just as obvious as slop. Good writing has a human behind it."**

**Source:** Wikipedia:Signs of AI writing → translated for thesis-ai-det-col

นี่คือ insight ที่พลิกมุมมอง — แทนที่จะแค่ "ลบ AI patterns" ให้คิดเป็น **"คืนเสียงมนุษย์"** ให้กับงานเขียน. หลักนี้ register-agnostic: งานวิชาการ งานธุรกิจ และงานทั่วไป ล้วนโดน flag ได้เท่ากันถ้า "ไม่มีคนอยู่หลังคำ"

ผลที่ตามมา: detector รุ่นใหม่ (Turnitin 2026, GPTZero 2026) ไม่ได้ตรวจแค่ "เป็น AI ไหม" แต่ตรวจ "writing without soul" ด้วย — ข้อความที่ลบ Tier 1 vocabulary หมดแล้วแต่ไม่มีตัวตน ก็ยังโดน flag

## 2. Signs of Soulless Writing — 3-Tier 🟦 (แม้ผ่าน Pass 1+2 แล้ว)

แม้ Tier 1 vocabulary ลบหมด, SD ≥ 5, Voice Profile ใส่แล้ว — ข้อความยังอาจไร้วิญญาณถ้ามีลักษณะเหล่านี้. สามชั้นนี้ register-agnostic — ใช้ตรวจได้ทุก branch

### 2.1 Sentence-level signs
- **ทุกประโยคมีความยาวและโครงสร้างใกล้เคียงกัน** — แม้ SD ≥ 5 แต่จังหวะอ่านยังเรียบราบ
- **ไม่มีจังหวะหยุด** — ไม่มีประโยคสั้นเด็ดขาด
- **ทุกประโยคพูดเหมือนกัน** — เนื้อหาต่าง แต่น้ำเสียงเหมือนกันหมด

### 2.2 Content-level signs
- **ไม่มีความเห็น** — แค่รายงานข้อเท็จจริงอย่างเป็นกลางตลอด
- **ไม่ยอมรับความซับซ้อน / มิติที่ขัดแย้ง** — ไม่มี "X ได้ผลดีแต่..."
- **ไม่มี first-person perspective** ในที่ที่ควรมี
- **ไม่มีอารมณ์ขัน, ไม่มีคม, ไม่มีบุคลิก**
- **อ่านเหมือน Wikipedia article หรือ press release**

### 2.3 Author-level signs
- **อ่านแล้วไม่รู้ว่าใครเขียน** — ใครเขียนก็ได้ ใช้แทนกันได้ทันที
- **ไม่มี trace ของวินิจฉัย** ของผู้เขียน
- **ไม่มี hesitation / acknowledgment of doubt**

## 3. Six Techniques สำหรับเติมเสียงมนุษย์ — S1-S6 🟦 (skeleton)

S1-S6 เป็น skeleton register-agnostic — กลไกเหมือนกันทุก branch. สิ่งที่เปลี่ยนตาม branch คือ **first-person marker** + **ระดับ nuance ที่ register ยอมรับ** (calibrate ใน §3.A/3.B/3.C)

### Technique S1: Have Opinions
**หลักการ:** อย่าแค่รายงานข้อเท็จจริง — ตอบสนองต่อมัน. ไม่ใช่ "รายงานว่าเกิดอะไร" แต่ "ผู้เขียนคิดอย่างไรกับสิ่งที่เกิด"

### Technique S2: Vary Your Rhythm Aggressively
**หลักการ:** ไม่ใช่แค่ SD ≥ 5 — ต้อง "อ่านแล้วรู้สึก" ความหลากหลาย
**สูตรง่าย:** ประโยคสั้นเด็ดขาด (3-5 คำ) → ตามด้วยประโยคยาวที่ค่อย ๆ เดิน → บางครั้งกลับเข้าประโยคกลาง

### Technique S3: Acknowledge Complexity
**หลักการ:** มนุษย์จริงมีความรู้สึกผสม — "X is impressive but also kind of unsettling" ดีกว่า "X is impressive". ยอมรับมิติที่ขัดแย้งกัน แทนการ flatten ให้เรียบ

### Technique S4: Use First-Person When It Fits
**หลักการ:** First person ไม่ใช่เรื่องไม่เป็นมืออาชีพ — มันคือความซื่อสัตย์ ที่บอกว่ามีคนจริงคิดอยู่. **marker ต่างกันตาม branch** (ผู้วิจัย / เรา-ทีมที่ปรึกษา / ผม-เรา) — ดู calibration

### Technique S5: Let Some Mess In
**หลักการ:** Perfect structure feels algorithmic. Tangents, asides, half-formed thoughts feel human. ใส่ข้อสังเกตข้าง ๆ ด้วย em dash, ยอมรับว่าบางจุดยังไม่แน่ใจ, ระบุข้อจำกัดของข้อเสนอเอง

### Technique S6: Be Specific About Feelings/Reactions
**หลักการ:** ไม่ใช่ "this is concerning" — ระบุว่า**ทำไม**รู้สึก concerned. ความเฉพาะเจาะจงคือสิ่งที่ทำให้ผู้อ่านเชื่อว่ามีคนจริงสังเกตเห็น

❌ Generic: "The findings are concerning."
✅ Specific: "There's something unsettling about agents churning away at 3am while nobody's watching the output."

> **CORE rule:** S1-S6 verbatim ห้ามเปลี่ยน. branch calibration ปรับได้แค่ marker + nuance posture — ห้ามแก้กลไก S เอง

## 3.A 🟩 Academic Soul Calibration

**First-person marker:** ใช้ **"ผู้วิจัย"** แทน "I" — แต่ต้องใช้จริง ไม่หลบ. "ผู้วิจัยจึงสนใจศึกษา..." / "ผู้วิจัยขอเสนอแนวทาง..." — มีจุดยืน. ห้ามแค่ "งานวิจัยนี้พบว่า..." อย่างเดียว ซึ่งทำให้ไม่มีคนอยู่หลังคำ

**Soul-demand สูงสุด = บทอภิปรายผล (Discussion).** บทนี้คือที่ที่ soul "ต้อง" ปรากฏมากที่สุด — เพราะเป็นที่ที่ผู้วิจัยตีความ ไม่ใช่แค่รายงาน. บทผลการวิจัย (Results) ตรงไปตรงมาได้ แต่ Discussion ที่ไร้ opinion = สัญญาณ AI ชัดที่สุดในงานวิชาการ

**S1 (Academic):** balance "มี opinion" กับ "academic register"
> "ผลการวิจัยน่าจะตีความได้สองทาง — ทางแรกสะท้อนว่านโยบายสำเร็จ ทางที่สองชี้ว่าระบบเก็บข้อมูลเองมีอคติ ผู้วิจัยเอนเอียงไปทางที่สอง เพราะการสำรวจครั้งก่อนของ Smith (๒๕๖๖) พบรูปแบบเดียวกัน"

**S3 (Academic):**
> "ข้อค้นพบนี้สนับสนุนสมมติฐานที่ ๑ ในขณะเดียวกันก็ตั้งคำถามต่อกรอบทฤษฎี ผู้วิจัยจึงต้องอภิปรายในข้อจำกัดต่อไป"
> "แม้ผลจะมีนัยสำคัญทางสถิติ แต่ขนาดของกลุ่มตัวอย่าง (๒๗๘ คน) เล็กเกินกว่าจะ generalize ได้อย่างมั่นใจ"

**S6 (Academic):**
> "ผลที่น่ากังวลที่สุดไม่ใช่ตัวเลขที่ลดลง แต่เป็นการที่บุคลากรรุ่นใหม่ ๓ ใน ๕ คนระบุว่าไม่เคยเห็นแบบฝึกอบรมมาก่อน ทั้งที่หน่วยงานจัดทุกปี"

**บริบท มจร (Buddhist Public Admin):** งานสาย มจร อ้างงานวิจัย มจร ≥ 60% ในบทที่ ๒, โครงสร้าง 5 บท, อ้างพระไตรปิฎกรูปแบบ "ที.ปา. (ไทย) ๑๑/๒๒๘/๒๕๙". ค่าสถิติตัวอย่าง: AGJ 5.72, x̄ = 4.13, SD = 0.32 — soul ในงานสายนี้แสดงผ่าน "ผู้วิจัย" + การตีความที่ผูกหลักธรรมเข้ากับข้อมูลจริง ไม่ใช่ท่องหลักธรรมลอย ๆ

### Before / After ตัวอย่างเต็ม (Academic 🟩 — อบจ. example)

**Before (Soulless — ผ่าน Pass 2 แล้วแต่ไม่มีวิญญาณ):**
> "การวิจัยนี้ศึกษาประสิทธิผลการบริหารงานขององค์การบริหารส่วนจังหวัดปทุมธานี ผลการวิจัยพบว่าการบริหารโดยใช้หลักไตรสิกขามีประสิทธิภาพในระดับมาก ปัจจัยที่ส่งผลคือการพัฒนาทุนมนุษย์และการสนับสนุนจากผู้บริหาร ผลการวิจัยสามารถนำไปใช้พัฒนานโยบายในอนาคต"

— ผ่าน detector ได้ แต่ไร้ตัวตน เป็นกลางเกินไป ไม่มี nuance

**After (มีวิญญาณ — ใช้ S1-S6):**
> "การวิจัยนี้ศึกษาประสิทธิผลการบริหารงานขององค์การบริหารส่วนจังหวัดปทุมธานี ผลการวิจัยพบว่าการบริหารโดยใช้หลักไตรสิกขาให้ค่าเฉลี่ย ๔.๒๓ — สูงกว่าที่ผู้วิจัยคาดไว้ก่อนเก็บข้อมูล. ปัจจัยที่ส่งผลชัดที่สุดไม่ใช่การพัฒนาทุนมนุษย์ตามทฤษฎีที่ผู้วิจัยตั้งไว้ตอนแรก แต่เป็นการสนับสนุนจากผู้บริหารระดับกลาง ที่ผู้สัมภาษณ์ ๑๒ ใน ๑๘ รูปหรือคน ระบุว่าเป็นจุดเปลี่ยน. ข้อค้นพบนี้สอดคล้องกับ Beer และ Nohria (๒๕๔๓) ในประเด็นการนำการเปลี่ยนแปลง แต่ขัดแย้งบางส่วนกับ Hood (๑๙๙๑) ที่เน้นโครงสร้าง — ผู้วิจัยจึงเสนอว่าในบริบทไทย หลักไตรสิกขาทำงานผ่าน 'ตัวกลาง' มนุษย์มากกว่าผ่านระบบ. ข้อจำกัดของงานวิจัยนี้คือกลุ่มตัวอย่างจำกัดในจังหวัดเดียว — การ generalize ต้องระมัดระวัง."

**สังเกตความเปลี่ยนแปลง:**
- ✅ S1 (Have opinions): "สูงกว่าที่ผู้วิจัยคาดไว้"
- ✅ S2 (Vary rhythm): ประโยคยาว/สั้นสลับ มี em dash หยุดเป็นจังหวะ
- ✅ S3 (Complexity): "สอดคล้องกับ X แต่ขัดแย้งบางส่วนกับ Y"
- ✅ S4 (Use ผู้วิจัย จริง): "ผู้วิจัยตั้งไว้ตอนแรก", "ผู้วิจัยจึงเสนอว่า"
- ✅ S5 (Mess in — ยอมรับข้อจำกัด): "ข้อจำกัดของงานวิจัยนี้คือ..."
- ✅ S6 (Specific): "๔.๒๓ — สูงกว่าที่ผู้วิจัยคาดไว้", "๑๒ ใน ๑๘ รูปหรือคน"

## 3.B 🟧 Business Soul Calibration

**First-person marker:** ใช้ **"เรา" / "ทีมที่ปรึกษา"** (ไม่ใช่ "ผู้วิจัย", ไม่ใช่ "ผม") — เสียงขององค์กรที่รับงาน. "เราเสนอแนวทาง..." / "ทีมที่ปรึกษาเห็นว่า..." / "จากประสบการณ์ของทีม...". เสียง business soul คือ **commitment voice** — ไม่ใช่แค่บอกว่า "ทำได้" แต่ "เรารับผิดชอบผลที่ตามมา"

**Soul ในงานธุรกิจ ≠ opinion ส่วนตัวเล่น ๆ.** มันคือ **judgment ที่กล้ารับผิดชอบ** — ที่ปรึกษาจริงไม่เขียน "อาจจะ...ก็เป็นได้...แล้วแต่ลูกค้า" ทุกประโยค (hedge = สัญญาณ AI/ไร้จุดยืน) แต่ระบุจุดยืน + เหตุผลธุรกิจ + trade-off ที่ยอมรับ

**S1 (Business) — มีจุดยืนเชิงคำแนะนำ:**
> ❌ Soulless: "ทั้งสองทางเลือกมีข้อดีข้อเสีย ขึ้นอยู่กับการตัดสินใจของท่าน"
> ✅ Has soul: "ทั้งสองทางเลือกทำได้จริง แต่เราแนะนำ Phase rollout มากกว่า big-bang — เพราะองค์กรท่านเพิ่งผ่านการควบรวมเมื่อปีก่อน การเปลี่ยนทีเดียวทั้งหมดจะชนกับช่วงที่ทีมงานยังปรับตัวกับโครงสร้างใหม่"

**S3 (Business) — ยอมรับ trade-off แทนการขายฝันด้านเดียว:**
> "โซลูชันนี้ลดเวลาปิดงบจาก ๑๐ วันเหลือ ๓ วันได้จริง — แต่เราตรงไปตรงมาว่าช่วง ๒ เดือนแรกหลัง go-live ทีมบัญชีจะทำงานหนักกว่าเดิม เพราะต้องคู่ขนานระบบเก่า-ใหม่. เราจึงวางแผนกำลังคนสำรองไว้ในข้อเสนอ"

**S6 (Business) — ระบุเฉพาะ ไม่พูดลอย:**
> ❌ "ระบบจะช่วยเพิ่มประสิทธิภาพอย่างมาก"
> ✅ "จุดที่ลูกค้าในอุตสาหกรรมเดียวกันบอกเราว่าเปลี่ยนชีวิตจริง ไม่ใช่รายงานสวย ๆ แต่เป็นการที่ผู้จัดการสาขาเลิกต้องโทรถามยอดทุกเช้า เพราะ dashboard ขึ้นเองตอนตี ๕"

**Business hedge → commitment (จาก 11_business_ai_patterns.md TB7):** "หวังว่าจะมีโอกาส..." → "ขอยืนยันความพร้อม...". คำกลุ่ม commitment voice (มุ่งมั่น / ยืนหยัด / เป็นขั้นเป็นตอน) เหมาะใน company-profile/vision section — แต่ห้ามโปรยทั่ว body (กลายเป็น AI-tell). รายละเอียด business AI-tell + density → SSOT `11_business_ai_patterns.md`

> **branch boundary:** Business soul = judgment + commitment + trade-off honesty. ห้ามยืม "ผู้วิจัย" voice (วิชาการ) หรือ conversational "ผม" (general) มาใช้ — register ชนกัน

## 3.C 🟪 General Soul Calibration

**First-person marker:** conversational — **"ผม / เรา / ฉัน"** ตามธรรมชาติของผู้เขียน. ไม่มีศัพท์เฉพาะ ไม่ต้องเป็นทางการแบบวิชาการหรือธุรกิจ. **plain-language เป็นค่าเริ่มต้น** — soul ในงานทั่วไปคือ "เสียงคนคุยกับคน"

**Anchor = dev-community / blog / สื่อสารทั่วไป** (ไม่ใช่ "ก่อนจะเป็นวิชาการ" — general เป็น register เต็มตัวเท่าเทียม branch อื่น). นี่คือ register ที่อนุญาตให้มี opinion แรง, อารมณ์ขัน, และ mess ได้มากที่สุดในสาม branch

**S1 (General) — opinion ตรง ๆ ได้เต็มที่:**
> ❌ Soulless: "The experiment produced interesting results. Some developers were impressed while others were skeptical."
> ✅ Has soul: "I genuinely don't know how to feel about this one. Half the dev community is losing their minds, half are explaining why it doesn't count. The truth is probably somewhere boring in the middle."

ฉบับไทย conversational:
> "บอกตรง ๆ ว่าผมเองก็ยังไม่รู้จะคิดยังไงกับอันนี้ดี ครึ่งหนึ่งของวงการตื่นเต้นกันสุด ๆ อีกครึ่งก็นั่งอธิบายว่ามันไม่ได้เจ๋งขนาดนั้น ความจริงน่าจะอยู่ตรงกลางแบบน่าเบื่อ ๆ นั่นแหละ"

**S2 (General) — จังหวะคุยจริง มีประโยคสั้นเด็ด:**
> "ระบบนี้ใช้งานยาก. ผู้ใช้บ่นมาตลอดสามเดือนแรก แต่หลังจากเดือนที่สี่ ทีมเริ่มเข้าใจว่าปัญหาไม่ได้อยู่ที่ระบบ แต่อยู่ที่กระบวนการที่ทีมเองสร้างขึ้นก่อนเปิดใช้ — และนั่นคือสิ่งที่ระบบไม่ได้แก้. มันแก้ได้แค่บางส่วน."

**S6 (General):**
> "There's something unsettling about agents churning away at 3am while nobody's watching the output."

> **branch boundary:** General soul = conversational + plain-language + opinion เปิดกว้าง. แต่ "ห้าม fabricate emotion" ยังบังคับ (กฎเหล็ก §7) — opinion ต้องเป็นของผู้เขียนจริง ไม่ใช่ที่ AI เติมให้ดูเหมือนมีชีวิต

## 4. Quick "Soul Check" — 7 คำถามก่อนส่งงาน 🟦

อ่านงานเขียนของท่านแล้วถาม (ใช้ได้ทุก branch):

1. **อ่านแล้วรู้สึกใครเขียนได้ไหม?** ถ้าใครเขียนก็ได้ → ขาดเสียง
2. **มีจังหวะหยุดเด็ดขาดไหม?** ประโยคสั้นเด็ด ๆ ที่บังคับให้ผู้อ่านหยุด
3. **มีจุดที่ผู้เขียนยอมรับว่าไม่แน่ใจไหม?** หรือมั่นใจ ๑๐๐% ทุกประโยค
4. **มีความเห็นที่ขัดแย้งกับมุมมองหลักไหม?** Counterargument ที่ผู้เขียนเอง raise ขึ้นมา
5. **มีรายละเอียดเฉพาะที่ทำให้คน believe ว่าผู้เขียนรู้จริงไหม?**
6. **อ่านออกเสียงได้ลื่นไหม?** หรือต้องหยุดเพราะไวยากรณ์อึดอัด
7. **อ่านจบแล้วจำได้ ๑ ประโยคไหม?** หรือเลื่อนไปข้างล่างทันที

ถ้าตอบ "ไม่" 3+ ข้อ → กลับไปใช้ Technique S1-S6 อีกครั้ง (calibrate ตาม branch ของงาน)

## 5. Integration with Two-Pass Method 🟦

แทรก "Soul Pass" หลัง Two-Pass หากจำเป็น (กลไกเดียวกันทุก branch):

```
[Draft] → Pass 1 (Rhythm) → ตรวจ SD ≥ 5
        ↓
        Pass 2 (Vocabulary + Voice Profile) → ตรวจ Tier 1 ผ่าน
        ↓
        Re-Detection → ผ่านเกณฑ์ AI ไหม?
        ↓ ใช่
        Soul Check (7 คำถามใน Section 4)
        ↓ ไม่ผ่าน
        Soul Pass — เติมเสียงด้วย Technique S1-S6 (calibrate ตาม branch §3.A/B/C)
        ↓
        Final Output
```

> **เมื่อใดต้องทำ Soul Pass:** เมื่อข้อความผ่าน 3-Layer Detection แล้วแต่อ่านยังรู้สึก "ขาดตัวตน" หรือ "เป็นกลางเกินไป" หรือ reviewer/peer feedback ว่า "อ่านเหมือน AI". สอดคล้องกับ skill Mode 4 (Soul step) + Mode 6 (ADD SOUL — soul เป็น goal โดยตรง)

## 6. Map: Soul Pass × Detection / Scan Discipline 🟦

Soul Pass ไม่ลบล้างเกณฑ์ detection — มันเสริมหลังผ่านเกณฑ์แล้ว. เกณฑ์ที่ต้องผ่านก่อน (SSOT ที่อื่น ห้าม override):

- **15-check + Scan Discipline 6 ข้อ + Pass ≥ 13/15 + Check 1b hard-gate** → `01_three_layer_detection.md`
- **3 academic-check (Citation / Reporting-Verb / Acronym)** → 🟩 branch-conditional, `01` + `05`
- **density 5-register (≤2 / ≤4 / ≤3 / ≤2 / ≤5 per 1000)** → `01` / `06` / `11` ตาม branch
- **Class A zero-tolerance counts** (เป็นที่ทราบกันดีว่า = 0 .. ปฏิเสธไม่ได้ว่า = 3) + **Green corpus** (ขับเคลื่อน 272/62, ยกระดับ 133/43, อย่างมีประสิทธิภาพ 195/55, บูรณาการ 432/59, นอกจากนี้ 156/69 ฯลฯ) → SSOT `06_verified_ai_signatures.md` (อย่าทำซ้ำตัวเลขในไฟล์นี้ — อ้าง SSOT)
- **corpus วิชาการ 101 ไฟล์ / 292K คำ + iCE 13 proposals** = ฐาน empirical ของทั้ง detection + business branch
- **7 academic sub-profile** (A1/A2/B1/B2/C1/C2/C3) → `05_thai_academic_patterns.md`; Profile A2 = พระไตรปิฎก + "ผู้วิจัย/รูปหรือคน"

> Soul Pass ทำ **หลัง** ผ่านเกณฑ์เหล่านี้ — ห้ามใช้ soul เป็นข้ออ้างละเมิด density/Class A. ถ้า soul technique ทำให้ density เกิน → ตัด soul element นั้น ไม่ใช่ปล่อย density เกิน

## 7. กฎเหล็กของ Soul Pass — 4 ข้อ 🟦 (verbatim, ทุก branch)

1. **ห้ามเติม fake personality** — อย่าเติมความคิดเห็นที่ผู้เขียนไม่ได้คิดจริง
2. **ห้าม fabricate emotions** — อย่าเขียน "I keep thinking about" ถ้าผู้เขียนไม่ได้รู้สึก
3. **Counterargument ต้องอิงงานวิจัยจริง** — ห้ามสร้าง "บางคนแย้งว่า..." ลอย ๆ
4. **First-person markers ต้องมาจาก facts ของผู้ใช้** — ผู้วิจัย/เรา/ผม ตัดสินใจอย่างไร, ระบุเฉพาะที่ผู้ใช้ระบุ

> กฎเหล็ก 4 ข้อนี้ register-agnostic — ใช้บังคับเท่ากันทั้ง 🟩/🟧/🟪. soul ที่ละเมิดข้อใดข้อหนึ่ง = hallucination ไม่ใช่ soul
