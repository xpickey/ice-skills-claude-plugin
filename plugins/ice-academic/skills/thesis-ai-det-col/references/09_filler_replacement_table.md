# Filler Phrase Replacement Table

## When to Read This

อ่านเมื่อทำ Pass 2 Step 2.4 (Add Specificity) — ใช้ตารางนี้แทนที่ filler phrases ที่ไม่เพิ่มความหมาย

---

## 1. English Filler Phrases — Direct Replacements

| Filler (AI ใช้) | Replacement |
|---|---|
| "In order to achieve this goal" | "To achieve this" |
| "In order to" | "To" |
| "Due to the fact that" | "Because" |
| "Despite the fact that" | "Although" |
| "At this point in time" | "Now" |
| "At the present time" | "Currently" |
| "In the event that" | "If" |
| "For the purpose of" | "For / To" |
| "With regard to" | "About / Regarding" |
| "In the case of" | "For" |
| "It is important to note that" | (delete) |
| "It should be noted that" | (delete) |
| "It is worth mentioning that" | (delete) |
| "The data shows that the system has the ability to process" | "The system can process" |
| "Has the ability to" | "Can" |
| "Is in the process of" | "Is" |
| "Has the capacity to" | "Can" |
| "In a number of cases" | "Sometimes / Often" |
| "A large number of" | "Many" |
| "A small number of" | "Few" |
| "In light of the fact that" | "Given that / Because" |
| "In the not too distant future" | "Soon" |
| "On a daily basis" | "Daily" |
| "On a regular basis" | "Regularly" |

## 3. Excessive Hedging — ลด Hedging ซ้อนกัน

### English

| Stacked Hedge (AI) | Single Hedge |
|---|---|
| "could potentially possibly" | "may" |
| "might possibly perhaps" | "might" |
| "it could be argued that it might" | "we argue that it might" |
| "appears to potentially indicate" | "indicates" |
| "may possibly contribute to" | "may contribute to" |

### Thai

| Stacked Hedge (AI) | Single Hedge |
|---|---|
| "อาจจะมีความเป็นไปได้ว่า" | "อาจ" |

## 4. Generic Positive Conclusions — ตัดทิ้ง / แทนด้วยข้อเท็จจริง

| Generic (AI ใช้) | Replacement Strategy |
|---|---|
| "The future looks bright" | ระบุแผนเฉพาะ + ตัวเลข + วันที่ |
| "Exciting times lie ahead" | ระบุเหตุการณ์เฉพาะ |
| "A major step in the right direction" | ระบุก้าวที่เฉพาะคืออะไร |
| "Continues to thrive" | ระบุตัวเลขที่ thriving |
| "Plays an important role" | ระบุบทบาทเฉพาะ |
| "Will undoubtedly transform" | ตัด "undoubtedly" + ระบุการเปลี่ยนแปลงเฉพาะ |
| "อนาคตที่สดใส" | ระบุแผนหรือตัวเลข |
| "ก้าวสำคัญสู่อนาคต" | ระบุก้าวที่เป็นอย่างไร |
| "เปิดศักราชใหม่ของ" | (ตัดทิ้ง / ระบุการเปลี่ยนแปลงเฉพาะ) |
| "ยังคงเติบโตอย่างต่อเนื่อง" | ระบุอัตราเติบโตเฉพาะ |

## 5. Sycophantic / Servile — ตัดออก

ห้ามใส่ในเอกสารวิชาการ/ทางการ/ราชการ:

**English:**
- "Great question!"
- "You're absolutely right!"
- "That's an excellent point"
- "Thank you for asking"
- "I'd be happy to..."
- "Of course!"
- "Certainly!"

**Thai:**
- "เป็นคำถามที่ดีมาก"
- "ถูกต้องเลยครับ"
- "ขอขอบคุณที่สอบถาม"
- "ยินดีอย่างยิ่งที่จะ..."

---

## 6. Knowledge-Cutoff Disclaimers — ห้ามใส่ในงานเขียน

ตัวข้อความนี้บ่งบอกว่ามาจาก AI Chatbot โดยตรง — **ห้ามมีในเอกสารส่ง**:

**English:**
- "As of my last training update..."
- "While specific details are limited..."
- "Based on available information..."
- "Up to my knowledge cutoff..."
- "Unfortunately, I cannot provide..."

**Thai:**
- "เท่าที่ข้อมูลที่ AI ทราบ..."
- "หากข้อมูลเป็นปัจจุบันถึง..."
- "ตามข้อมูลที่มี..."

**Action:** ตัดทิ้ง 100% — ระบุข้อเท็จจริงที่มีจริง ๆ ถ้าไม่มี ระบุเป็น `[NEEDS USER INPUT: <data needed>]`

---

## 7. Chatbot Communication Artifacts — ตัดออก

**ลบทุกอย่างที่บ่งบอกว่ามาจาก chatbot conversation:**

- "Here is an overview of..."
- "Below is a list of..."
- "I hope this helps!"
- "Let me know if you'd like me to expand"
- "Would you like me to..."
- "Feel free to ask if you have questions"
- "นี่คือภาพรวมของ..."
- "ด้านล่างเป็นรายการของ..."
- "หวังว่าคำตอบนี้จะเป็นประโยชน์"
- "หากท่านต้องการให้ขยายความ"
- "หากมีคำถามเพิ่มเติม กรุณาแจ้ง"

**Action:** ตัดทิ้ง — เริ่มเนื้อหาเลยโดยไม่มี preamble

---

## 8. Application in Pass 2

ใช้ตารางนี้ใน **Pass 2 Step 2.4 (Add Specificity)** — ทุก filler ที่พบ:

1. ตัดถ้าไม่เพิ่มความหมาย
2. แทนที่ด้วย short form ตามตาราง
3. ถ้าเป็น generic conclusion → แทนด้วยข้อเท็จจริงเฉพาะ (ใช้ user-provided facts)
4. ถ้าเป็น chatbot artifact → ลบทันที

**กฎเหล็ก:** หลังลบ filler ข้อความต้องสั้นลง ๓๐%+ — ถ้าไม่ลด แสดงว่ายัง substitute ไม่ดี
