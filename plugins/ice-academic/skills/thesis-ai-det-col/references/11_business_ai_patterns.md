## 1. 🔴 CONFIRMED AI-TELLS (CADENCE — ยืนยันจาก corpus)

AI-tell แท้ในงานธุรกิจ iCE คือจังหวะ/โครง ไม่ใช่คำ:

| # | AI-cadence pattern | ตัวอย่างจริง | การแก้ |
|---|---|---|---|
| B1 | **Boilerplate opener** | "This document **outlines** the proposed..." (generic) | เปิดด้วย business intent เฉพาะ (ดู Real Opening Library O1-O6) |
| B2 | **Echo-opening-in-summary** | Summary restate "flexibility, transparency, and cost control" คำต่อคำซ้ำ Objective | restate ด้วยถ้อยคำต่าง ไม่ verbatim |
| B3 | **Filler superlative** | "provides a **clear** recommendation" ("clear" ตัดได้ความหมายไม่เสีย) | ตัดคำขยายที่ไม่เพิ่มสาระ |
| B4 | **Passive + hedge cadence** | "The majority of... **can be addressed through**..." | ใช้ active + ระบุ scope จริง |

**TH cadence (TB) — corpus-grounded จาก BCG/Exim/KingPower/TMB:**

| # | Cadence | ตัวอย่าง | แก้ |
|---|---|---|---|
| TB1 | Triadic benefit-stack ลอย (3 ท่อน generic ไร้ตัวเลข) | "ทันสมัยให้คล่องตัว รวมถึงเพิ่มประสิทธิภาพ และรองรับการขยายตัว" | ยุบเหลือ benefit วัดได้ 1 ผูก capability จริง |
| TB2 | Buzzword-list คั่นจุลภาคไร้ประโยคเชื่อม (feature-dump) | "AI Agentic, Touchless AP, AI Reconciliation, Pillar Two" รวด | ผูกแต่ละตัวกับ value / เหลือ 1 ประโยค "so what" |
| TB3 | Superlative-stack 3 ชั้น/ประโยคเดียว | "ระดับโลก + Leader + ติดต่อกันหลายปี" | เหลือ 1-2 ชั้น + ปีจริงแทน "หลายปี" |
| TB4 | RFP-echo boilerplate (paraphrase ข้อกำหนดซ้ำ ≥3 จุด) | "ตามเงื่อนไข RFP" ×3 ในจดหมายสั้น | พูดครั้งเดียว |
| TB5 | List-everything flat (checklist tone ทุก item น้ำหนักเท่ากัน) | จดหมายไล่แนบเอกสาร 6 อย่างจังหวะเท่ากัน | จัดลำดับ เนื้อ→ความพร้อม→แนบ |
| TB6 | Passive-neutral ไร้ voice (ใส่ vendor ไหนก็ได้) | "ขอเรียนยืนยันว่าได้รับทราบและยอมรับ..." | เติม 1 ประโยค value/ความเข้าใจลูกค้า |
| TB7 | Hedge ปิดท้าย default-AI | "หวังว่าจะมีโอกาส..." | commitment: "ขอยืนยันความพร้อม..." |
| TB8 | Parallel-contrast "ไม่ใช่เพียง X หากเป็น Y" ซ้ำ >1 ครั้ง | (ครั้งเดียว+มี fact = iCE reframe จริง; ซ้ำ = AI echo) | ใช้ครั้งเดียว; ซ้ำ=ตัด |

**EN cadence (EB) — corpus-grounded จาก BANPU/KTC/Forth (เสริม ref 06 §4 ที่มีแต่ vocab เดี่ยว):**

| # | Cadence | ตัวอย่าง | แก้ |
|---|---|---|---|
| EB1 | Section-opener meta-frame "The following X verb..." ซ้ำ | "The following table identifies..." ×3 | ตัด frame ขึ้นเนื้อเลย |
| EB2 | "This + verb + abstract-noun" closer ปิดย่อหน้าซ้ำ | "This undermines credibility / demonstrates maturity" | ปิดด้วย field มีโครงสร้าง (owner/risk) |
| EB3 | "not just X but Y" antithesis สำเร็จรูป | "not just a finance system but the digital backbone" | ครั้งเดียว+มี fact; ซ้ำ=ตัด |
| EB4 | Rule-of-three ✓✓✓ feature-as-benefit เท่ากันหมด | "eliminating.../ integration for.../ implementation" checkmark | ประโยคธุรกิจเต็ม ไม่ใช่ ✓-echo |
| EB5 | Tri-stack buzzword/ประโยค | "ensures **zero-risk** migration through **comprehensive** testing" | "validated through SIT/UAT + production rehearsal before cutover" |
| EB6 | Density echo (register-word ซ้ำถี่ในเอกสารสั้น) | "comprehensive" ×4 ในเอกสารสั้น | เก็บ 1 ที่ที่ผูก noun เฉพาะ ที่เหลือตัด |
| EB7 | Passive-advisory hedge frame | "It is worth noting / should be incorporated to strengthen" | ตัด frame ระบุ action ตรง |
| EB8 | Floating adjective ลอย (robust/seamless/end-to-end ไม่ผูก noun) | "comprehensive change management / robust" | ผูก noun เฉพาะ ("end-to-end visibility from infra to user transaction" = OK) หรือตัด |
| EB9 | Gerund-chain ไร้ burstiness | "extraction, cleansing, transformation, and validity" ทั้งย่อหน้าไม่มีประโยคสั้นคั่น | เพิ่ม burstiness (ประโยคสั้นคั่น) ไม่ใช่ตัดคำ |

## 2. 🟠 BUSINESS AI-TELL WATCHLIST (คำที่ "ไม่พบ" ใน corpus — flag ถ้าโผล่)

> corpus iCE **สะอาดจากคำกลุ่มนี้โดยธรรมชาติ**. ถ้า draft AI-assisted เริ่มโผล่ = flag ทันที (เบี่ยงจาก baseline).

| กลุ่ม | คำเฝ้าระวัง |
|---|---|
| **EN buzzword** | synergy · leverage · seamless · robust · empower · holistic · streamline |
| **EN hollow superlative** | world-class · state-of-the-art · cutting-edge · industry-leading · best-in-class · game-changer |
| **EN absolute guarantee** | zero-risk · seamless migration · intelligent (traffic distribution) · 100% (ลอย) |
| **EN metaphor/inflation** | backbone · digital backbone · synergy · narrative · Transformation (กับงาน technical upgrade) · Strategic Business Value (heading) |
| **EN intensifier-as-evidence** | significantly · substantially (โดยไม่มีตัวเลขตาม) |
| **TH buzzword** | ครบวงจร · ยกระดับ · ขับเคลื่อน · บูรณาการ · ไร้รอยต่อ |
| **TH vague value** | "เพิ่มประสิทธิภาพ" (ลอย ไม่มีตัวเลข) · "ตอบทุกความต้องการ" · "ทันสมัย/เหมาะสม/คล่องตัว" (filler-positive) |
| **emoji-as-bullet** | ✓✓✓ / 🔥 / 🌟 (ไม่เหมาะเอกสารธุรกิจ) |

> ⚠ **vendor-claim ลอย** (~40% time savings · #1 partner · ได้รับการยอมรับจากสากล) — flag เป็น hollow ถ้าไม่มีแหล่ง+ปี. corpus iCE จริงผูก claim กับแหล่ง (Gartner Magic Quadrant + ปี) ไม่ทิ้งลอย

⚠ **กฎคู่ (กัน over-correction):** ถ้าคำในกลุ่มนี้ปรากฏใน corpus iCE จริง (เช่น "พลิกโฉม" ใน vision) → ดู Section 3 — ใช้ได้แบบ density-controlled ไม่ใช่ห้ามขาด.

## 3. 🟢 iCE REGISTER WORDS (ใช้ได้ — density-controlled, Class A/B)

### CLASS A — Core register (ใช้ได้เต็มที่, density-controlled)
คำที่ปรากฏซ้ำใน corpus + ผูกบริบทจริง = voice iCE:

| คำ/วลี | หมายเหตุ |
|---|---|
| **ตอบโจทย์** / ตอบโจทย์...ได้ ดังนี้ | ใช้ ×3 ในย่อหน้าเดียว = signature จริง (อยู่ใน watchlist แต่ NOT a tell) |
| **รองรับ** (การจัดการ/การเติบโต/การทำงานร่วมกัน) | คำแกน |
| ความซับซ้อน / ความถูกต้องแม่นยำสูง / ลดความซ้ำซ้อน | Finance value core |
| การลงทุน / ข้อเสนอการลงทุน | investment framing (แทน "ราคา") = house style |
| ข้อเสนอหลัก (Mandatory) / ข้อเสนอเพิ่มเติม (Optional) | offer-typing |
| Single Source of Truth · Leading Practice / Best Practices | concept-as-proof (สะกดแบบ iCE) |
| Standard Feature / Standard Modules / ไม่ต้อง Customize · 3-tier scope | scope language signature |
| comprehensive solution / unified cloud platform | NetSuite-partner register (มี module นับได้รองรับ) |
| flexibility, transparency, and cost control | triad ผูกกลไก post-paid |
| post-paid manday / Manday Rate / actual effort used | service-model core |

### CLASS B — Watchlist-overlap (ใช้ได้เฉพาะมี anchor / เฉพาะ section)

| คำ/วลี | เงื่อนไขใช้ |
|---|---|
| **พลิกโฉม**การทำงาน | ใช้ได้เฉพาะใน **vision statement** (voice เรา) — ไม่ใช่ใน body ทั่วไป |
| operation excellency | lexical fingerprint — เก็บไว้ ไม่ normalize เป็น excellence |
| transparency / on-demand / flexible | ใช้ได้เมื่อ **ผูกกลไกจริง** (post-paid+confirmation) เท่านั้น |
| มุ่งมั่น / ยืนหยัด / เป็นขั้นเป็นตอน | commitment voice — เหมาะ company-profile section |

## 4. DENSITY RULE (กฎคุมความหนาแน่น)

1. **Anchor rule** — ทุก register/value-word ต้องมี **1 ใน 3 anchor** มิฉะนั้น = vague filler:
   - (a) ตัวเลข/spec จริง (4 ตำแหน่งทศนิยม / 20-20-25% / TCO 5 ปี) — แข็งสุด
   - (b) mechanism/กลไก (transparency ↔ post-paid+confirmation; comprehensive ↔ 4 module นับได้)
   - (c) proof จับต้องได้ (28 ปี / first Oracle Partner)
2. **Repeat cap** — register-word แกนเดิม **≤ 3 ครั้ง/ย่อหน้า** และ **ต้องต่างบริบท** (baseline "ตอบโจทย์" ×3). ซ้ำคำเดิม+บริบทเดิมติดกัน = filler.
3. **Section density:**
   - **Investment/Commercial** → value-prose ≈ 0 (**ห้ามใส่ register-buzzword ใน section ราคา**)
   - **Exec Summary / Why-iCE / Benefits** → density สูงสุดได้ แต่ทุก vague-value ต้องมี proof ตามใน body
   - **Company Profile / Values** → values-as-proof แทน metrics-as-proof (ไม่ต้องมีตัวเลขในส่วนแนะนำตัว)
4. **Anti-hallucination lock (H3):** ห้ามแต่ง/เปลี่ยนตัวเลข "28 ปี" และ "first Oracle Application Partner"; ห้ามเติม metric ที่ไม่มีใน TOR/scope จริง.

## 6. ANTI-ลิเก / OVER-ORNATE CATALOG (TH+EN) ⭐ V01R02

> **ลิเก = "หรูเกิน/ขยายเกินสาระ" — คนละแกนกับ AI-tell** (ลิเก = มนุษย์เขียนหรูเกิน · AI-tell = เครื่องเขียนแบน).
> **กฎแก้:** ไม่ใช่ "ตัดคำสวย" เฉย ๆ แต่ **"แลกคำหรูเป็น fact/capability/กลไกที่ตรวจสอบได้ มิฉะนั้นตัด"** (cut→anchor-swap).

**TH ลิเก (L1-L8):**

| # | สัญญาณ | ตัวอย่าง | แก้ (cut→anchor-swap) |
|---|---|---|---|
| L1 | สร้อยคำหรูลอย (คุณศัพท์ไร้ตัวเลข) | "เทคโนโลยีทันสมัย/เหมาะสม/คล่องตัว", "ระดับโลก" | แลกเป็นกลไกจริง: "ลด manual reconciliation ด้วย Account Reconciliation อัตโนมัติ" |
| L2 | บรรณสโวหารโฆษณา/echo claim ไร้แหล่ง | "ได้รับการยอมรับจากสากลในฐานะผู้นำ" | มีแหล่ง=ใส่ชื่อ+ปี; ไม่มี=ตัด แทนด้วยประโยชน์ต่อลูกค้า |
| L3 | redundant 2 ท่อนความหมายเดียว | "คล่องตัว...รวมถึงเพิ่มประสิทธิภาพ" | ยุบเหลือ 1 ผูก driver จริง |
| L4 | vague-hedge เลี่ยงตัวเลข | "ผ่านการใช้งานมานาน", "ติดต่อกันหลายปี" | ใส่ตัวเลขจริง หรือตัด |
| L5 | อุปมา/วลีหรูเชิงวิสัยทัศน์ลอย | "วางรากฐาน Finance Function 5-10 ปี" | ผูกผลรูปธรรม 1 อย่าง ไม่งั้นลดเหลือ "วางรากฐานระยะยาว" |
| L6 | inflation ยกระดับงานเกินจริง | technical upgrade เรียก "Transformation/พลิกโฉม" | เรียกตามจริง "upgrade/migrate"; "พลิกโฉม" เฉพาะ vision (§3 Class B) |
| L7 | feature-name showcase ผิดที่ (Exec Summary) | ไล่ "AI Agentic, Touchless AP" ใน Exec Summary | ย้าย list ลง section เทคนิค; Exec เหลือ 1 ประโยค value |
| L8 | flourish ปิดท้ายอ่อนแรง | "หวังว่าจะมีโอกาสนำเสนอ..." | proactive: "ยินดีนำเสนอรายละเอียดและตอบข้อซักถาม" |

**EN purple prose (P1-P9):**

| # | Signal | ตัวอย่าง | Fix |
|---|---|---|---|
| P1 | Hollow superlative/marketing-filler | "best-in-class ERP", "world-class" | Cut, let fact speak: "covering all 56 legal entities across 7 countries" |
| P2 | Stacked authority/superlative (3 layers) | "global Cloud SaaS that Gartner rates Leader for consecutive years" | Keep 1-2 layers; let Gartner Leader carry it; drop "global" |
| P3 | Empty intensifier as evidence | "significantly strengthen/under-scoped" | Replace with the number |
| P4 | AI/marketing metaphor | "digital backbone", "AI Synergy" | Name the real capability |
| P5 | Filler adjective triad | "modern, intuitive, next-generation UI" | State the fact: "Redwood Theme, Oracle's current design standard" |
| P6 | Abstract self-praise closer | "This demonstrates maturity and builds confidence" | Cut, or make concrete (risk register with named owners) |
| P7 | Inflated noun stack | "digital energy platforms / strategic priorities" | Keep only the real object |
| P8 | Absolute guarantee (over-claim) | "zero-risk migration", "intelligent traffic distribution" | "low-risk, staged migration with rollback" |
| P9 | Inflation heading | "TRANSFORMATION APPROACH / Strategic Business Value" | Concrete outcome heading |

> **⚠ GUARD (กัน over-correction):** คำที่ "ดูหรูแต่จำเป็น" = register/procurement **ห้ามตัด** — Single Point of Contact · sealed · ผลใช้ได้ 90 วัน · Compliance Matrix · Scalability · EPM module list · CEMLI inventory · Active-Active HA · "comprehensive/end-to-end/best practice" **เมื่อผูก noun+requirement** ("end-to-end visibility from infrastructure to user transaction" = OK). ลิเกกระจุกที่ **sell-zone** (Value/Why-us); โซน spec/scope (FSD Purpose, SoW version-from-to, Investment) ต้อง factual ห้ามมี value-prose.
