## 0.1 บทคัดย่อภาษาไทย & อังกฤษ

### Steps:
1. เขียนส่วนต้นตามแบบฟอร์ม (ชื่อเรื่อง, ผู้วิจัย, ปริญญา, คณะกรรมการ, วันสำเร็จ)
2. เขียนส่วนเนื้อหาตามรูปแบบ **OMF**:
   - **O** = Objective → วัตถุประสงค์ (ต้องตรงบทที่ ๑ เท่านั้น)
   - **M** = Methodology → วิธีดำเนินการย่อ
   - **F** = Finding → ผลการวิจัยตามวัตถุประสงค์
3. เชิงปริมาณ: ระบุประชากร กลุ่มตัวอย่าง วิธีสุ่ม ค่าความเชื่อมั่น สถิติ
4. เชิงคุณภาพ: ระบุผู้ให้ข้อมูลสำคัญ ผู้ทรงคุณวุฒิ เครื่องมือ วิธีวิเคราะห์
5. ผลวิจัยแต่ละข้อ = หนึ่งย่อหน้า กระชับ ชัดเจน
6. บทคัดย่ออังกฤษ: อนุโลมตามไทย ใช้ศัพท์เฉพาะทางการวิจัย

### Validation Rules:
- ❌ FAIL: วัตถุประสงค์ไม่ตรงกับบทที่ ๑
- ❌ FAIL: รวมทุกส่วน > ๒ หน้า
- ❌ FAIL: ไม่ระบุรูปแบบการวิจัย

### 💡 Tips:
- เขียนบทคัดย่อหลังจากเขียนบทที่ ๑-๕ เสร็จแล้ว เพื่อให้ข้อมูลครบถ้วนตรงกัน
- ใช้ "การวิจัยนี้มีวัตถุประสงค์ คือ ๑) เพื่อ... ๒) เพื่อ... ๓) เพื่อ..." เป็นประโยคเปิด

## 0.2 กิตติกรรมประกาศ
**ลำดับบังคับ:** ผู้บริหารหลักสูตร → อาจารย์ที่ปรึกษา → คณะกรรมการสอบ → ผู้เชี่ยวชาญตรวจเครื่องมือ → ผู้ให้ข้อมูลสำคัญ/กลุ่มตัวอย่าง → ผู้มีอุปการคุณ

## 0.3 สารบัญ / สารบัญตาราง / สารบัญแผนภาพ
- ❌ FAIL: หัวข้อในสารบัญ ≠ หัวข้อในเนื้อหา
- ❌ FAIL: เลขหน้าสารบัญ ≠ เลขหน้าจริง
- เลขลำดับตาราง/แผนภาพ: ยึด **บทเป็นตัวตั้ง** เช่น ตารางที่ ๔.๑

## 0.4 คำอธิบายสัญลักษณ์และคำย่อ
- เฉพาะคำย่อพระไตรปิฎกที่อ้างจริงในเล่ม
- แยก: ฉบับบาลี / ฉบับไทย

## 1.1 ความเป็นมาและความสำคัญของปัญหา

### Steps & Checklist:
- [ ] **ความเป็นมา**: เรียงจากเรื่องใหญ่ → เล็ก, เชื่อมโยงเป็นเนื้อเดียว, เรียงลำดับกาลเวลา
- [ ] **ความสำคัญของปัญหา**: นำข้อมูลปัญหามาเสนอให้มากพอ, ตรงประเด็นกับเรื่องวิจัย, เชื่อมโยงพื้นที่วิจัย
- [ ] มีเชิงอรรถอ้างอิงที่มาข้อมูลทุกจุด
- [ ] สรุปเชื่อมโยงไปสู่คำถามวิจัย+วัตถุประสงค์

### Validation Rules:
- ❌ FAIL: ความยาวรวม ≠ ประมาณ ๔-๕ หน้า (น้อยเกินไป/มากเกินไป)
- ❌ FAIL: ไม่มีเชิงอรรถอ้างอิง
- ❌ FAIL: ปัญหาไม่เชื่อมโยงกับหัวข้อวิจัย

### 💡 Tips:
- เปิดย่อหน้าแรกด้วยบริบทระดับชาติ/นโยบาย → ลงมาที่องค์กร/พื้นที่วิจัย
- ใช้สถิติ/ข้อมูลเชิงประจักษ์สนับสนุนว่าปัญหามีจริง
- จบด้วยย่อหน้าเชื่อมว่า "ด้วยเหตุนี้ผู้วิจัยจึงสนใจศึกษา..."

## 1.2 คำถามการวิจัย
- [ ] ประโยคคำถาม สั้นกะทัดรัด | จำนวนข้อ = วัตถุประสงค์
- **Validation:** ❌ FAIL: จำนวนข้อ ≠ วัตถุประสงค์ | โดยทั่วไป = **๓ ข้อ**

## 1.3 วัตถุประสงค์ของการวิจัย
- [ ] ประโยคบอกเล่า ขึ้นต้นด้วย **"เพื่อ"** (เพื่อศึกษา... / เพื่อวิเคราะห์... / เพื่อนำเสนอ...)
- **Validation:** ❌ FAIL: ไม่ขึ้นต้นด้วย "เพื่อ" | จำนวน ≠ ๓ ข้อ

### 💡 Technique — รูปแบบวัตถุประสงค์ ๓ ข้อ (สำหรับ ป.เอก มจร):
```
ข้อ ๑: เพื่อศึกษา [ตัวแปรต้น/ตัวแปรตาม] ของ [บริบท]
ข้อ ๒: เพื่อศึกษา [หลักพุทธธรรม] ที่ส่งเสริม [ตัวแปร]
ข้อ ๓: เพื่อนำเสนอ [รูปแบบ/โมเดล/องค์ความรู้]
```

## 1.4 ขอบเขตการวิจัย (๔ ด้าน)
| ด้าน | Checklist |
|---|---|
| **เนื้อหา** | ระบุตามตัวแปร, สอดคล้องกรอบแนวคิด, อ้างอิงเชิงอรรถที่มาตัวแปร |
| **ประชากร/ผู้ข้อมูล** | ปริมาณ: กลุ่มใด/จำนวน+อ้างอิง | คุณภาพ: กี่กลุ่ม/กลุ่มละกี่รูปหรือคน |
| **พื้นที่** | ระบุหน่วยงาน+ที่ตั้ง+เหตุผลที่เลือก |
| **ระยะเวลา** | ระยะเนื้อหา (พ.ศ.) + ระยะทำวิจัย (≥ ๔-๕ เดือน) |

## 1.5 สมมติฐานการวิจัย (ถ้ามี)
- สอดคล้องวัตถุประสงค์ | ทดสอบได้ด้วยสถิติ | ตั้งจากทฤษฎี+ผลวิจัยที่ผ่านมา
- **Validation:** ❌ FAIL: สมมติฐานตั้งขึ้นมาลอย ๆ ไม่มีฐานทฤษฎี

## 1.6 นิยามศัพท์เฉพาะ ⚠️ กฎสำคัญ
- [ ] นำคำหลักจากชื่อเรื่องมานิยาม
- [ ] นิยามทุกตัวแปร → นำไปสู่การสร้างเครื่องมือได้
- ⚠️⚠️ **ABSOLUTE RULE: ผู้วิจัยนิยามเองเท่านั้น ห้ามมีการอ้างอิงใด ๆ ทั้งสิ้น**
- **Validation:** ❌ FAIL: มีเชิงอรรถอ้างอิงในนิยามศัพท์

## 1.7 ประโยชน์ที่ได้รับ
- ขึ้นต้น: "ได้องค์ความรู้..." / "ได้รูปแบบ..." / "ได้นำเสนอ..."
- ๓ ข้อแรก → สอดคล้องวัตถุประสงค์ | ข้อสุดท้าย → นำไปใช้ประโยชน์อย่างไร

## 2.0 เตรียมการ — ยุทธวิธีการค้นวรรณกรรม (Literature Search Strategy)

> *ขั้นเตรียมก่อนลงมือเขียน §2.1-2.4 — ดัดแปลงสาย วช./NRCT. เป็น "ขั้นตอนคิดเพื่อสร้างคำค้น" ไม่ใช่เนื้อหาที่เขียนลงเล่ม; ผลลัพธ์คือชุดคำค้นที่ใช้ทบทวนทฤษฎี (§2.1) และค้นงานวิจัย (§2.4)*

### Steps:
1. **ตีความ PICO/PICo จากหัวข้อ** — P (ประชากร/หน่วยศึกษา) · I (สิ่งที่ศึกษา/ปรากฏการณ์) · C (เปรียบเทียบ — *ถ้าไม่มี ปล่อยว่าง ห้ามแต่งให้มี*) · O (ผลลัพธ์) · Context (บริบทไทย/พุทธ/หน่วยงาน)
2. **แตกแต่ละองค์ประกอบเป็นคำกว้าง ↔ คำแคบ + คำพ้อง/สะกดหลายแบบ ๒ ภาษา** (เช่น "ธรรมาภิบาล/good governance"; "ดิจิทัล/ดิจิตอล")
3. **(ออปชัน) ประกอบ Boolean** — `(คำพ้อง๑ OR คำพ้อง๒) AND (...)` + คำตัดออก (NOT) กัน false positives; ปรับ syntax ตามฐาน TDC (tdc.thailis.or.th) / ThaiJO / Google Scholar
4. นำชุดคำค้นไปใช้จริงใน §2.1 (ทบทวนทฤษฎี) และ §2.4 (งานวิจัยที่เกี่ยวข้อง)

### Validation Rules:
- ❌ FAIL: นำ "คำค้นที่เสนอ" ไปอ้างเป็นเอกสารจริง โดยยังไม่ได้ค้น
- ❌ FAIL: คำค้นขาดมิติ Context (พุทธ/ไทย/หน่วยงาน) ทำให้ค้นไม่ตรงเรื่อง
- ❌ FAIL: แต่งชื่อบทความ/ผู้แต่ง/ปี จากการประกอบคำค้น (ดู Source Integrity)

### 💡 Tips:
- **Bridge:** คำค้นที่กลั่นแล้ว → §2.1 ทบทวนทุกตัวแปร · §2.4 ค้นงานวิจัย โดยคง Technique "นับ ๖๐% มจร" (กรอง มจร ใน TDC) เดิม
- Context พุทธ → เป็น input ของ §2.2 หลักพุทธธรรม ภายหลัง
- **Guard:** คำค้น = *สมมติฐานการสืบค้น* — เอกสารจริงต้องมาจากการค้นของผู้วิจัย · ไม่แน่ใจคำเทียบเคียง = ระบุ "ต้องตรวจโดยผู้วิจัย" · ห้าม hallucinate

## 2.1 แนวคิด ทฤษฎีที่เกี่ยวข้อง

### Steps:
1. ทบทวนเอกสารครอบคลุมทุกตัวแปรในกรอบแนวคิด
2. แยกเขียนเป็นหัวข้อย่อย นำเสนอนักวิชาการอย่างเป็นระบบ
3. ไม่เขียนชื่อนักวิชาการในเนื้อหา → อ้างในเชิงอรรถ
4. จบแต่ละหัวข้อย่อย → สรุปเป็นความเรียง + **สรุปเป็นตาราง**
5. แผนภาพจากคนอื่น → ต้องมีเชิงอรรถ

### Validation Rules:
- ❌ FAIL: หัวข้อย่อยใดอ้างอิง < ๕ แหล่ง
- ❌ FAIL: ตัวแปรในกรอบแนวคิดไม่มีการทบทวนในบทนี้
- ❌ FAIL: ไม่มีตารางสรุปท้ายหัวข้อย่อย

### 💡 Tips:
- จัดหัวข้อย่อยตามลำดับตัวแปรในกรอบแนวคิด (ตัวแปรต้น ก่อน → ตัวแปรตาม)
- ตารางสรุป: คอลัมน์ "ผู้เขียน/ปี" | "แนวคิด/ทฤษฎี" | "องค์ประกอบ/ตัวชี้วัด"

## 2.2 หลักพุทธธรรมที่เกี่ยวข้อง

### Validation Rules:
- ❌ FAIL: หลักธรรมไม่ตรง/สัมพันธ์กับเรื่องที่วิจัย
- ❌ FAIL: ไม่อ้างพระไตรปิฎกฉบับ มจร
- ❌ FAIL: ขยายความหลักธรรมโดยไม่อ้างผู้เขียนที่น่าเชื่อถือ

### 💡 Tips:
- อ้างพระไตรปิฎกก่อน → ขยายความด้วย ป.อ. ปยุตฺโต (พจนานุกรมพุทธศาสตร์ / พุทธธรรม)
- เชื่อมโยง: "หลักธรรมนี้สอดคล้องกับแนวคิด/ทฤษฎี... เพราะ..."

## 2.3 ข้อมูลบริบทเรื่องที่วิจัย
- ระบุ: โครงสร้างองค์กร พันธกิจ แผนงาน กิจกรรม
- **Validation:** ❌ FAIL: เขียนประวัติ/ภูมิศาสตร์แทนข้อมูลที่เกี่ยวข้องกับเรื่องวิจัย

## 2.4 งานวิจัยที่เกี่ยวข้อง

### Steps:
1. แยกเป็นกลุ่มตามตัวแปร
2. เกริ่นนำก่อนนำเสนอ
3. นำเสนอ **เฉพาะผลการวิจัย** (ไม่นำระเบียบวิธีมา)
4. มีเชิงอรรถ | จบแต่ละหัวข้อ → **สรุปตาราง**

### Validation Rules:
- ❌ FAIL: แต่ละหัวข้อย่อย < ๓ เรื่อง
- ❌ FAIL: งานวิจัยไม่ทันสมัย
- ❌ FAIL (ป.เอก): อ้างวิทยานิพนธ์ ป.โท (ต้องอ้าง ป.เอก/วิจัยทั่วไป/บทความวิจัย)
- ⚠️⚠️ ❌ FAIL: อ้างงานวิจัย มจร < ๖๐% ของงานวิจัยที่เกี่ยวข้องทั้งหมด

### 💡 Technique — นับ ๖๐% มจร:
- นับจำนวนงานวิจัยทั้งหมดในหัวข้อ 2.4
- ≥ ๖๐% ต้องเป็นดุษฎีนิพนธ์/วิทยานิพนธ์/บทความวิจัย ของ มจร
- ค้นหาจาก: TDC (tdc.thailis.or.th) กรอง มจร

## 2.4.5 วิเคราะห์ช่องว่างการวิจัย (Research Gap Analysis)

> *เครื่องมือคิดเชิงระบบ (ดัดแปลงสาย วช./NRCT) — หาช่องว่างเชิงวิชาการจากงานวิจัยที่ทบทวนใน §2.4 เพื่อยืนยันเหตุผลของหัวข้อ/RQ และหนุนความนิ่งของกรอบ §2.5. เป็นขั้นตอนวิเคราะห์ ไม่ใช่หัวข้อแยกในเล่ม — ผลเรียบเรียงเข้า §2.4 (สรุปช่องว่าง) + §2.5 (กรอบ) ตามโครง มจร เดิม*

### Steps:
1. **สรุปองค์ความรู้ปัจจุบัน (Current Knowledge)** — จากงานวิจัย §2.4 ระบุ: วงวิชาการรู้อะไรแล้ว · สอดคล้องกัน · ขัดแย้งกัน · หลักฐานยังไม่พอ (ข้อค้นพบที่ขัดแย้ง = แหล่ง gap ที่ดี)
2. **จำแนกช่องว่าง ๗ ชนิด** (ใช้เท่าที่มีหลักฐานจริง ไม่ครบ ๗ ก็ได้): Theoretical (ทฤษฎีอธิบายไม่ครบ) · Conceptual (มโนทัศน์คลุมเครือ) · Methodological (วิธีเดิมมีข้อจำกัด) · Empirical/Contextual (ยังไม่ศึกษาในบริบทไทย/พื้นที่นี้) · Population (กลุ่มนี้ศึกษาน้อย) · Practical/Policy (ขาดข้อเสนอใช้จริง) · Temporal (องค์ความรู้เก่า). แต่ละ gap ระบุ ๕ ช่อง: ศึกษาแล้ว / ยังขาด / หลักฐานหนุน / เหตุผลเป็น gap / ศักยภาพเป็นโจทย์
3. **จัดลำดับความสำคัญ → Top-5** ตาม ๔ เกณฑ์: วิชาการ × นโยบาย-สังคม × ความใหม่ (Novelty) × ความเป็นไปได้
4. **เขียน Novelty Statement** (สำหรับ gap อันดับ ๑) — องค์ความรู้ใหม่ที่คาดว่าจะได้ + ผูกกับ RQ/วัตถุประสงค์ (บทที่ ๑)

### Validation Rules:
- ❌ FAIL: ระบุ gap โดยไม่มีหลักฐานจากวรรณกรรม (§2.4) หนุน
- ❌ FAIL: Novelty Statement ไม่ผูกกับ RQ/กรอบแนวคิด (ลอย)
- ❌ FAIL: นำพุทธบูรณาการใส่เป็น gap แบบผิวเผิน (ต้องผูกหลักธรรมเข้า gap จริง)

### 💡 Tips:
- **Bridge:** ช่องว่างที่สรุป → เขียนเข้า **§2.4** (ความเรียง + ตารางสรุปท้ายหัวข้อ) · Top-1 gap + Novelty → หนุน **§2.5 กรอบแนวคิด** ให้ตัวแปรทุกตัวมีที่มา · RQ/Novelty → ป้อนกลับ **บทที่ ๑** ถ้าต้องปรับหัวข้อ
- ดุษฎีนิพนธ์รับ Top-5/Matrix ได้ (ภาชนะใหญ่) — ต่างจากบทความสั้น
- **Guard:** วิเคราะห์เฉพาะข้อมูล/วรรณกรรมที่มี · ข้อมูลไม่พอ = "ต้องตรวจโดยผู้วิจัย" · แยก ๓ ชั้น: ข้อค้นพบ ≠ การตีความ ≠ ข้อเสนออนาคต · ห้าม hallucinate (คงกฎ มจร ≥๖๐% / ป.เอก only ใน §2.4)

## 2.5 กรอบแนวคิดในการวิจัย

### Steps:
1. เขียนระบุแหล่งที่มาของตัวแปรแต่ละด้าน (สังเคราะห์จากหลักธรรม/ทฤษฎี/เกณฑ์ใคร)
2. เขียนแผนภาพ มีลูกศรแสดงทิศทาง IV → DV
3. ระบุเชิงอรรถอ้างอิงแหล่งที่มาแต่ละตัวแปร

### Validation Rules:
- ❌ FAIL: ตัวแปรในกรอบแนวคิดไม่ปรากฏในบทที่ ๒
- ❌ FAIL: ไม่มีเชิงอรรถที่มาตัวแปร

### 💡 Tips:
- วางลูกศร: ตัวแปรต้น (ซ้าย) → ตัวแปรตาม (ขวา)
- หลักธรรม: วางเป็นตัวแปรต้นร่วม หรือ ตัวแปรกำกับ
- ก่อนแผนภาพ: เขียน "จากการทบทวนแนวคิด ทฤษฎี และหลักพุทธธรรมข้างต้น ผู้วิจัยสังเคราะห์กรอบแนวคิดดังนี้..."

# ═══════════════════════════════════════
# PHASE 3 — บทที่ ๓ วิธีดำเนินการวิจัย
# ═══════════════════════════════════════

## 3.1 รูปแบบการวิจัย
- ระบุชัดเจน: ผสานวิธี / คุณภาพอย่างเดียว
- ผสานวิธี: ระบุวิธีนำ + วิธีสนับสนุน

## 3.2 การวิจัยเชิงคุณภาพ

### 3.2.1 ผู้ให้ข้อมูลสำคัญ (สัมภาษณ์เชิงลึก)

| เกณฑ์ | ผสานวิธี | คุณภาพอย่างเดียว |
|---|---|---|
| จำนวนขั้นต่ำ | **≥ ๑๗ รูป/คน** | **≥ ๒๕ รูป/คน** |

- แบ่งกลุ่ม: ผู้กำหนดนโยบาย / ผู้ปฏิบัติงาน / ผู้รับบริการ / นักวิชาการ / นักวิชาการพุทธศาสนา / กลุ่มอื่น

### 3.2.2 ผู้ทรงคุณวุฒิ (สนทนากลุ่มเฉพาะ)
- จำนวน: **๙-๑๒ รูป/คน**
- ⚠️ **ไม่ซ้ำกับผู้ให้สัมภาษณ์เชิงลึก**
- ⚠️ **อาจารย์ประจำหลักสูตรเข้าร่วม ≥ ๑ รูป/คน ทุกครั้ง**

### 3.2.3 เครื่องมือวิจัยเชิงคุณภาพ
- สอดคล้องวัตถุประสงค์ + กรอบแนวคิด + นิยามศัพท์
- ผ่าน **IOC** → ผู้เชี่ยวชาญ **๕ คน** (≥ ๒ ภายนอก, ≥ ๓ ด้าน)
- ๓ ด้าน: (1) เนื้อหา/ทฤษฎี (2) ระเบียบวิธีวิจัย (3) พุทธศาสนา/การปฏิบัติงาน

### Validation Rules สำหรับ 3.2:
- ❌ FAIL: ผู้ให้ข้อมูล < ๑๗ (ผสาน) / < ๒๕ (คุณภาพ)
- ❌ FAIL: สนทนากลุ่ม < ๙ หรือ > ๑๒
- ❌ FAIL: ผู้ทรงคุณวุฒิซ้ำกับผู้สัมภาษณ์
- ❌ FAIL: ไม่มีอาจารย์ประจำหลักสูตรในสนทนากลุ่ม
- ❌ FAIL: ผู้เชี่ยวชาญ ≠ ๕ คน หรือ ภายนอก < ๒

## 3.3 การวิจัยเชิงปริมาณ

### 3.3.1 ประชากรและกลุ่มตัวอย่าง
- กำหนดประชากรชัดเจน + อ้างอิง
- แสดงวิธีสุ่ม ขนาดกลุ่มตัวอย่าง สูตร/ตาราง
- ทำตารางจำแนกสัดส่วน
- ใช้สูตร → แสดงสูตร | ใช้ตาราง → แนบภาคผนวก

### 3.3.2 เครื่องมือวิจัยเชิงปริมาณ
- คำชี้แจงลักษณะเครื่องมือ แบ่งเป็นตอน ๆ
- ⚠️ ตรวจสอบคุณภาพ **๒ ขั้นตอน**:
  1. **Validity** (IOC/CVI): ผู้เชี่ยวชาญ ๕ คน (≥ ๒ ภายนอก, ≥ ๓ ด้าน)
  2. **Reliability** (Try out): **๓๐ ชุด** กับกลุ่มลักษณะใกล้เคียง

### 3.3.3 การวิเคราะห์ข้อมูล
- แยกชัดเจน:
  - **สถิติพรรณนา** (Descriptive): ความถี่, ร้อยละ, X̄, S.D.
  - **สถิติอนุมาน** (Inferential): t-test, F-test, Correlation, Regression, SEM

### Validation Rules สำหรับ 3.3:
- ❌ FAIL: ไม่แสดงวิธีสุ่ม/สูตรคำนวณ
- ❌ FAIL: ไม่ผ่าน IOC/CVI
- ❌ FAIL: Try out ≠ ๓๐ ชุด
- ❌ FAIL: ไม่ระบุค่าความเชื่อมั่น

### 💡 Technique — การเขียนเครื่องมือวิจัย:
```
แบบสอบถาม แบ่งออกเป็น ๓ ตอน ดังนี้
ตอนที่ ๑ ข้อมูลทั่วไปของผู้ตอบแบบสอบถาม (Checklist)
ตอนที่ ๒ [ตัวแปรต้น] เป็นแบบมาตราส่วนประมาณค่า ๕ ระดับ (Likert Scale)
ตอนที่ ๓ [ตัวแปรตาม] เป็นแบบมาตราส่วนประมาณค่า ๕ ระดับ
```

# ═══════════════════════════════════════
# PHASE 4 — บทที่ ๔ ผลการวิเคราะห์ข้อมูล
# ═══════════════════════════════════════

## 4.1 การนำเสนอข้อมูลเชิงคุณภาพ

### Steps:
1. นำเสนอข้อมูลสอดคล้องวัตถุประสงค์
2. มีประโยคนำอธิบายบริบท
3. นำเสนอคำพูดจากสัมภาษณ์จริง
4. ⚠️ **ปรโตโฆสะ → โยนิโสมนสิการ**: หลังนำเสนอข้อมูลสัมภาษณ์ → ผู้วิจัยต้องสรุปวิเคราะห์/สังเคราะห์ด้วยตนเองทุกประเด็น
5. สรุปเป็น **ตารางแจกแจงความถี่** หรือ **แผนภาพ**

### Validation Rules:
- ⚠️⚠️ ❌ CRITICAL FAIL: **นำข้อความจากหนังสือ/เอกสารมาเป็นผลการวิจัย** → ต้องจากสัมภาษณ์/สนทนากลุ่มที่ปฏิบัติจริงเท่านั้น
- ❌ FAIL: ไม่มีการสรุปวิเคราะห์ด้วยตนเอง (โยนิโสมนสิการ)
- ❌ FAIL: ข้อมูลไม่ตอบวัตถุประสงค์

### 💡 Tips:
- ใช้รูปแบบ: "ผู้ให้ข้อมูลสำคัญลำดับที่ ๑ ให้สัมภาษณ์ว่า '...'" → แล้วผู้วิจัยสรุปตาม
- สรุปตารางความถี่: "จากการสัมภาษณ์ผู้ให้ข้อมูลสำคัญทั้ง ๑๗ รูป/คน สรุปประเด็นดังตาราง..."

## 4.2 การนำเสนอข้อมูลเชิงปริมาณ
- นำเสนอในรูปตาราง + คำอธิบายท้ายตาราง
- สัญลักษณ์สถิติถูกต้องและสอดคล้องกันทั้งเล่ม

## 4.3 องค์ความรู้ที่ได้รับจากการวิจัย
- สร้างแผนภาพองค์ความรู้ (⚠️ **ไม่ใช่เขียนกรอบแนวคิดเข้ามาอีก** — ต้องแสดงผลที่ค้นพบ)
- แสดงผลตามวัตถุประสงค์ทั้ง ๓ ข้อ
- ⚠️ **อธิบาย/ขยายความ ≥ ๓ หน้า**

## 4.4 องค์ความรู้ที่ได้สังเคราะห์
- นำองค์ความรู้ → สังเคราะห์เป็นความรู้เชิงนวัตกรรม
- แผนภาพไม่ซ้ำ 4.3 → เชิงสร้างสรรค์โดยผู้วิจัย
- ⚠️ **ขยายความ ≥ ๒ หน้า**

### Validation Rules:
- ❌ FAIL: องค์ความรู้ < ๓ หน้า
- ❌ FAIL: สังเคราะห์ < ๒ หน้า
- ❌ FAIL: แผนภาพซ้ำกรอบแนวคิด

# PHASE 5 — บทที่ ๕ สรุป อภิปรายผลและข้อเสนอแนะ
# ═══════════════════════════════════════

## 5.1 สรุปผลการวิจัย
- สรุปตามวัตถุประสงค์แต่ละข้อ ครบถ้วน กระชับ
- ⚠️ **สรุปผลรวม — ไม่แยกเชิงปริมาณ/เชิงคุณภาพ**

## 5.2 อภิปรายผลการวิจัย

### Steps:
1. อภิปรายเป็นข้อ เรียงตามวัตถุประสงค์
2. **หลักการ**: ค้นพบอะไร → เหตุใดเป็นเช่นนั้น → สอดคล้อง/ไม่สอดคล้องกับใคร
3. เชื่อมโยงกับงานวิจัยในบทที่ ๒
4. อ้างอิงเชิงอรรถทุกงานวิจัยที่นำมา

### Validation Rules:
- ❌ FAIL: ไม่เชื่อมโยงกับบทที่ ๒
- ❌ FAIL (ป.เอก): นำวิทยานิพนธ์ ป.โท มาอภิปราย
- ❌ FAIL: ไม่ครอบคลุมทุกตัวแปร

### 💡 Tips:
- รูปแบบ: "ผลการวิจัยข้อที่ ๑ พบว่า... สอดคล้องกับผลการวิจัยของ [ชื่อ] ที่ศึกษาเรื่อง... พบว่า... ทั้งนี้เนื่องจาก..."
- ถ้าไม่สอดคล้อง: "ผลการวิจัยนี้ไม่สอดคล้องกับ... เนื่องจากบริบทแตกต่างกัน คือ..."

## 5.3 ข้อเสนอแนะ (๓ ระดับ)

| ระดับ | แหล่งที่มา | ลักษณะ |
|---|---|---|
| **เชิงนโยบาย** | ค่าเฉลี่ยต่ำ/น้อยที่สุด + ผลสัมภาษณ์ | เรื่องใหญ่/กว้าง กำหนดนโยบายได้ |
| **เชิงปฏิบัติการ** | จากข้อเสนอเชิงนโยบาย ลงรายละเอียด | จำนวนข้อมากกว่าเชิงนโยบาย |
| **เพื่อการวิจัยครั้งต่อไป** | เนื้อหาที่ควรวิจัยเพิ่ม + เทคนิควิธีวิจัย | สั้น กะทัดรัด เข้าใจง่าย |

# PHASE 7 — ภาคผนวก + ประวัติผู้วิจัย

### ภาคผนวกบังคับ:
| ภาคผนวก | เนื้อหา |
|---|---|
| ก | เครื่องมือวิจัยเชิงปริมาณ/เชิงคุณภาพ |
| ข | หนังสือขอผู้เชี่ยวชาญตรวจเครื่องมือ |
| ค | ผลวิเคราะห์ IOC / CVI |
| ง | หนังสือขอ Try out |
| จ | ผลวิเคราะห์ Reliability |
| ฉ | หนังสือขอเก็บข้อมูลเชิงปริมาณ |
| ช | หนังสือขอเก็บข้อมูลเชิงคุณภาพ |
| ซ | ประมวลภาพสัมภาษณ์ |
| ฌ | หนังสือขอสนทนากลุ่ม |
| ญ | ประมวลภาพสนทนากลุ่ม |
| ฎ | ตารางคำนวณกลุ่มตัวอย่าง (ถ้าใช้ตาราง) |

# ═══════════════════════════════════════
# REF-A — รูปแบบเชิงอรรถ (Footnote) ครบทุกประเภท
# ═══════════════════════════════════════

### กฎพื้นฐาน:
- ขนาด: **๑๔ พ้อยท์** | ย่อหน้า: **๐.๗ นิ้ว** (๑.๗๕ ซม.)
- เส้นคั่น: ≈ ๒๐ ตัวพิมพ์ | spacing: before 12pt / after 6pt
- **เริ่มเลข ๑ ใหม่ทุกบท** นับต่อเนื่องจนจบบท

### ประเภทและรูปแบบ:

**๑. พระไตรปิฎก:**
`ขุ.ธ. (บาลี) ๒๕/๑๒/๒, ขุ.ธ. (ไทย) ๒๕/๑๒/๒.`
สรุปความ → `ดูรายละเอียดใน วิ.มหา. (บาลี) ๑/๔-๖/๕๕-๕๗.`

**๒. หนังสือทั่วไป:**
`ผู้แต่ง, ชื่อเรื่อง, พิมพ์ครั้งที่, (สถานที่: สำนักพิมพ์, ปี), หน้า.`

**๓. บทความในวารสาร:**
`ผู้เขียน, "ชื่อบทความ", ชื่อวารสาร, ปีที่/ฉบับที่ (เดือน ปี) เลขหน้า.`

**๔. ดุษฎีนิพนธ์/วิทยานิพนธ์:**
`ชื่อผู้วิจัย, "ชื่อเรื่อง", ระดับ สาขาวิชา, (บัณฑิตวิทยาลัย: สถาบัน, ปี), หน้า.`

**๕. สัมภาษณ์:**
`สัมภาษณ์ ผู้ให้ข้อมูลสำคัญลำดับที่ ๑, ๘ มกราคม ๒๕๖๔.`

**๖. สนทนากลุ่ม:**
`สนทนากลุ่มเฉพาะ ผู้เข้าร่วมสนทนากลุ่มเฉพาะลำดับที่ ๑, ๑๒ มกราคม ๒๕๖๔.`

**๗. ออนไลน์:**
`ผู้แต่ง, ชื่อเรื่อง, [ออนไลน์], แหล่งที่มา: URL [วัน เดือน ปี].`

**๘. อ้างในดุษฎีนิพนธ์ผู้อื่น:**
`ผู้แต่งต้นฉบับ, ชื่อเรื่อง, (สถานที่: สำนักพิมพ์, ปี), หน้า, อ้างใน ผู้วิจัย, "ชื่อเรื่อง", ระดับ สาขาวิชา, (บัณฑิตวิทยาลัย: สถาบัน, ปี), หน้า.`

### การอ้างซ้ำ:
| สถานการณ์ | ไทย | English |
|---|---|---|
| ไม่มีคั่น + หน้าเดียวกัน | เรื่องเดียวกัน. | Ibid. |
| ไม่มีคั่น + หน้าต่าง | เรื่องเดียวกัน, หน้า ๒๗. | Ibid., p. 27. |
| ไม่มีคั่น + ข้อความเดิม + หน้าเดิม | อ้างแล้ว. | Loc. cit. |
| ไม่มีคั่น + ข้อความเดิม + หน้าต่าง | อ้างแล้ว, หน้า ๒๗. | Loc. cit., p. 27. |
| **มีเอกสารคั่น** | ลงรายการย่อ (ตัดสถานที่/สำนักพิมพ์/ปี) | Short form |
