# Big Four Consulting Methodology

## Structured Problem Solving

### Pyramid Principle

```
                    ┌─────────────────┐
                    │   KEY MESSAGE   │  ← Answer first
                    │   (So What?)    │
                    └────────┬────────┘
                             │
           ┌─────────────────┼─────────────────┐
           │                 │                 │
    ┌──────▼──────┐   ┌──────▼──────┐   ┌──────▼──────┐
    │  Supporting │   │  Supporting │   │  Supporting │
    │  Argument 1 │   │  Argument 2 │   │  Argument 3 │
    └──────┬──────┘   └──────┬──────┘   └──────┬──────┘
           │                 │                 │
    ┌──────▼──────┐   ┌──────▼──────┐   ┌──────▼──────┐
    │   Evidence  │   │   Evidence  │   │   Evidence  │
    │   & Data    │   │   & Data    │   │   & Data    │
    └─────────────┘   └─────────────┘   └─────────────┘
```

### MECE Principle

**Mutually Exclusive, Collectively Exhaustive**

- **Mutually Exclusive**: No overlap between categories
- **Collectively Exhaustive**: All possibilities covered

Example - Revenue Growth Analysis:
```
Revenue Growth
├── Volume Growth (MECE)
│   ├── New Customers
│   └── Existing Customer Expansion
├── Price Growth (MECE)
│   ├── List Price Increase
│   └── Mix Improvement
└── Scope Growth (MECE)
    ├── New Products
    └── New Markets
```

## Hypothesis-Driven Approach

### Process Flow

```
1. STRUCTURE THE PROBLEM
   └── Define issue tree

2. PRIORITIZE HYPOTHESES
   └── "We believe X because of Y"

3. DESIGN ANALYSES
   └── What data proves/disproves hypothesis?

4. CONDUCT ANALYSES
   └── Execute workplan

5. SYNTHESIZE FINDINGS
   └── So what? Now what?

6. COMMUNICATE RECOMMENDATIONS
   └── Pyramid structure presentation
```

### Example Hypothesis Set

**Problem**: Should Bank X enter digital lending?

| # | Hypothesis | Analyses Required |
|---|------------|-------------------|
| H1 | Market is attractive | Market size, growth, profitability analysis |
| H2 | We can win | Competitive analysis, capability assessment |
| H3 | We can execute | Resource requirements, timeline, risks |
| H4 | Returns justify investment | Business case, NPV, payback analysis |

## Engagement Management

### Work Planning

```
WORKSTREAM STRUCTURE

Project
├── Workstream 1: [Topic]
│   ├── Module 1.1
│   │   ├── Activity 1.1.1
│   │   └── Activity 1.1.2
│   └── Module 1.2
├── Workstream 2: [Topic]
└── Workstream 3: [Topic]

Each Activity has:
├── Owner
├── Duration
├── Dependencies
├── Deliverable
└── Status
```

### RACI Matrix

| Activity | Sponsor | PM | SME | Team |
|----------|---------|-----|-----|------|
| Scope Definition | A | R | C | I |
| Data Collection | I | A | R | R |
| Analysis | I | A | C | R |
| Recommendations | A | R | C | R |
| Final Report | A | R | C | R |

**R** = Responsible, **A** = Accountable, **C** = Consulted, **I** = Informed

## Quality Standards

### Review Checklist

**Content Quality**
- [ ] Addresses the actual business question
- [ ] Findings are supported by evidence
- [ ] Recommendations are actionable
- [ ] Risks and limitations acknowledged

**Analytical Quality**
- [ ] Data sources are credible
- [ ] Methodology is sound
- [ ] Calculations are verified
- [ ] Assumptions are stated

**Communication Quality**
- [ ] Key message is clear
- [ ] Structure is logical (MECE)
- [ ] Visuals are professional
- [ ] Free of errors
