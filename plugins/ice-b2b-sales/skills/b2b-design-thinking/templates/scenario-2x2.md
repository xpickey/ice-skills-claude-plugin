# Scenario Planning 2x2 Template

Scenario planning (Shell / GBN tradition) using the two most uncertain and most important axes to generate four plausible futures. Used to stress-test strategy against an uncertain world.

## Why Use It

Point forecasts are usually wrong. A strategy robust across multiple scenarios beats a strategy optimized for one. Enterprises software bets are multi-year — scenarios prevent over-commitment to a single future.

## Step-by-Step

### 1. List driving forces
PESTEL + industry-specific forces. Aim for 10–20.

### 2. Rank them
For each force, score:
- **Uncertainty** (1–5) — how unpredictable is it?
- **Importance** (1–5) — how much does it matter to our strategy?

### 3. Pick the two axes
Select the two forces highest on uncertainty × importance. These become your axes. Ideal axes are **independent** (not correlated).

### 4. Build the 2x2
Label each quadrant with a scenario name (evocative, not generic). Draft a 1-paragraph description of each.

### 5. Name scenario implications
For each scenario, ask:
- How does our business perform?
- What strategy would be optimal in this scenario?
- What signals would tell us this scenario is emerging?

### 6. Find the robust moves
Identify strategies that work across multiple scenarios. These are your no-regrets bets.

## Example — Thai Enterprise Software 2026–2030

### Axes
- **Axis X**: Cloud adoption pace in Thai enterprises (fast vs. slow)
- **Axis Y**: Regulatory stringency (high / strict vs. low / lenient)

### The Four Scenarios

```
                HIGH Regulation
                      │
    "COMPLIANCE       │     "COMPLIANCE
      MORASS"         │       RACE"
  slow cloud +        │   fast cloud +
  high reg = stuck    │   high reg = win
                      │    the local game
  ────────────────────┼──────────────────
                      │
    "STATUS QUO"      │    "DIGITAL
  slow cloud +        │     LEAP"
  low reg = drift     │   fast cloud +
                      │   low reg = global
                      │    playbook works
                LOW Regulation
                      │
```

### Scenario Descriptions

**Compliance Morass** — Slow cloud + strict regulation. Thai enterprises bogged in PDPA, Cloud Standard 2567, sectoral rules. Cloud adoption stalls. Advantage: whoever can deliver compliant on-prem / hybrid. Risk: low margin.

**Compliance Race** — Fast cloud + strict regulation. Enterprises race to the cloud, but only with compliance-credible partners. Huge win for partners with strong regulatory chops (ours if we invest early).

**Status Quo** — Slow cloud + lenient regulation. Market drift. Commoditized on-prem continues. Little new demand. Unattractive.

**Digital Leap** — Fast cloud + lenient regulation. Global playbook works. International vendors flood in; margins compressed by scale players.

### Strategic Implications

- **Robust moves (work in multiple scenarios):** invest in PDPA / Cloud Standard 2567 fluency (wins in Compliance Morass + Compliance Race); invest in cloud migration capability (wins in Compliance Race + Digital Leap)
- **Scenario-specific bets:** deep compliance IP / GRC offering (Compliance Morass); hyperscale partnership tier (Digital Leap)
- **Signals to monitor:** PDPA enforcement actions, MDES rulings on data localization, hyperscaler Thai data center build-out, Oracle / SAP / MSFT partner program changes

## Workshop Run (half-day to 1-day)

1. Brainstorm driving forces (45 min)
2. Score and rank (45 min)
3. Select axes (30 min)
4. Build 2x2 + name scenarios (45 min)
5. Narrate each scenario (60 min)
6. Score strategy robustness across scenarios (60 min)
7. Identify robust moves + contingencies (60 min)
8. Monitoring plan (30 min)

## Common Pitfalls

- **Axes too correlated** — if high regulation automatically means slow cloud, the 2x2 collapses to one diagonal
- **Scenarios too similar** — each quadrant must be materially different
- **No monitoring plan** — scenarios without signals are entertainment, not strategy
- **One "winning" scenario** — the team falls in love with the rosiest future and plans only for it

## Thai Context

Thai-specific driving forces worth considering as axes:
- BOI incentive continuity
- Regional geopolitical stability (Mekong, Indochina)
- Thailand's role in regional supply chain (China + 1)
- Baht volatility
- Domestic political cycle
- ESG reporting mandate timing

## Quality Bar

- Two axes are independent and high-uncertainty + high-importance
- Four scenarios are materially different and plausible
- Strategy robustness scored across scenarios
- Robust moves named (no-regrets)
- Signals and monitoring plan in place
