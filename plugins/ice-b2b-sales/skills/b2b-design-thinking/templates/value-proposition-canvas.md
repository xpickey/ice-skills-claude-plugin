# Value Proposition Canvas Template

Strategyzer's Value Proposition Canvas links user needs (the customer profile) to what a solution offers (the value map). Use it to test whether a proposed product, service, or workshop output actually solves the real problem.

## Two Halves

### Customer Profile (right side)
- **Customer Jobs** — functional, emotional, social jobs the customer is trying to get done
- **Pains** — bad outcomes, obstacles, risks
- **Gains** — desired outcomes, benefits, aspirations

### Value Map (left side)
- **Products & Services** — what we offer
- **Pain Relievers** — how our offer reduces specific pains
- **Gain Creators** — how our offer produces specific gains

## Structure

```
+-----------------------------------+-----------------------------------+
|        VALUE MAP                  |        CUSTOMER PROFILE           |
|                                   |                                   |
|        Products &                 |             Customer              |
|        Services                   |              Jobs                 |
|                                   |                                   |
|     Gain                Pain      |     Gains              Pains      |
|    Creators           Relievers   |                                   |
|                                   |                                   |
+-----------------------------------+-----------------------------------+
                         ↑
                         |
                    FIT = every pain / gain
                    matched by a reliever / creator
```

## Fit Test

A proposed solution has **problem-solution fit** when:
- Every major customer pain has a corresponding pain reliever
- Every major customer gain has a corresponding gain creator
- Pain relievers and gain creators are specific, not generic

Unmatched pains = adoption risk. Unmatched gains = missed value. Orphan pain relievers = wasted investment.

## Workshop Run

1. Fill the Customer Profile first (right side), from empathy work
2. Rank pains and gains by importance to the customer
3. Draft the Value Map (left side) — list all proposed products / services
4. Map each pain to a pain reliever; each gain to a gain creator
5. Highlight mismatches — pains without relievers, relievers without pains
6. Iterate the solution until fit is achieved

## Example — Thai Mid-Market Manufacturing CFO

### Customer Profile
- Jobs: close books fast; pass OAG audit; defend capex to board; comply with IFRS 9
- Pains: 12-day close; audit findings; customizations block upgrade; manual reconciliations
- Gains: 5-day close; clean audit; executive confidence; path to S/4HANA

### Value Map
- Products: SAP S/4HANA RISE + Thai localization + automated close accelerator
- Pain Relievers: pre-built close templates reduce manual work; localized e-Tax / e-GP connectors; clean-core migration path
- Gain Creators: 5-day close dashboard; audit-ready transaction trails; BOI-defensible capex plan

### Fit Assessment
- ✅ 12-day close → close accelerator (matched)
- ✅ OAG audit findings → audit-ready trails (matched)
- ⚠️ Manual reconciliations → need additional RPA or low-code build (gap)
- ❓ IFRS 9 compliance → consider FS Solutions add-on (open)

## Why It Matters for Enterprise Sales

The Value Proposition Canvas forces the sales team to articulate value in customer terms, not product terms. *"Our platform has AI"* is a feature. *"Our AI reduces your close from 12 days to 5, which means you pass the OAG audit and protect your BOI privileges"* is a value proposition.

Use the filled canvas as the source document for:
- Why-Invest / Why-Now / Why-Us narrative
- Proposal value section
- ROI model
- Objection handling

## Quality Bar

- Pains and gains are specific to this persona, not generic
- Relievers and creators trace to specific product capabilities (named, not abstract)
- At least one explicit gap is identified and flagged
- The canvas has been tested with at least one real customer interview
