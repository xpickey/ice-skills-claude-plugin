# Executive Readout Template

The one-page / one-deck synthesis delivered to C-suite after discovery, envisioning, or strategy work. Must pass the "will an executive read it in 5 minutes?" test.

## The Readout Structure

Five sections, in this order:

### 1. Answer (the headline)
One sentence. State the conclusion first, not the journey.

Template:
> *"We recommend [action] because [top reason], enabling [outcome] by [date]."*

Example:
> *"We recommend approving a ฿45M Oracle Fusion Cloud migration starting Q3 FY27 to close the books in 5 days and pass the FY28 OAG audit."*

### 2. Context (situation → complication)
Two sentences maximum.

- Situation — the current state the audience already accepts
- Complication — what is changing / pressing that requires a decision

Example:
> *"Monthly close runs 12 days on the aging EBS system. With ECC sunset 2027 and the IFRS 9 mandate for FY28, the current platform cannot meet audit or close timelines."*

### 3. Options (with trade-offs)
3 options minimum; never 1.

Format per option:
- Name
- Description (2 lines)
- Key benefit
- Key trade-off
- Approximate cost
- Risk rating

Include a "status quo / do nothing" option explicitly to highlight the cost of inaction.

### 4. Recommendation (with rationale)
One paragraph. Name the chosen option, explain why, and acknowledge what we are giving up.

Structure:
> *"We recommend [Option X] because it delivers [top 2 benefits] at [cost]. We are accepting the trade-off of [what we give up] because [rationale]. [Option Y] was considered but [reason to decline]."*

### 5. Ask (the specific next step)
One sentence. What do we need from the executive?

Examples:
- *"Approve capital of ฿45M and appoint a CFO-sponsor for the program."*
- *"Endorse the direction and task IT to prepare a full business case by [date]."*
- *"Decline Option A formally so we can redirect resources to Option B."*

## Output Formats

### One-page version (for memo / email)

```
## Recommendation
[one-sentence headline]

## Why
[situation + complication in 2 sentences]

## Options considered
- Option A — [one line]. Benefit / trade-off.
- Option B — [one line]. Benefit / trade-off.
- Option C (status quo) — [one line]. Benefit / trade-off.

## Why Option B
[one paragraph]

## What we need
[one sentence]
```

### Three-slide deck (for boardroom)

**Slide 1 — Recommendation**
- Action title = the recommendation
- Body: Why-Invest (pain), Why-Now (trigger), Why-Us (differentiator) in 3 columns

**Slide 2 — Options with trade-offs**
- Action title = "We considered three paths; Option B balances value and risk"
- Body: 3-column table of options with pros / cons / cost / risk

**Slide 3 — Ask and next steps**
- Action title = "We request [specific decision] by [date]"
- Body: decision needed, next 30/60/90 day actions, named owners

### Five-minute talk-track (for verbal readout)

- 30 sec — the recommendation
- 60 sec — situation and complication
- 120 sec — options and trade-offs
- 60 sec — why this recommendation
- 30 sec — the ask

## Language Discipline

- Short sentences, average ≤ 20 words
- Active voice
- Concrete numbers ("14% decline", not "significant decline")
- No jargon unless defined
- No vendor marketing speak

## Visual Discipline

- Every slide has an **action title** (the point), not a topic title
  - Bad: "Options Considered"
  - Good: "Three paths considered — Option B balances value and risk"
- Every chart title is the **insight**, not the metric
- One idea per slide
- Sources and dates on every data slide

## For Thai Audiences

- Bilingual parallel structure if the audience is mixed Thai-English
- Use Thai formal register (ภาษาทางการ) for written sections
- Emphasize risk mitigation and audit defensibility alongside financial uplift
- Cite Thai references / comparable peers when available
- Respect hierarchy in the ask — do not demand; propose

## Self-Check

- [ ] Reader gets the answer in the first 10 seconds
- [ ] 3+ options presented with explicit trade-offs
- [ ] Recommendation names the trade-off being accepted
- [ ] Specific ask with a date
- [ ] Every chart title is an insight
- [ ] Every slide title is an action
- [ ] Language is concrete; no generic marketing
- [ ] Thai elements present when audience is Thai
