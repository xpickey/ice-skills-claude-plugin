# Storyboard Template

A visual narrative (6–8 panels) showing the user experience of a proposed solution end to end. Turns abstract concepts into something testable. Used at the Prototype phase of Design Thinking.

## When to Use
- After ideation converged on a top concept
- Preparing a demo script for a customer validation session
- Building the narrative for a proposal or pitch deck
- Validating a user flow before UX / dev investment

## Structure

Six panels is the sweet spot. Each panel has:
- **Title** — one-line description of what happens
- **Visual** — sketch, screenshot mockup, or wireframe
- **Dialogue / thought bubble** — what the user says or thinks
- **System action** — what the system does in response

## Six-Panel Pattern

### Panel 1 — The Hook (before)
Show the user in their current pain state. The audience must feel the problem.

### Panel 2 — The Trigger
The moment the user engages with the solution. The doorway in.

### Panel 3 — The First Delight
Early moment of value — "oh, that is easier than I expected"

### Panel 4 — The Work
The main task. Show the solution doing the heavy lifting.

### Panel 5 — The Payoff
The outcome. What the user got out of it. Quantified where possible.

### Panel 6 — The Ripple
The downstream effect on the organization / customer / stakeholder.

## Example — AI Invoice Exception Handler (Thai Manufacturing)

| Panel | Title | Visual | User says / thinks | System action |
|---|---|---|---|---|
| 1 | Monday morning chaos | AP manager at desk, stack of PDFs | "Another 200 invoices to match." | — |
| 2 | Email alert | Laptop shows AI agent pinging | "Oh, the agent flagged 3 exceptions." | Agent completes 197, flags 3 |
| 3 | Clear context | Agent summary card | "It already pulled the PO and GR." | Shows 3-way match deltas |
| 4 | One-click resolution | Manager reviews, approves 2 of 3 | "Approve, approve, reject — one minute." | Posts to ERP, notifies supplier |
| 5 | Dashboard green | KPI tile shows 5-day close on track | "First time we are on pace." | Close dashboard updates real-time |
| 6 | CEO briefing | CEO in boardroom | "AP productivity up 3x." | BOI audit defensible |

## Rules

- **Show, do not tell** — every panel is visual
- **Start before the solution** — panel 1 must establish pain
- **End beyond the task** — final panel must show downstream value
- **Use real data / names** — specific is more credible than generic
- **Annotate metrics** — numbers make the story testable

## Variations

### Three-panel (rapid)
Before / During / After. Used in 60-minute workshops.

### Twelve-panel (detailed)
For complex multi-user flows; includes parallel actions by different personas.

### Demo script format
Time-stamped version used for executive demos:
- 0:00 — hook
- 0:30 — trigger
- 1:30 — first delight
- 3:00 — work
- 5:00 — payoff
- 6:00 — ripple

## Validation Method

After creating the storyboard, test it with:
1. **Sponsor user** — does the panel sequence match their real experience?
2. **Skeptic** — where does the story fall apart?
3. **Economic buyer** — does the payoff justify the investment?

Capture reactions on sticky notes next to each panel. Iterate.

## Quality Bar

- Sequence is logical and temporal
- Every panel has a concrete visual element
- At least one panel quantifies the value
- A stranger can understand the solution from the storyboard alone
- It is specific enough to build a prototype against
