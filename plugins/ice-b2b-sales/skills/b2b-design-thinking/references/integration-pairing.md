# Design Thinking ↔ Strategic Thinking Integration

The two disciplines are complementary, not substitutes. This file documents how they pair — what each produces, how to hand off between them, and when to run them together.

## Core Distinction

| Dimension | Strategic Thinking | Design Thinking |
|---|---|---|
| Primary question | Where to play / How to win | What to build / For whom |
| Direction | Top-down (market → firm → offer) | Bottom-up (user → solution → scale) |
| Time horizon | 1–5 years, Horizon-tagged | Sprint to one year |
| Unit of analysis | Market, firm, portfolio, account | User, journey, feature, moment |
| Core skill | Pattern recognition, synthesis, trade-off | Empathy, divergent ideation, prototyping |
| Key output | Choice cascade, strategy map, scenario set | Prototype, workshop kit, value case |
| Risk if missing | Build the wrong thing | Build for the wrong audience |

Rule of thumb: If the question is "should we?" use ST. If the question is "how might we?" use DT.

## The Integrated Flow

For any substantial enterprise software opportunity, run both in sequence:

### 1. Strategic gate (ST) — Where to Play
Produces:
- Account / vertical / geography priority list
- ICP (Ideal Customer Profile) definition
- Positioning guardrails
- Pricing envelope
- Non-goals (what we will NOT pursue)

### 2. Strategic gate (ST) — How to Win
Produces:
- Differentiation narrative
- Moat diagnosis (7 Powers)
- Partnership / channel choices
- Capability requirements

### 3. Customer discovery (DT) — Empathize
With accounts that pass the ST gates:
- Empathy interviews, stakeholder mapping
- Journey mapping (AS-IS)
- Pain / gain / job capture
- Regulatory and context scan

### 4. Problem framing (DT) — Define
- Point-of-View statements
- How Might We questions
- Opportunity sizing

### 5. Solution ideation (DT) — Ideate
- Divergent option generation
- Convergence on 2–3 solution concepts
- Value case draft

### 6. Strategic fit gate (ST)
Ask: does this opportunity still fit our Where / How? Kill, adjust, or accelerate. This is the critical gate many teams skip — without it, DT can pull the firm into the wrong deals.

### 7. Prototype & validate (DT)
- Storyboard, wireframe, demo script
- Value case refinement
- Sponsor user validation

### 8. Scale decision (ST)
For validated opportunities:
- Invest in offering (productize, H2 horizon)
- One-off delivery (H1 execution)
- Kill / park (does not fit portfolio)

### 9. Executive readout (DT + ST joint)
Produces the **Why Invest / Why Now / Why Us** narrative for the customer and the portfolio-impact narrative for internal leadership.

## Handoff Contracts

The two disciplines hand off concrete artifacts. Name them explicitly in every paired engagement.

### DT → ST (what DT feeds back to strategy)
- Customer pain insights (from empathy work)
- Unmet jobs discovered
- Willingness-to-pay signals
- Adoption blockers encountered
- New personas or segments identified
- Patterns across multiple customer engagements

### ST → DT (what strategy feeds into discovery)
- Target accounts list with priority
- ICP definition
- Positioning guardrails (what to emphasize, what to avoid)
- Pricing envelope (floor, ceiling, typical deal shape)
- Non-goals (accounts / problems to decline)
- Competitive positioning notes
- Horizon classification (H1 immediate, H2 emerging, H3 exploratory)

## Request Classification — Which Discipline(s) to Run

Use this classifier at the start of any task:

| User request signal | Discipline(s) to engage |
|---|---|
| "Empathize / journey / HMW / brainstorm / prototype" | DT only |
| "Where to play / how to win / portfolio / positioning" | ST only |
| "Full pre-sales envisioning / proposal / C-suite pitch" | DT + ST integrated |
| "Account strategy / market entry / competitive response" | ST primary, DT supporting |
| "Workshop kit for customer" | DT primary, ST embedded in readout |
| "Why invest / why now / why us" narrative | DT + ST synthesized for the narrative |
| "Internal innovation / new offering" | ST primary, DT for concept validation |

## Handling the Common Failure Modes

### Failure mode 1 — Running DT without ST
Symptom: brilliantly designed solutions in accounts that should not have been pursued. Customer loves the workshop, deal stalls or discounts heavily, or the offering is outside firm strategy.
Correction: always apply the ST strategic-fit gate (step 6 above) before prototyping.

### Failure mode 2 — Running ST without DT
Symptom: strategically sound bets that fail on customer adoption. The thesis is right but the user experience is wrong.
Correction: after any major ST decision, run a short DT discovery (empathy + journey) with 3–5 target customers before committing capital.

### Failure mode 3 — Running them in parallel without handoff
Symptom: strategy team and sales team produce contradictory messages; the proposal does not match the positioning; the customer senses incoherence.
Correction: enforce the handoff contracts. Name the DT→ST and ST→DT artifacts in every engagement kickoff.

## Workshop Patterns by Pairing

### Pattern A — Account Discovery Workshop (DT-led with ST bookends)
- Pre-workshop (ST): account strategy + ICP check + positioning brief
- Workshop (DT): empathy + journey + HMW + ideation
- Post-workshop (ST + DT): strategic fit gate + readout with Why-Invest/Now/Us

### Pattern B — Executive Envisioning Session (DT + ST integrated)
- Part 1 (ST lens): where the client is playing, where they should play
- Part 2 (DT lens): what the experience should be for their customers / employees
- Part 3 (DT + ST synthesis): priority initiatives, Horizon-tagged
- Output: executive readout with Why-Invest/Now/Us

### Pattern C — Internal Portfolio Session (ST-led with DT validation)
- Part 1 (ST): three-horizons portfolio review
- Part 2 (ST): build-vs-buy-vs-partner for each H2/H3 option
- Part 3 (DT): quick customer validation of top options
- Output: portfolio roadmap with capital allocation

## Closing Note

The skill should **name the discipline** at the start of any substantial response. Tell the user: *"I am running this as a DT exercise"* or *"this calls for ST because of X"* or *"we need both — here is the handoff plan"*. Named discipline choices make the deliverable more credible and the reasoning auditable.
