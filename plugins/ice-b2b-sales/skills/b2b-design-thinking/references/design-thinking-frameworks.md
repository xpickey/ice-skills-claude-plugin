# Design Thinking Frameworks — Pick the Right Tool

Use this catalog to select the right Design Thinking framework for the task. Do not apply all of them every time — pick what fits the question.

## The Five-Phase Default (Stanford d.school)

**Empathize → Define → Ideate → Prototype → Test**

Use as the default backbone for any workshop. Each phase has its own method set:

### Empathize — Know the user
- **Empathy interviews** — open-ended, what/how/why questions, 30–60 min per stakeholder
- **Shadowing / contextual inquiry** — observe in the real work context
- **Empathy map** — Say / Think / Do / Feel quadrants
- **Persona canvas** — goals, frustrations, behaviors, context
- **Journey map (AS-IS)** — stages, actions, touchpoints, pain, emotions
- **Jobs-to-Be-Done interview** — the "progress" being sought, functional/social/emotional jobs

### Define — Frame the problem
- **Point-of-View statement** — *"[User] needs [need] because [insight]"*
- **How Might We (HMW)** — convert pain into a generative question
- **Problem Tree / 5 Whys / Fishbone** — root cause
- **Opportunity canvas** — problem / outcome / solution / metric

### Ideate — Generate options
- **Crazy 8s** — 8 ideas in 8 minutes, one per panel
- **6-3-5 Brainwriting** — 6 people × 3 ideas × 5 minutes, rotate sheets
- **SCAMPER** — Substitute, Combine, Adapt, Modify, Put to another use, Eliminate, Reverse
- **Worst Possible Idea** — inversion to break fixation
- **Analogous inspiration** — borrow from unrelated industries
- **Mind mapping** — branch associations
- **Silent ideation** — critical for hierarchical Thai/APAC rooms
- **Convergence tools**: Dot voting, Impact-Effort 2×2, MoSCoW, RICE, Now/Next/Later

### Prototype — Make it tangible
- **Storyboard** — 6-panel user scenario
- **Wireframe / paper sketch** — click-through
- **Demo script** — scenario-based, persona-driven
- **Envisioning slides** — Oracle/SAP/MSFT style narrative deck
- **Value case prototype** — ROI model, TCO, payback
- **PoC scope** — minimum viable proof

### Test — Validate and iterate
- **Usability test** — 5-user rule
- **Business-value test** — the "so what" check
- **A/B framing** — let stakeholders choose between two solution options
- **Pivot / persevere / kill** — explicit decision gate

## Alternative / Complementary Frameworks

### Double Diamond (UK Design Council)
Discover → Define → Develop → Deliver. Visualizes divergence/convergence twice. Best for executive briefings where the 5-phase is too granular.

### IDEO Human-Centered Design (HCD)
Hear → Create → Deliver. Strong on field research; use when voice-of-customer is the bottleneck.

### IBM Enterprise Design Thinking
Hills (big aspirations) → Playbacks (regular reviews with sponsor users) → Sponsor Users (real customers embedded). Best for scaling DT across large enterprise programs.

### SAP AppHaus Method
Blends pre-sales envisioning with implementation. Use when selling SAP S/4HANA or RISE.

### Google Design Sprint (Knapp)
5-day time-boxed format: Monday Map → Tuesday Sketch → Wednesday Decide → Thursday Prototype → Friday Test. Use to force a decision on a stuck deal or product question.

### PwC BXT (Business × eXperience × Technology)
Big Four sales model — three lenses held in tension. Use for proposal workshops where all three must balance.

## Framework Selection Heuristic

| User situation | Recommended framework |
|---|---|
| First customer call, discovery needed | 5-phase, start at Empathize |
| Stuck deal, need to force a decision | Google Design Sprint (5-day or compressed 3-day) |
| C-suite envisioning session | Double Diamond + BXT |
| Large multi-stakeholder program | IBM Enterprise DT (Hills + Playbacks) |
| SAP-specific pre-sales | SAP AppHaus |
| Cross-industry inspiration needed | IDEO HCD with analogous research |
| Pricing / value-case focus | BXT with heavy Business lens |
| AI use-case ideation | 5-phase with Crazy 8s + impact-effort matrix |

## Convergence — How to Narrow Down

After divergent ideation, always converge. Pick **one** convergence tool, not many:

- **Dot voting** — fast, democratic, 3–5 dots per person
- **Impact-Effort 2×2** — plot ideas on impact (y) vs effort (x); pick top-right quadrant
- **RICE** — Reach × Impact × Confidence ÷ Effort
- **MoSCoW** — Must / Should / Could / Won't (for scope)
- **Now / Next / Later** — for roadmap output

## Anti-Patterns to Avoid

- Running ideation without empathy first (produces irrelevant ideas)
- Skipping convergence (leaves the customer with too many options)
- Single-persona empathy (enterprise deals have 5–12 stakeholders)
- Over-polished prototypes (signals premature commitment; use low-fi)
- Testing with internal stakeholders only (needs real sponsor users)
