# ขั้นตอนการปฏิบัติงาน GFMIS โดยละเอียด

## การสร้างข้อมูลหลักผู้ขาย (Vendor Master Creation)

### เอกสารที่ต้องใช้
1. สำเนาสัญญาซื้อขาย/จ้าง/เช่า
2. ใบสั่งซื้อ/ใบสั่งจ้าง
3. หนังสือรับรองบริษัท + สำเนาบัตรประชาชนกรรมการ
4. สำเนาหน้าสมุดบัญชีธนาคาร
5. แบบฟอร์มขออนุมัติข้อมูลหลักผู้ขาย (Vendor Master Request Form)

### ขั้นตอน (ตามหนังสือ กค. ด่วนที่สุด ที่ กค 0409.3/ว 88 ลว. 4 ต.ค. 2549)
1. หน่วยงานจัดทำแบบขออนุมัติข้อมูลหลักผู้ขาย
2. หัวหน้าส่วนราชการ/ผู้มีอำนาจลงนามรับรอง
3. ส่งมายัง กรมบัญชีกลาง หรือ สำนักงานคลังจังหวัด
4. กรมบัญชีกลาง/คลังจังหวัดบันทึก Vendor Master ในระบบ
5. แจ้งกลับหน่วยงาน เพื่อดำเนินการขอเบิกเงิน

## การบันทึกการขอเบิกเงิน (Payment Request)

### กระบวนการ GFMIS Web Online
```
Step 1: Login ด้วย Username/Password หรือ Token Key
Step 2: เลือกเมนู "การเบิกเงิน" → "บันทึกขอเบิกเงิน"
Step 3: เลือกประเภทงบรายจ่าย และหน่วยงาน
Step 4: กรอกข้อมูลเจ้าหนี้ (Vendor ID) ยอดเงิน และรายละเอียด
Step 5: แนบเอกสารหลักฐาน (Digital Copy)
Step 6: ผู้มีสิทธิ์อนุมัติ → Approve ในระบบ
Step 7: กรมบัญชีกลาง → ตรวจสอบและโอนเงินเข้าบัญชีเจ้าหนี้
Step 8: หน่วยงาน → ตรวจสอบผลการจ่ายจากรายงานในระบบ
```

### การตรวจสอบประจำวัน (Daily Reconciliation)
ทุกสิ้นวันทำการ:
1. รายงานสรุปการขอเบิกเงินกับข้อมูลการขอเบิกของส่วนราชการ
2. รายงานเงินสดคงเหลือประจำวันจากระบบ GFMIS กับรายงานเงินคงเหลือนอกระบบ
3. หากพบข้อคลาดเคลื่อน → พิสูจน์ยอดก่อนพิมพ์รายงาน → เสนอหัวหน้าส่วนราชการลงนาม

## การกันเงินเบิกเหลื่อมปี

### เงื่อนไขการกันเงิน
- มีการก่อหนี้ผูกพันก่อนสิ้นปีงบประมาณ
- วงเงิน ≥ 100,000 บาท
- ไม่สามารถเบิกได้ทันสิ้นปีงบประมาณ

### ระยะเวลา
- กันเงินครั้งแรก: ขยายได้ถึง 31 มีนาคมของปีงบประมาณถัดไป (6 เดือน)
- ขยายครั้งที่ 2: ทำความตกลงกับ กค. → ได้อีก 6 เดือน (ถึง 30 ก.ย.)
- หากยังไม่ได้เบิก: เงินจะถูก Lapse (หมดอายุ)

### ขั้นตอนการกันเงิน
1. ดำเนินการก่อนสิ้นปีงบประมาณ (ก่อน 30 ก.ย.)
2. บันทึกข้อมูลในระบบ GFMIS ตามวิธีการที่ กค. กำหนด
3. ส่งข้อมูลพร้อมเอกสารหลักฐานการก่อหนี้

## รหัสงบประมาณ (Budget Coding) ใน GFMIS

### โครงสร้างรหัส
```
[หน่วยงาน] [แผนงาน] [ผลผลิต] [กิจกรรม] [งบรายจ่าย] [รายการ]
ตัวอย่าง:
- 02 = กระทรวงการคลัง
- 14302 = แผนงานพัฒนาระบบการเงินการคลัง
- 01 = งบบุคลากร, 02 = งบดำเนินงาน, 03 = งบลงทุน
```

### ประเภทงบรายจ่าย (Budget Category)
| รหัส | ประเภท | IT ที่เกี่ยวข้อง |
|------|--------|----------------|
| 01 | งบบุคลากร | เงินเดือน IT Staff |
| 02 | งบดำเนินงาน | ค่า Software License, ค่าบำรุงรักษาระบบ, Cloud Subscription |
| 03 | งบลงทุน | ซื้อ Hardware, พัฒนาระบบ (Capex) |
| 05 | งบอุดหนุน | เงินอุดหนุน IT แก่หน่วยงานอื่น |
| 06 | งบรายจ่ายอื่น | ค่าใช้จ่ายอื่นๆ |

## ข้อควรระวังสำหรับ IT Projects ใน GFMIS

1. **Software License (Subscription)**: จัดเป็น **งบดำเนินงาน** (รายจ่ายประจำ) ไม่ใช่งบลงทุน
2. **การพัฒนาระบบ (Development)**: จัดเป็น **งบลงทุน** → เมื่อรับงานแล้ว ต้องบันทึกเป็นสินทรัพย์ไม่มีตัวตน (Intangible Asset)
3. **Cloud Services (SaaS/IaaS)**: จัดเป็น **งบดำเนินงาน** ตามหนังสือกรมบัญชีกลางที่เกี่ยวข้อง
4. **Annual Maintenance**: งบดำเนินงาน; ต้องมีสัญญาการบำรุงรักษาก่อนเบิกจ่าย
