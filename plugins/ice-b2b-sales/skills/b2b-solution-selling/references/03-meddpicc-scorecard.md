# Reference 03 — MEDDPICC Deal Qualification Scorecard

**Use when:** Qualifying any opportunity in the pipeline. Running weekly pipeline review. Building forecast for the quarter. Diagnosing why a deal is stuck. Coaching a seller whose forecast keeps slipping.

MEDDPICC originated at PTC (Dunkel) and was popularised by Jack Napoli and Andy Whyte. It is a discipline for honest qualification — not a sales pitch.

---

## A. The 8 elements (score each 0–3)

| Letter | Element | What good looks like | Score 3 | Score 2 | Score 1 | Score 0 |
|---|---|---|---|---|---|---|
| **M** | **Metrics** | Quantified business outcome the EB has agreed to | THB / days / % numbers signed off by EB | Numbers exist but EB has not validated | Generic "improve efficiency" language | None |
| **E** | **Economic Buyer** | The single person who can spend the money | Named, met in last 21 days, acknowledged budget exists | Named, met once, budget unclear | Identified by inference, never met | Unknown |
| **D** | **Decision Criteria** | Technical + business + relationship criteria documented | Procurement-issued evaluation matrix in hand | Verbal criteria from Champion | "We'll know it when we see it" | None |
| **D** | **Decision Process** | Steps + dates + decision-makers per gate | Documented process with dates committed | Steps known, dates fluid | "Sometime this quarter" | Unknown |
| **P** | **Paper Process** | Legal, procurement, security, audit reviews mapped | Legal kickoff scheduled; security questionnaire in flight; procurement template received | Legal aware; procurement TBD | Discovered post-verbal-yes | Will be a surprise |
| **I** | **Identify Pain + Compelling Event** | Critical Pain admitted + a deadline that forces decision | Pain Sheet completed + deadline-driven event (regulatory, EOL, audit, M&A) | Pain admitted, no deadline | Pain hypothesised, not validated | None |
| **C** | **Champion** | Has power, has influence, AND has taken an action for you in the last 14 days | Champion has done something uncomfortable for you (got you the EB meeting, blocked competitor, defended budget) | Champion is supportive but passive | Friend, not Champion | None |
| **C** | **Competition** | Named alternatives + status quo + your position vs each | Named 2–3 alternatives, know their pricing posture, know which way EB is leaning | Named alternatives, fuzzy on positioning | Assume you are the only vendor | Don't know |

## B. The traffic-light rule

- **GREEN** — score 2 or 3 on this element
- **YELLOW** — score 1
- **RED** — score 0

**Forecast rule:** No deal moves to commit forecast unless **7 of 8 elements are GREEN**. This is the single discipline that separates accurate-forecasting sales orgs from chronic-misser sales orgs.

If a seller wants to forecast a deal that is not 7-of-8 green, the manager's first question is: "Which element will turn green this week, and what is the action?"

## C. Champion vs Coach vs Friend (the most common confusion)

| Type | Has power | Has influence | Will risk something for you | Useful for |
|---|---|---|---|---|
| **Champion** | Yes | Yes | Yes — has done it in last 14 days | Driving the deal |
| **Coach** | No | Some | Sometimes — gives intel | Information, intel, navigation |
| **Friend** | No | No | No — likes you but won't fight | Coffee. That is all. |

**Champion litmus test (ask weekly):** "What did the Champion do for me this week?" If the answer is "nothing" twice in a row, you have a Coach, not a Champion. Find a real Champion or de-risk the deal in CRM.

## D. Compelling Event examples (Enterprise Software)

A compelling event is an external or internal deadline that forces the buyer to decide *this period* rather than *some day*.

| Type | Example |
|---|---|
| Regulatory | "New e-Tax Invoice rules require system change by 31 Dec — current system cannot comply" |
| System EOL | "Oracle EBS 12.1 Premier Support ended; sustaining support cost up 30% per year" |
| Post-M&A integration | "Acquired three companies; must consolidate four ERPs into one within the 18-month integration mandate from the board" |
| Audit finding | "External audit flagged segregation-of-duties gap; remediation deadline is next year-end audit" |
| Infrastructure crisis | "On-prem server EOL; data centre lease expires in 14 months — cloud-first decision required" |
| Cost pressure | "Legacy maintenance costs THB 90M/year; CFO has mandated 30% IT cost reduction by FY27" |
| Market / competitive | "Competitor launched real-time supply chain visibility; we are losing tier-1 retail customers without it" |
| Government policy | "BOI promotion requires e-invoicing readiness as a condition of tax holiday" |

If the seller cannot name a compelling event with a date, **the deal is not real this period** — even if the customer is enthusiastic.

## E. Weekly seller disciplines (mandatory cadence)

| Discipline | Frequency | What it produces |
|---|---|---|
| MEDDPICC scorecard refresh per deal | Weekly | Updated red/yellow/green per element |
| Champion activity log | Weekly | What did Champion do for you this week? If nothing for 2 weeks → escalate or downgrade |
| Competition audit | Weekly | Named alternatives + your position relative to each |
| Paper Process check-in | Monthly (or when stage advances) | Legal, security, procurement timelines surfaced — no end-of-quarter surprises |
| Economic Buyer touchpoint | Every 21 days minimum | EB attends at least one deal review per month — if not, deal is yellow at best |
| Pipeline review (with manager) | Weekly | Manager asks MEDDPICC questions; deal moves stage only when 7/8 green |

## F. Output-format guidance

When the user asks for a MEDDPICC scorecard, deliver:

1. **The 8-element table** with score, evidence (one sentence of what is known), and a colour (GREEN / YELLOW / RED)
2. **Overall deal status** — Forecast / Best Case / Pipeline / Disqualify
3. **The 3 weakest elements** with a specific action to turn each one green this week
4. **One question to ask the Champion at the next meeting** that would advance the weakest element
5. **A "kill criteria"** — name the condition under which the seller will walk away (without this, the deal will rot in pipeline forever)

## H. Common failure modes

- **Coach mistaken for Champion.** The Champion test ("what have they done for you in 14 days") is the cure.
- **EB never met.** Deal forecast is fiction. Until EB sits in a meeting with you, the deal is YELLOW at best.
- **Paper Process discovered post-verbal-yes.** This kills more enterprise deals than any technical objection. Map it in week 2 of the cycle, not week 22.
- **Compelling event invented by seller.** A real CE is admitted by the buyer in writing. If only the seller can articulate it, it is not real.
- **Sandbagging the green count.** Sellers who routinely forecast 5/8-green deals miss quota; those who only forecast 7+/8 hit consistently.
