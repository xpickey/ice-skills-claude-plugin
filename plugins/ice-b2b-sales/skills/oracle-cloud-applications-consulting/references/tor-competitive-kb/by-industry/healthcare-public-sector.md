## F-EPM-01 — Driver-based planning & budgeting (xP&A)

- **Capability (TH):** การวางแผน/งบประมาณแบบ driver-based
- **Capability (EN):** Driver-based planning & budgeting
- **Domain:** EPM · **iCE severity:** ต่ำ (Low)
- **Oracle Fusion advantage (Standout):** Oracle EPM Cloud (EPBCS) is a market-leading, native enterprise planning platform: driver-based planning across **workforce, capital and project** dimensions with built-in statistical/predictive forecasting and multi-scenario what-if — all first-party on one Hyperion-lineage engine. NetSuite's core ERP has **no predictive/multi-scenario planning in the box**; that capability lives in NSPB, an OEM-trimmed edition of Oracle PBCS that currently ships **only the Financials module** ("A Workforce module is not currently available," per Oracle Help). For a healthcare buyer running hospitals / blood-service centres / biologics units with real headcount and capex to budget, Fusion delivers pre-built Workforce and Capital planning that NetSuite cannot match natively.
- **TOR wording to weaponize (ภาษาไทย):** "ระบบต้องมีแพลตฟอร์มการวางแผนและงบประมาณแบบ driver-based ในตัว (native) ที่รองรับ workforce, capital และ project planning พร้อมการพยากรณ์เชิงสถิติและการจำลองสถานการณ์ (predictive / what-if) โดยไม่ต้องใช้เครื่องมือภายนอก"
- **TOR wording (English):** "The solution must provide a native driver-based planning and budgeting platform supporting workforce, capital and project planning with built-in statistical forecasting and what-if scenario modelling, without any external tool."
- **iCE caveat:** The "no predictive/scenario" premise behind a knock-out is over-stated. **NSPB is a NetSuite first-party add-on built on the same Oracle PBCS/Hyperion engine as Oracle EPM Cloud** — it does driver-based, what-if/multi-scenario, Predictive Planning and rolling forecast with automatic GL sync, so a bidder can cover this outcome. The real limit is that NSPB is a **separately-licensed add-on** without pre-built Workforce/Capital modules — not a missing core. For a single Thai charitable/public entity, headcount/capex budgeting runs through driver-based models in NSPB Financials; mandating "native workforce/capital modules + enterprise predictive multi-scenario … without any external tool" is **over-spec** and tilts the spec toward one suite — a สตง. single-vendor-lock exposure. Prefer outcome-based wording (must produce driver-based + scenario budgets).
- **Confidence:** high

## F-EPM-04 — Multidimensional profitability & activity-based costing

- **Capability (TH):** การวิเคราะห์กำไร/ต้นทุนหลายมิติ
- **Capability (EN):** Multidimensional profitability & costing
- **Domain:** EPM · **iCE severity:** ต่ำ (Low)
- **Oracle Fusion advantage (Standout):** Oracle PCMCS provides a purpose-built, multidimensional profitability and cost-allocation engine for full activity-based costing (ABC) — model-driven allocation waterfalls, traceable cost flows and profitability by many dimensions, native in the EPM suite. NetSuite core has **no full ABC engine**; ABC/multidimensional profitability at PCMCS grade requires a separately-licensed add-on. For a foundation that must allocate indirect cost across programmes / funds / cost centres and report profitability by multiple axes, Fusion's native PCMCS is a genuine class-above.
- **TOR wording to weaponize (ภาษาไทย):** "ระบบต้องรองรับการวิเคราะห์กำไรและต้นทุนแบบหลายมิติ (multidimensional profitability & cost allocation) สำหรับ activity-based costing ในตัว"
- **TOR wording (English):** "The solution must support native multidimensional profitability and cost-allocation modelling for activity-based costing."
- **iCE caveat:** "No engine" is over-stated. NetSuite **native** has Allocation Schedules (Fixed and Dynamic) spreading cost across account/department/class/location/custom segment, weighted by Statistical Accounts — which NetSuite Help itself calls "useful in advanced costing such as Activity Based Costing and Usage Based Costing" — plus Custom Segments and Project Profitability. A full ABC engine exists as **first-party NSPCM** (NetSuite Profitability & Cost Management) on the same PCMCS/Hyperion platform. For charitable/public work, allocating indirect cost into programmes/funds/cost centres via native allocation is enough; mandating a "native multidimensional ABC engine" ties the spec to a PCMCS-class product and is over-spec. Prefer an outcome clause (must allocate cost into programmes/funds).
- **Confidence:** high

- **Capability (TH):** Process / Mixed-mode Manufacturing
- **Capability (EN):** Process / mixed-mode manufacturing
- **Domain:** Manufacturing · **iCE severity:** สูง (High)

NetSuite's standard base is **light discrete only**; process manufacturing (recipe/formula, batch genealogy, co-product) requires the add-on NetSuite Advanced Manufacturing, and lot/expiry is native but FEFO ships as a SuiteApp to install. **The part that is genuinely hard to close on NetSuite is GMP-grade / biologics process manufacturing — regulatory electronic batch record and potency — which has no native answer.** For a serum/vaccine (biologics) unit under GMP, this is a real Fusion win.

What is real and hard is GMP electronic batch record + potency for the serum/vaccine unit — that needs **custom SuiteScript or an external SuiteApp (blendAPPS / BatchMaster)** planned before go-live, but it is a targeted gap, not a whole-of-manufacturing blocker; blood-bank lot/expiry/co-product work is already sufficient. So weaponize the **GMP batch record + potency** clause honestly (real), but avoid a "full mixed-mode manufacturing native" mandate that pulls toward SAP PP-PI / Oracle Manufacturing with no matching use case — that is over-spec / lock-spec.
